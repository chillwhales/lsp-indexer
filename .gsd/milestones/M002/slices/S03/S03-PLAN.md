# S03: AI Compatibility тАФ llms.txt, llms-full.txt, .md Sidecars

**Goal:** Add three AI-agent HTTP surfaces to `apps/docs`: `/llms.txt` (project summary for LLMs), `/llms-full.txt` (all docs concatenated), and static `/llm/{slug}.md` files served from `public/llm/` тАФ generated by a script from the MDX sources, committed to the repo. The `/llm/` path avoids any collision with Fumadocs's `[[...slug]]` catch-all under `/docs/`.
**Demo:** With `pnpm --filter docs dev` running: `curl http://localhost:3001/llms.txt` returns a plain-text project summary; `curl http://localhost:3001/llms-full.txt` returns all 5 docs pages concatenated; `curl http://localhost:3001/llm/quickstart.md` returns the Quickstart page as raw markdown with no frontmatter. Then run `pnpm --filter docs generate` тАФ it writes/updates all `.md` files in `public/llm/`. Run it again with `--check` тАФ exits 0. Modify a source MDX, run with `--check` тАФ exits 1.

## Must-Haves

- `apps/docs/scripts/generate-md.mjs` тАФ when called without flags, reads each `content/docs/{slug}.mdx`, strips frontmatter, writes `public/llm/{slug}.md`; when called with `--check`, compares what would be written to what's on disk and exits 1 with a descriptive message if any file is stale or missing
- `public/llm/quickstart.md`, `indexer.md`, `node.md`, `react.md`, `next.md` тАФ generated, committed to repo, served as static assets by Next.js at `/llm/{slug}.md`
- `GET /llm/quickstart.md` (and the other 4 slugs) returns raw markdown via Next.js static file serving тАФ no Fumadocs route collision
- `GET /llms.txt` тАФ route handler returning plain-text project summary; per-page links use the `/llm/{slug}.md` pattern
- `GET /llms-full.txt` тАФ route handler returning all 5 docs concatenated
- `"generate": "node scripts/generate-md.mjs"` script in `apps/docs/package.json`
- `"generate:check": "node scripts/generate-md.mjs --check"` script in `apps/docs/package.json`
- `pnpm --filter docs build` exits 0

## Verification

- `pnpm --filter docs generate` exits 0 and writes/updates all 5 `.md` files in `public/llm/`
- `pnpm --filter docs generate:check` exits 0 immediately after generate
- Append one word to `content/docs/quickstart.mdx`, run `pnpm --filter docs generate:check` тАФ exits 1 naming `quickstart`
- Revert and rerun тАФ exits 0
- In dev server: `curl -s http://localhost:3001/llm/quickstart.md | head -3` returns a markdown heading (no `---` frontmatter)
- In dev server: `curl -s http://localhost:3001/llms-full.txt | wc -l` returns > 100

## Tasks

- [x] **T01: generate-md.mjs script** `est:25m`
  - Why: The script is the source of truth for .md sidecar generation and the CI check тАФ both generate mode and check mode use the same strip-frontmatter logic
  - Files: `apps/docs/scripts/generate-md.mjs`, `apps/docs/package.json`
  - Do: (1) Create `apps/docs/scripts/generate-md.mjs` as a Node.js ESM script. Define `SLUGS = ['quickstart', 'indexer', 'node', 'react', 'next']`. For each slug: read `content/docs/{slug}.mdx` as UTF-8, strip the leading frontmatter block (`/^---[\s\S]*?---\n*/`), trim leading whitespace. (2) In default (generate) mode: `mkdirSync('public/llm', { recursive: true })`, write `public/llm/{slug}.md`. Print which files were written. Exit 0. (3) In `--check` mode: for each slug, read the existing `public/llm/{slug}.md` (if it doesn't exist, it's stale). Compare to the stripped MDX content. Collect all stale slugs. If any stale, print each slug name + "needs update" and exit 1. If all match, print "All sidecars up to date" and exit 0. (4) Parse `--check` from `process.argv`. (5) Add `"generate": "node scripts/generate-md.mjs"` and `"generate:check": "node scripts/generate-md.mjs --check"` to `apps/docs/package.json` scripts. (6) Run `pnpm --filter docs generate` тАФ verify 5 `.md` files created in `public/llm/`.
  - Verify: Script exits 0 in both modes on clean state; exits 1 after modifying a source MDX
  - Done when: Both modes work as described; 5 `.md` files present in `public/llm/`

- [x] **T02: llms.txt and llms-full.txt route handlers** `est:20m`
  - Why: AI agents need a quick-context summary (llms.txt) and a full-content dump (llms-full.txt) тАФ served as route handlers since they're dynamic aggregations, not static files
  - Files: `apps/docs/app/llms.txt/route.ts`, `apps/docs/app/llms-full.txt/route.ts`
  - Do: (1) Create `apps/docs/app/llms.txt/route.ts` тАФ export `GET()` returning `Response` with `Content-Type: text/plain`. Content: project name, tagline, list of 5 docs pages with one-line descriptions, and note: "Per-page markdown: /llm/{slug}.md". (2) Create `apps/docs/app/llms-full.txt/route.ts` тАФ export `GET()` that reads all 5 `public/llm/{slug}.md` files (already generated, no stripping needed), concatenates with `\n\n---\n\n# {title}\n\n` separators (hardcode the 5 titles), returns as `text/plain`. Reading from `public/llm/` keeps this handler lean тАФ it reuses the already-generated sidecars.
  - Verify: `curl /llms.txt` returns non-empty text; `curl /llms-full.txt` returns content > 100 lines
  - Done when: Both handlers return correct content in dev server; build passes
