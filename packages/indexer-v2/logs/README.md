# Indexer V2 Logs

This directory contains log files generated by the indexer v2.

## Log Files

### Structured Logs (JSON)
- **`indexer-YYYY-MM-DD.log`** - Daily rotating JSON logs from pino
  - Contains structured log data from the pipeline
  - Includes all pipeline steps: EXTRACT, PERSIST_RAW, HANDLE, PERSIST_DERIVED, VERIFY, ENRICH
  - Each log entry includes: timestamp, level, step, blockRange, and contextual data
  - Rotates daily at midnight

### Console Logs (Plain Text)
- **`console-YYYY-MM-DD.log`** - Daily console output from the indexer
  - Captures both stdout and stderr
  - Includes all console output from the process
  - Appends to file while also displaying in terminal via `tee`

## Configuration

### Environment Variables

- **`LOG_LEVEL`** - Set log level (debug, info, warn, error)
  - Default: `debug` in development, `info` in production
  
- **`LOG_DIR`** - Override log directory location
  - Default: `./logs`

### Examples

```bash
# Run with debug logging
LOG_LEVEL=debug pnpm start

# Run with custom log directory
LOG_DIR=/var/log/indexer pnpm start

# Run without file logging (console only)
pnpm start:simple
```

## Scripts

- **`pnpm start`** - Runs indexer with dual logging (console + files via tee)
- **`pnpm start:simple`** - Runs indexer with console output only (no tee)

## Log Rotation

- JSON logs rotate daily via `pino-roll`
- Console logs create new files per day (via filename timestamp)
- Old logs are kept indefinitely (manual cleanup required)

## Debugging Tips

### View live logs
```bash
# Follow JSON logs (pretty-printed)
tail -f logs/indexer-$(date +%Y-%m-%d).log | pnpm exec pino-pretty

# Follow console logs
tail -f logs/console-$(date +%Y-%m-%d).log
```

### Search logs
```bash
# Find errors in JSON logs
cat logs/indexer-*.log | grep '"level":50'

# Find specific block range
cat logs/indexer-*.log | grep '1000000-1001000'

# Search console logs
grep "ERROR" logs/console-*.log
```

### Filter by pipeline step
```bash
# Only VERIFY step logs
cat logs/indexer-*.log | grep '"step":"VERIFY"'

# Only ENRICH step errors
cat logs/indexer-*.log | grep '"step":"ENRICH"' | grep '"level":50'
```
