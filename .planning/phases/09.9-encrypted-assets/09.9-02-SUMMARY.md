## Decisions Made

- **LSP29 images use domain-specific parser:** `parseEncryptedAssetImage` has `imageIndex` and `verificationSource` fields not present in LSP4 images — separate from shared `parseImage` in `parsers/utils.ts`
- **fileSize filter uses `_gte` with `String()` conversion:** Hasura `numeric` scalar input type is `string` per codegen — `String(filter.fileSize)` ensures correct type
- **revision filter uses `_eq` (not `_ilike`):** Revision is `Int` type in Hasura, not a string — exact numeric comparison, not case-insensitive pattern matching
- **universalProfile parsed without sub-include in result construction:** Uses `as Parameters<typeof parseProfile>[0]` cast for structural subtyping; stripExcluded with nestedConfig `{ universalProfile: { baseFields: ['address'] } }` handles recursive sub-field stripping — same pattern as creators and issued assets
