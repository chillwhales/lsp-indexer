## Decisions Made

- **ImageList generic shape:** Accepts `ReadonlyArray<{ url?: string | null; [key: string]: unknown }>` — works for both LSP4 images (with `verification`) and LSP29 images (with `imageIndex`, `verificationSource`). No type narrowing needed at component level.
- **Card title strategy:** Shows title text when available, otherwise truncated address. ContentId shown as secondary monospace text in header.
- **Encryption collapsible design:** Uses ghost Button + ShieldCheck icon + ChevronDown (matching unified collapsible style). Shows method, ciphertext (truncated at 100 chars), data hash, decrypt code. Access control conditions rendered as bordered cards with all condition fields.
- **File size formatting:** Uses `toLocaleString()` for human-readable byte counts (e.g., "1,024 bytes")
- **3-field composite getKey:** `${ea.address}-${ea.contentId ?? ''}-${ea.revision ?? ''}` for ResultsList unique keys — encrypted assets have no single natural key.
