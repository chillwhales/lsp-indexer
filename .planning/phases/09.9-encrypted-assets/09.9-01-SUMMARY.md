## Decisions Made

- **Encryption include as union type:** `z.union([z.boolean(), EncryptedAssetEncryptionIncludeSchema])` — gives consumers fine-grained control: `true` fetches everything including access control conditions, `{ accessControlConditions: false }` fetches encryption scalars only
- **ResolveEncryption handles 4 cases:** true → full, { acc: true } → full, {} or { acc: false } → base only (no accessControlConditions), false/omitted → field absent
- **LSP29 images get their own schema:** `EncryptedAssetImageSchema` has `imageIndex`, `verificationSource` fields that LSP4 `ImageSchema` lacks — separate schema prevents type confusion
- **`followed_aggregate` for "following count":** Profile sub-document uses `followed_aggregate` (not `following_aggregate`) — matches established pattern from profiles document
