**Create `apps/test/src/components/image-list.tsx`** — reusable component for rendering flat arrays of images.

Per CONTEXT.md: "New component handles flat array of images (thumbnail + URL + metadata). Create and backport to all existing cards in this phase."

**Design decisions (Claude's discretion per CONTEXT.md):**

The component must handle two image shapes:

1. **LSP4 images** (profiles, digital assets, NFTs): `{ url: string; width?: number | null; height?: number | null; verification?: { method: string; data: string } | null }` — from `Image` type in `@lsp-indexer/types`
2. **LSP29 images** (encrypted assets): `{ imageIndex: number; url: string | null; width?: number | null; height?: number | null; verificationData?: string | null; verificationMethod?: string | null; verificationSource?: string | null }` — from `EncryptedAssetImage` type

**Component interface:**

```typescript
'use client';

import React from 'react';
import { isSafeUrl, resolveUrl } from '@/lib/utils';

export interface ImageListProps {
  /** Section label, e.g. "Images (3)" */
  label: string;
  /** Array of image objects — must have at least a `url` field (string | null) */
  images: ReadonlyArray<{ url?: string | null; [key: string]: unknown }>;
}
```

**Rendering pattern** — extract the duplicated pattern from existing cards (DigitalAssetCard images section is the canonical pattern):

```tsx
export function ImageList({ label, images }: ImageListProps): React.ReactNode {
  if (images.length === 0) return null;

  return (
    <div>
      <h5 className="text-xs font-medium text-muted-foreground mb-1">{label}</h5>
      <div className="space-y-1.5">
        {images.map((image, i) => (
          <div key={i} className="flex items-center gap-2">
            {image.url && isSafeUrl(image.url) ? (
              // eslint-disable-next-line @next/next/no-img-element
              <img
                src={resolveUrl(image.url)}
                alt=""
                className="size-8 rounded object-cover shrink-0"
                onError={(e) => {
                  e.currentTarget.style.display = 'none';
                }}
              />
            ) : (
              <div className="size-8 rounded bg-muted shrink-0" />
            )}
            <span className="font-mono text-xs text-muted-foreground truncate">
              {image.url ?? '(no url)'}
            </span>
          </div>
        ))}
      </div>
    </div>
  );
}
```

**IMPORTANT:** Check `@/lib/utils` for `isSafeUrl` and `resolveUrl` helpers. If they don't exist, check where they're defined in the existing cards (they may be inline in each card file) and extract to utils.

```typescript
function buildFilter(values: Record<string, string>): EncryptedAssetFilter | undefined {
  const filter: EncryptedAssetFilter = {};
  if (values.address) filter.address = values.address;
  if (values.universalProfileName) filter.universalProfileName = values.universalProfileName;
  if (values.contentId) filter.contentId = values.contentId;
  if (values.revision) filter.revision = Number(values.revision); // Parse string → number
  if (values.encryptionMethod) filter.encryptionMethod = values.encryptionMethod;
  if (values.fileType) filter.fileType = values.fileType;
  if (values.fileSize) filter.fileSize = Number(values.fileSize); // Parse string → number
  if (values.timestamp) filter.timestamp = values.timestamp;
  return Object.keys(filter).length > 0 ? filter : undefined;
}
```

**Encryption include building** — special handling required:

When building the `include` param for the hook, encryption needs to be:

- If no encryption toggle is on → omit from include
- If encryption toggle is on but no accessControlConditions → `encryption: true` (simplest form)
- If encryption toggle is on AND accessControlConditions → still `encryption: true` (since `true` includes everything)
- If encryption toggle is on but accessControlConditions explicitly off → `encryption: { accessControlConditions: false }`

Actually, for the playground, the simplest approach:

- Top-level "Encryption" toggle in the scalar include section
- When encryption is toggled ON, show a sub-section for "Access Control Conditions" toggle
- Build include value: if encryption on → `{ accessControlConditions: accToggle }` or `true` if accToggle is also on

Use `useSubInclude` for the encryption sub-include, similar to how profile sub-includes work. The buildNestedInclude helper builds the include object. For encryption, create a custom builder:

```typescript
// In useListState:
const encryptionSubInclude = useSubInclude(ENCRYPTED_ASSET_ENCRYPTION_INCLUDE_FIELDS);

const include = buildNestedInclude(includeValues, {

if (include && includeValues.encryption) {

  const hasAccToggle = encryptionSubInclude.values.accessControlConditions;
  include.encryption = hasAccToggle ? true : { accessControlConditions: false };
}
```

Or, if this is too complex, just use `encryption: true` when the encryption toggle is on (fetch everything including access control conditions). The sub-include can be a future refinement. **Claude's discretion** — choose whichever approach is cleaner.
