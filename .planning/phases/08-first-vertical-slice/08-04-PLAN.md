**Step 2: Create `apps/test/src/app/profiles/page.tsx`**

This is a `'use client'` component (hooks require client-side rendering). It should be a full-featured playground that exercises ALL three hooks.

**Page structure (using shadcn Tabs for the three views):**

```
Profiles Playground (h1)
├── Tabs
│   ├── Tab: Single Profile
│   │   ├── Address Input (shadcn Input) + Preset Buttons (shadcn Button)
│   │   ├── Loading: shadcn Skeleton (card-shaped)
│   │   ├── Error: shadcn Alert (destructive variant)
│   │   ├── Success: shadcn Card with profile data rendered visually
│   │   │   ├── Name, description, address
│   │   │   ├── Tags as Badge components
│   │   │   ├── Links as clickable items
│   │   │   ├── Follower/following counts
│   │   │   └── Image URLs displayed (not actual images — the URLs may be IPFS)
│   │   └── Collapsible: Raw JSON (JSON.stringify with formatting)
│   │
│   ├── Tab: Profile List
│   │   ├── Filter controls: name search (Input), sort (Select), limit (Input)
│   │   ├── Loading: Skeleton rows
│   │   ├── Error: Alert
│   │   ├── Success: Grid/list of profile Cards (compact — address, name, counts)
│   │   └── Total count display
│   │
│   └── Tab: Infinite Scroll
│       ├── Same filter controls as list
│       ├── Loading: Skeleton
│       ├── Profiles rendered as cards
│       ├── "Load More" button (calls fetchNextPage)
│       ├── hasNextPage / isFetchingNextPage state display
│       └── Total loaded count
```

**Preset address buttons:**
Use 3 well-known LUKSO Universal Profile addresses. These are for developer convenience — the actual addresses should be verified against the live endpoint. Use:

- `0x79eb93CC915D23E52aD18aCCF7897ACbd1031D42` — "LUKSO"
- `0x98b83eB9BDEcB8dB47BEA4579eb7e1C92C127E1e` — "Universal Page"
- One more known address from the live endpoint (or use a generic label like "Sample UP")

If these addresses don't return data, the developer can type any address — the preset buttons are just convenience shortcuts.

**Hook usage in each tab:**

Tab 1 — Single Profile:

```typescript
const [address, setAddress] = useState('');
const { profile, isLoading, error } = useProfile({ address });
```

`enabled` is handled inside the hook (checks `Boolean(params.address)`).

Tab 2 — Profile List:

```typescript
const { profiles, totalCount, isLoading, error } = useProfiles({
  filter: nameFilter ? { name: nameFilter } : undefined,
  sort: selectedSort,
  limit: 10,
});
```

Tab 3 — Infinite Scroll:

```typescript
const { profiles, hasNextPage, fetchNextPage, isFetchingNextPage, isLoading, error } =
  useInfiniteProfiles({
    filter: nameFilter ? { name: nameFilter } : undefined,
    sort: selectedSort,
    pageSize: 10,
  });
```

**Collapsible raw JSON (on single profile tab):**

```tsx
<Collapsible>
  <CollapsibleTrigger asChild>
    <Button variant="ghost" size="sm">
      Toggle Raw JSON
    </Button>
  </CollapsibleTrigger>
  <CollapsibleContent>

    </pre>
  </CollapsibleContent>

```

**Loading states:** Use shadcn Skeleton that mimics the card layout shape:

```tsx
<Card>
  <CardHeader>
    <Skeleton className="h-6 w-48" />
    <Skeleton className="h-4 w-96" />
  </CardHeader>
  <CardContent className="space-y-3">
    <Skeleton className="h-4 w-full" />
    <Skeleton className="h-4 w-3/4" />
  </CardContent>
</Card>
```

**Error states:** Use shadcn Alert with destructive variant:

```tsx
<Alert variant="destructive">
  <AlertTitle>Error</AlertTitle>
  <AlertDescription>{error.message}</AlertDescription>
</Alert>
```

**Sort select:** Use shadcn Select with options for name, followerCount, followingCount + asc/desc.
