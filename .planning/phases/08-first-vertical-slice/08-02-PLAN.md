**Part A: Create `packages/react/src/parsers/profiles.ts`**

The parser transforms raw Hasura response shape to the clean `Profile` type. It MUST handle:

1. **Nested LSP3 traversal:** `universal_profile.lsp3_profile.name.value` → `profile.name`
2. **Optional chaining for @include fields:** When `@include(if: false)` omits a field, the field won't be in the response. Use `?.` and default to `[]` for arrays, `null` for scalars.
3. **Aggregate count flattening:** `followed_by_aggregate.aggregate.count` → `followerCount`, `followed_aggregate.aggregate.count` → `followingCount`
4. **Image parsing:** Map Hasura image types to `ProfileImage`. Avatar (`lsp3_profile_asset`) has no width/height — set to `null`. Profile image and background image (`lsp3_profile_image`) have width/height.
5. **Verification object:** `{ verification_method, verification_data }` → `{ method, data } | null`

Define a local `HasuraUniversalProfile` interface that matches the shape returned by the GraphQL documents from Plan 01. Alternatively, use the codegen-generated types if they're usable (check `GetProfileQuery` type in graphql.ts). Using codegen types is preferred for type safety, but if they're deeply nested or awkward, a local interface is fine.

Export:

- `parseProfile(raw: HasuraRawType): Profile` — single profile
- `parseProfiles(raw: HasuraRawType[]): Profile[]` — just maps parseProfile

**Part B: Create `packages/react/src/services/profiles.ts`**

The service layer is the Hasura translation boundary. It:

1. Takes clean flat params (ProfileFilter, ProfileSort, ProfileInclude)
2. Builds Hasura `where` clause (universal_profile_bool_exp)
3. Builds Hasura `order_by` array
4. Builds `include` → boolean variable mapping
5. Calls `execute()` with the document and variables
6. Pipes result through parser
7. Returns clean typed result

**Builder functions (internal, not exported):**

`buildProfileWhere(filter?: ProfileFilter)` — translates flat filter to Hasura where:

- `filter.name` → `{ lsp3_profile: { name: { value: { _ilike: '%${name}%' } } } }`
- `filter.followedBy` → `{ followed_by: { follower_address: { _eq: address } } }`
  (CRITICAL: "profiles that X follows" = profiles appear in followed_by where X is follower)
- `filter.following` → `{ followed: { followed_address: { _eq: address } } }`
  (CRITICAL: "profiles that follow X" = profiles appear in followed where X is followed)
- `filter.tokenOwned` → `{ owned_assets: { address: { _eq: tokenAddress }, ... } }`
  Include `balance: { _gt: "0" }` if no minBalance specified. Include `token_id` if specified.
- Multiple filters combine with `_and`

`buildProfileOrderBy(sort?: ProfileSort)` — translates flat sort to Hasura order_by:

- `'name'` → `[{ lsp3_profile: { name: { value: sort.direction } } }]`
- `'followerCount'` → `[{ followed_by_aggregate: { count: sort.direction } }]`
- `'followingCount'` → `[{ followed_aggregate: { count: sort.direction } }]`
- If no sort specified, return undefined (Hasura default ordering)

`buildIncludeVars(include?: ProfileInclude)` — translates include to GraphQL boolean variables:

- If `include` is undefined → return `{}` (GraphQL defaults all to true)
- If `include` is provided → return `{ includeTags: include.tags ?? false, includeLinks: include.links ?? false, ... }`
- When include IS provided, omitted fields default to `false` (explicit opt-in)

**Service functions (exported):**

`fetchProfile(url: string, params: { address: string; include?: ProfileInclude }): Promise<Profile | null>`

- Builds where: `{ address: { _eq: params.address } }`
- Builds include vars
- Calls execute with GetProfileDocument
- Returns first result parsed, or null if no results

`fetchProfiles(url: string, params: { filter?: ProfileFilter; sort?: ProfileSort; limit?: number; offset?: number; include?: ProfileInclude }): Promise<{ profiles: Profile[]; totalCount: number }>`

- Builds where from filter, order_by from sort, include vars
- Calls execute with GetProfilesDocument
- Parses all results
- Returns parsed profiles + totalCount from aggregate

**Type safety for Hasura types:** Import the generated Hasura types from `../graphql/graphql` for the where/order_by builders. This ensures type safety when constructing Hasura queries. If the codegen output doesn't have these types (local fallback schema), use type assertions with a TODO comment noting they'll be fully typed when codegen runs against Hasura.

**Important:** The builder functions should NOT be exported — they are internal implementation details. Only `fetchProfile` and `fetchProfiles` are public.
