**Step 1: Update `schema.graphql` for local codegen fallback.**

The current `schema.graphql` is minimal (just scalars + placeholder). For documents with `@include` directives, codegen in local-fallback mode needs the `universal_profile_bool_exp`, `universal_profile_order_by`, and related input types to exist.

Extend `schema.graphql` to include enough type stubs so codegen can process the documents WITHOUT Hasura introspection. At minimum, add:

- `universal_profile` type with: `id`, `address`, `lsp3_profile` (object), aggregate relations
- `lsp3_profile` type with nested `name`, `description`, `tags`, `links`, `avatar`, `profile_image`, `background_image` relations
- Sub-types for each nested relation (name/description with `value`, tag with `value`, link with `title`/`url`, image types with appropriate fields)
- Aggregate types: `universal_profile_aggregate` with `aggregate { count }`
- Input types: `universal_profile_bool_exp`, `universal_profile_order_by` (can be simplified stubs ‚Äî the full types come from Hasura introspection)
- Enum: `order_by` with `asc`, `desc` values

The goal is that `pnpm codegen` succeeds WITHOUT `HASURA_GRAPHQL_ENDPOINT` set ‚Äî the local schema is the CI/build fallback.

**Step 2: Create `documents/profiles.ts` with TWO GraphQL documents.**

Use the `graphql()` tagged template from `../graphql` (same pattern as `_placeholder.ts`).

**Document 1: `GetProfileDocument`** ‚Äî single profile by address.

Variables:

- `$where: universal_profile_bool_exp!` ‚Äî the service layer builds this
- `$includeTags: Boolean! = true`
- `$includeLinks: Boolean! = true`
- `$includeAvatar: Boolean! = true`
- `$includeProfileImage: Boolean! = true`
- `$includeBackgroundImage: Boolean! = true`

Query body: `universal_profile(where: $where, limit: 1)` with:

- `id`, `address`
- `lsp3_profile { name { value }, description { value } }`
- `lsp3_profile { tags @include(if: $includeTags) { value } }`
- `lsp3_profile { links @include(if: $includeLinks) { title, url } }`
- `lsp3_profile { avatar @include(if: $includeAvatar) { url, file_type, verification_method, verification_data } }`
- `lsp3_profile { profile_image @include(if: $includeProfileImage) { url, width, height, verification_method, verification_data } }`
- `lsp3_profile { background_image @include(if: $includeBackgroundImage) { url, width, height, verification_method, verification_data } }`
- `followed_by_aggregate { aggregate { count } }`
- `followed_aggregate { aggregate { count } }`

**Document 2: `GetProfilesDocument`** ‚Äî list with filtering, sorting, pagination, AND total count.

Variables: Same include variables as above, plus:

- `$where: universal_profile_bool_exp`
- `$order_by: [universal_profile_order_by!]`
- `$limit: Int`
- `$offset: Int`

Query body: `universal_profile(where: $where, order_by: $order_by, limit: $limit, offset: $offset)` with same nested fields as GetProfile, PLUS:

- `universal_profile_aggregate(where: $where) { aggregate { count } }` for totalCount

NOTE: Both `useProfiles` and `useInfiniteProfiles` use `GetProfilesDocument` ‚Äî the difference is how the hook manages pagination, not the document.
