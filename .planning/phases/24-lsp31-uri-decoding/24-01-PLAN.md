# Plan 24-01: Switch LSP29 handler from VerifiableURI to LSP31 decoding

## Objective

Replace `decodeVerifiableUri()` (LSP2) with `parseLsp31Uri()` + `resolveUrl()` from `@chillwhales/lsp31` in the LSP29 encrypted asset data key handler. LSP29 stores its metadata URL using LSP31 encoding (multi-backend entries with content hash verification), NOT LSP2 VerifiableURI encoding. LSP3/LSP4 continue using VerifiableURI — only LSP29 changes.

## Context

- `@chillwhales/lsp31` is already installed (Phase 23)
- LSP31 binary format: `[reserved prefix][verification method][hash length][hash][JSON entries array]`
- Each entry has a `backend` (ipfs/s3/lumera/arweave) and backend-specific fields (cid, key, txId, etc.)
- `parseLsp31Uri(hex)` → `{ verificationMethod, verificationData, entries }`
- `selectBackend(entries, preference?)` → sorted entries by preference
- `resolveUrl(entry)` → fetchable URL string (e.g. `ipfs://QmXxx`)
- `isLsp31Uri(hex)` → boolean check for LSP31 prefix
- The handler should try LSP31 first; fall back to VerifiableURI for backward compatibility with any pre-v2 data

## Tasks

### Task 1: Update LSP29 data key handler to use LSP31 decoding
type="auto"

**Changes:**
1. In `packages/indexer/src/handlers/lsp29EncryptedAsset.handler.ts`:
   - Import `{ isLsp31Uri, parseLsp31Uri, selectBackend, resolveUrl }` from `@chillwhales/lsp31`
   - In `extractFromIndex()`, replace the `decodeVerifiableUri(dataValue)` call with:
     - Check `isLsp31Uri(dataValue)` first
     - If LSP31: `parseLsp31Uri(dataValue)` → `selectBackend(entries, ['ipfs'])` → `resolveUrl(selectedEntries[0])`
     - If not LSP31: fall back to existing `decodeVerifiableUri(dataValue)` for backward compatibility
   - Store the resolved URL in `entity.url` (same as before)
   - Store the raw LSP31 entries JSON in a new approach or just keep raw hex in `rawValue`

2. Remove the `decodeVerifiableUri` import if no longer used (but it IS still used — LSP3/LSP4 handlers use it via other handler files, and backward compat path needs it)

**Done criteria:**
- `pnpm --filter=@chillwhales/indexer build` passes
- LSP31-encoded data values decode to correct `ipfs://` URLs
- Non-LSP31 data values still fall back to VerifiableURI decoding
