## Decisions Made

- Parser calls sub-parsers (parseProfile, parseDigitalAsset) without sub-include forwarding — matching the data-changed-events parser pattern. The stripExcluded call at the parent level handles field removal. This avoids TypeScript overload resolution issues with optional include parameters.
- 3 prefix replacement loops in buildUniversalReceiverEventIncludeVars: receiving UP (`includeProfile*` → `includeUniversalProfile*`), sender UP (`includeProfile*` → `includeFromProfile*`), sender DA (`include*` → `includeFromAsset*`) — this is the most relation-heavy include vars builder in the project.
