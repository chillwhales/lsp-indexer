**Core domain schemas:**

```typescript
// Token type clean enum (maps to LSP4_TOKEN_TYPES in service layer)
export const TokenTypeSchema = z.enum(['TOKEN', 'NFT', 'COLLECTION']);

// Standard (derived by parser from decimals presence)
export const StandardSchema = z.enum(['LSP7', 'LSP8']);

// LSP4 metadata sub-schemas
export const DigitalAssetImageSchema = z.object({
  url: z.string(),
  width: z.number().nullable(),
  height: z.number().nullable(),
  verification: z
    .object({
      method: z.string(),
      data: z.string(),
    })
    .nullable(),
});

export const DigitalAssetLinkSchema = z.object({
  title: z.string(),
  url: z.string(),
});

export const DigitalAssetAttributeSchema = z.object({
  key: z.string(),
  value: z.string(),
  type: z.string(),
});

export const DigitalAssetOwnerSchema = z.object({
  address: z.string(),
  timestamp: z.string(),
});

// Main DigitalAsset schema
export const DigitalAssetSchema = z.object({
  /** Contract address — always present */
  address: z.string(),
  /** Derived standard: LSP7 (has decimals) or LSP8 (no decimals) */
  standard: StandardSchema,
  /** Token name from LSP4 metadata */
  name: z.string().nullable(),
  /** Token symbol from LSP4 metadata */
  symbol: z.string().nullable(),
  /** Token type: TOKEN (0), NFT (1), or COLLECTION (2) */
  tokenType: TokenTypeSchema.nullable(),
  /** Number of decimal places (LSP7 only, null for LSP8) */
  decimals: z.number().nullable(),
  /** Total token supply */
  totalSupply: z.string().nullable(),
  /** Description from LSP4 metadata */
  description: z.string().nullable(),
  /** Category from LSP4 metadata */
  category: z.string().nullable(),
  /** LSP4 metadata icon images */
  icons: z.array(DigitalAssetImageSchema),
  /** LSP4 metadata images */
  images: z.array(DigitalAssetImageSchema),
  /** LSP4 metadata links */
  links: z.array(DigitalAssetLinkSchema),
  /** LSP4 metadata attributes */
  attributes: z.array(DigitalAssetAttributeSchema),
  /** Contract owner/controller (NOT token holder) */
  owner: DigitalAssetOwnerSchema.nullable(),
  /** Number of unique token holders */
  holderCount: z.number(),
  /** Number of creator entries */
  creatorCount: z.number(),
  // LSP8-specific fields (null for LSP7)
  /** LSP8 reference contract address (null for LSP7) */
  referenceContract: z.string().nullable(),
  /** LSP8 token ID format (null for LSP7) */
  tokenIdFormat: z.string().nullable(),
  /** LSP8 token metadata base URI (null for LSP7) */
  baseUri: z.string().nullable(),
});
```

**IMPORTANT: For lsp4Metadata, use a SINGLE selection with all sub-fields having their own @include directives:**

```graphql
lsp4Metadata {
  description @include(if: $includeDescription) { value }
  category @include(if: $includeCategory) { value }
  icon @include(if: $includeIcons) { url, width, height, verification_method, verification_data }
  images @include(if: $includeImages) { url, width, height, image_index, verification_method, verification_data }
  links @include(if: $includeLinks) { title, url }
  attributes @include(if: $includeAttributes) { key, value, type }
}
```

Wait — `lsp4Metadata` is a relation (object) so it always needs to be present for sub-field @include to work. GraphQL requires at least one field in a selection set. Solution: always include `lsp4Metadata` but put @include on each sub-field. The `lsp4Metadata` block can be included when ANY of its sub-fields are requested. Since we default everything to `true`, this is fine — the block is always present by default.

Actually, the cleanest approach: always fetch `lsp4Metadata` (it's always present on digital assets) and put @include on each nested field within it. This matches the profile pattern where `lsp3Profile` is always fetched but sub-fields have @include.

- `owner @include(if: $includeOwner) { address, timestamp }`
- `ownedAssets_aggregate @include(if: $includeHolderCount) { aggregate { count } }`
- `lsp4CreatorsLength @include(if: $includeCreatorCount) { value }`
- `lsp8ReferenceContract @include(if: $includeReferenceContract) { value }`
- `lsp8TokenIdFormat @include(if: $includeTokenIdFormat) { value }`
- `lsp8TokenMetadataBaseUri @include(if: $includeBaseUri) { value }`
