**Part A: Create `packages/node/src/parsers/digital-assets.ts`**

The parser transforms raw Hasura response shape to the clean `DigitalAsset` type. Follow the same pattern as `packages/node/src/parsers/profiles.ts`.

**CRITICAL: Two derivations that make this parser special:**

1. **`standard` derivation (LOCKED DECISION):** Derive `"LSP7" | "LSP8"` from the presence of the `decimals` field. If `raw.decimals` exists and has a value → `"LSP7"`. If `raw.decimals` is null/undefined → `"LSP8"`. This is because only LSP7 contracts have a `decimals` value.

2. **`tokenType` mapping:** Raw Hasura stores `"0"`, `"1"`, `"2"` as strings. Map to clean enum:
   - `"0"` → `"TOKEN"`
   - `"1"` → `"NFT"`
   - `"2"` → `"COLLECTION"`

**Type extraction from codegen:**

```typescript
import type { DigitalAsset, DigitalAssetImage, Standard, TokenType } from '@lsp-indexer/types';
import type { GetDigitalAssetQuery } from '../graphql/graphql';

type RawDigitalAsset = GetDigitalAssetQuery['digital_asset'][number];
```

**Image parsing:** LSP4 metadata icons and images follow a similar pattern. Use a structural interface for images (same approach as profile's `RawImageWithDimensions`):

```typescript
interface RawImageLike {
  url?: string | null;
  width?: number | null;
  height?: number | null;
  verification_method?: string | null;
  verification_data?: string | null;
}

function parseImage(raw: RawImageLike): DigitalAssetImage {
  return {
    url: raw.url ?? '',
    width: raw.width ?? null,
    height: raw.height ?? null,
    verification:
      raw.verification_method != null
        ? { method: raw.verification_method, data: raw.verification_data ?? '' }
        : null,
  };
}
```

**Token type mapping function:**

```typescript
function mapTokenType(raw: string | null | undefined): TokenType | null {
  switch (raw) {
    case '0':
      return 'TOKEN';
    case '1':
      return 'NFT';
    case '2':
      return 'COLLECTION';
    default:
      return null;
  }
}
```

**Main parseDigitalAsset function:**

```typescript
export function parseDigitalAsset(raw: RawDigitalAsset): DigitalAsset {
  const metadata = raw.lsp4Metadata;
  const standard: Standard = raw.decimals != null ? 'LSP7' : 'LSP8';

  return {
    address: raw.address,
    standard,
    name: raw.lsp4TokenName?.value ?? null,
    symbol: raw.lsp4TokenSymbol?.value ?? null,
    tokenType: mapTokenType(raw.lsp4TokenType?.value),
    decimals: raw.decimals?.value ?? null,
    totalSupply: raw.totalSupply?.value ?? null,
    description: metadata?.description?.value ?? null,
    category: metadata?.category?.value ?? null,
    icons: metadata?.icon?.map(parseImage) ?? [],
    images: metadata?.images?.map(parseImage) ?? [],
    links: metadata?.links?.map((l) => ({ title: l.title ?? '', url: l.url ?? '' })) ?? [],
    attributes:
      metadata?.attributes?.map((a) => ({
        key: a.key ?? '',
        value: a.value ?? '',
        type: a.type ?? '',
      })) ?? [],
    owner: raw.owner ? { address: raw.owner.address, timestamp: raw.owner.timestamp ?? '' } : null,
    holderCount: raw.ownedAssets_aggregate?.aggregate?.count ?? 0,
    creatorCount: raw.lsp4CreatorsLength?.value != null ? Number(raw.lsp4CreatorsLength.value) : 0,
    referenceContract: raw.lsp8ReferenceContract?.value ?? null,
    tokenIdFormat: raw.lsp8TokenIdFormat?.value ?? null,
    baseUri: raw.lsp8TokenMetadataBaseUri?.value ?? null,
  };
}

export function parseDigitalAssets(raw: RawDigitalAsset[]): DigitalAsset[] {
  return raw.map(parseDigitalAsset);
}
```

**Builder functions (internal, NOT exported):**

`buildDigitalAssetWhere(filter?: DigitalAssetFilter)`:

- `filter.name` → `{ lsp4TokenName: { value: { _ilike: '%name%' } } }`
- `filter.symbol` → `{ lsp4TokenSymbol: { value: { _ilike: '%symbol%' } } }`
- `filter.tokenType` → Map clean enum to raw value using LSP4_TOKEN_TYPES, then `{ lsp4TokenType: { value: { _eq: rawValue.toString() } } }`. Mapping: `'TOKEN'` → `LSP4_TOKEN_TYPES.TOKEN.toString()` = `"0"`, `'NFT'` → `"1"`, `'COLLECTION'` → `"2"`.
- `filter.category` → `{ lsp4Metadata: { category: { value: { _ilike: '%category%' } } } }`
- `filter.holderAddress` → `{ ownedAssets: { owner: { _ilike: filter.holderAddress } } }` (the `owned_asset` type uses `owner` for the holder address field)
- `filter.ownerAddress` → `{ owner: { address: { _ilike: filter.ownerAddress } } }`
- Multiple filters combine with `_and`

**IMPORTANT for tokenType filter:** Import `LSP4_TOKEN_TYPES` from `@lukso/lsp4-contracts`:

```typescript
import { LSP4_TOKEN_TYPES } from '@lukso/lsp4-contracts';

}
```
