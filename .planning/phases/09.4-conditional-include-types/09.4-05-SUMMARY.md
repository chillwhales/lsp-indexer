## Decisions Made

- **Card prop types as `Record<string, unknown>`:** Cards accept any subset of the full domain type. This is cleaner than keeping the full domain type as prop because it directly reflects the runtime reality — the card renders whatever fields exist, using `'key' in obj` guards. The typed narrowing is enforced at the hook consumer level, not the card level.
- **`as Record<string, unknown>` casts at page boundaries:** Pages cast the narrowed `XResult<I>` types when passing to cards. This is the only place where casts are needed — a clean boundary between typed hook results and field-presence-based rendering.
- **Removed explicit `ResultsList<T>` generics:** Removed `ResultsList<Profile>`, `ResultsList<DigitalAsset>`, etc. in favor of letting TypeScript infer the generic from the `items` prop. This avoids type mismatch errors when hooks return narrowed result types.
- **No changes to include-toggles.tsx:** The existing toggle component already correctly produces the include shape needed by hooks. Toggling off a field results in the field being absent from the runtime response (via `stripExcluded`), and the card's `'key' in obj` check correctly hides the UI section.
