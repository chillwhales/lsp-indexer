## Decisions Made

- **NFT base fields:** `address | tokenId | isBurned | isMinted` — identity fields and boolean flags always present regardless of include
- **OwnedAsset base fields:** `id | digitalAssetAddress | holderAddress` — identity fields always present
- **Nested relation type pattern:** `ResolveX<I> = I extends { field: infer C } ? C extends SubInclude ? { field: SubResult<C> | null } : {} : {}` — uses conditional extends with `infer` to extract sub-include type and pass to sub-result
- **NftHolder timestamp:** Handled via `ProfileResult<H> & { timestamp: string }` intersection because NftHolder = Profile + timestamp
- **OwnedAsset holder is plain Profile:** No timestamp merge like NftHolder — just `ProfileResult<H>`
- **Recursive stripping:** Parser calls `parseDigitalAsset(raw, include?.collection)` and `parseProfile(raw, include?.holder)` with sub-include, then `stripExcluded` on top-level — nested relations handled by their own parsers
