  <action>
Update all 5 card components to check field presence using `'key' in obj` instead of null checks. This ensures cards only show UI sections for fields that were actually included in the query.

**Pattern change across all cards:**

Before (null check):

```tsx
{
  profile.tags != null && profile.tags.length > 0 && <div>...</div>;
}
```

After (field presence check):

```tsx
{
  'tags' in profile && Array.isArray(profile.tags) && profile.tags.length > 0 && <div>...</div>;
}
```

**For each card component:**

1. **Change the prop type** from the full domain type to `Record<string, unknown>` so TypeScript doesn't complain about accessing possibly-absent fields. Alternatively, keep the domain type but use a runtime `'key' in obj` guard.

   RECOMMENDED approach: Use a generic prop type that accepts any subset of the domain:

   ```tsx
   export interface ProfileCardProps {
     profile: Record<string, unknown>;
     isFetching?: boolean;
   }
   ```

   This is cleaner because the card renders whatever fields exist. The typed narrowing is enforced at the hook consumer level, not the card level.

2. **Replace all null/undefined checks with `'field' in obj` checks:**

   - `profile.name` → `'name' in profile ? (profile.name as string | null) : null`
   - `profile.tags != null` → `'tags' in profile && Array.isArray(profile.tags)`
   - `profile.followerCount !== null` → `'followerCount' in profile`
   - Same pattern for all fields across all 5 cards

3. **Special cases:**

   - `address` is always present — no check needed, direct access
   - NFT card: `tokenId`, `isBurned`, `isMinted` always present — no check needed
   - OwnedAsset card: `id`, `digitalAssetAddress`, `holderAddress` always present
   - OwnedToken card: `id`, `digitalAssetAddress`, `holderAddress`, `tokenId` always present
   - Nested relations (`collection`, `holder`, `digitalAsset`, `nft`, `ownedAsset`): check `'relation' in obj && obj.relation` before rendering nested content

4. **RawJsonToggle** already receives the object directly via `data={profile}` — after stripping, it will naturally show only included fields. No changes needed to `RawJsonToggle` itself.

**Per-card specifics:**

**ProfileCard:**

- `profile.name` in title: use `('name' in profile ? (profile.name as string | null) : null) ?? 'Unnamed Profile'`
- `profile.followerCount` / `profile.followingCount`: guard with `'followerCount' in profile`
- `profile.description`: guard with `'description' in profile && profile.description`
- `profile.tags`: guard with `'tags' in profile && Array.isArray(profile.tags) && profile.tags.length > 0`
- `profile.links`: guard with `'links' in profile && Array.isArray(profile.links) && profile.links.length > 0`
- `profile.avatar`, `profile.profileImage`, `profile.backgroundImage`: same array pattern

**DigitalAssetCard:**

- Guard every field section with `'fieldName' in asset`
- `standard` field: guard with `'standard' in asset`
- `owner` object: guard with `'owner' in asset && asset.owner`

**NftCard:**

- Base fields (`address`, `tokenId`, `isBurned`, `isMinted`): always accessible
- `name`, `formattedTokenId`: guard with `'name' in nft`
- `collection`, `holder`: guard with `'collection' in nft && nft.collection`
- All metadata fields: same pattern

**OwnedAssetCard:**

- Base fields (`id`, `digitalAssetAddress`, `holderAddress`): always accessible
- `balance`, `block`, `timestamp`, `tokenIdCount`: guard with `'fieldName' in asset`
- `digitalAsset`, `holder`: guard with `'relation' in asset && asset.relation`

**OwnedTokenCard:**

- Base fields (`id`, `digitalAssetAddress`, `holderAddress`, `tokenId`): always accessible
- `block`, `timestamp`: guard with `'fieldName' in token`
- All 4 nested relations: guard with `'relation' in token && token.relation`
