## Accomplishments

- Created `IncludeResult<Full, Base, Map, I>` utility type in `include-types.ts` — core type algebra that narrows a domain type based on include parameter (undefined → full, {} → base only, { name: true } → base + name)
- Created `stripExcluded` runtime utility in `parsers/strip.ts` — ensures excluded fields are absent from returned objects at runtime (Object.keys only shows included fields)
- Added `ProfileResult<I>` conditional type alias — maps `ProfileIncludeFieldMap` to `IncludeResult` for the Profile domain
- Threaded generic `I` parameter through all 8 functions: `fetchProfile`, `fetchProfiles`, `getProfile`, `getProfiles`, `useProfile`, `useProfiles`, `useInfiniteProfiles` (×2 for react + next)
- Full backward compatibility: omitting `include` returns the full `Profile` type

## Decisions Made

- **`const I` generic parameter** — Ensures TypeScript preserves `{ name: true }` as a literal type rather than widening to `{ name: boolean }`. Critical for the `IncludeResult` type to work correctly.
- **`as ProfileResult<I>` casts at service boundaries** — Necessary because the parser returns `Profile` (full type) which is then stripped at runtime. TypeScript can't infer that `stripExcluded` narrows the type.
- **`derivedFields` parameter on `stripExcluded`** — Forward-looking design for digital assets where `standard` is derived from `decimals` (Plan 02). Not used for profiles but part of the API.
- **Intersection type `& { include?: I }` for hook params** — Allows the generic `I` to override the `ProfileInclude` in the base param type, preserving literal inference through the hook.
