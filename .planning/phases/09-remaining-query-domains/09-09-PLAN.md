    **packages/types/src/data-changed.ts:**
    ERC725 data change events — emitted when a key-value pair is set on an ERC725Y contract.
    ```ts
    DataChangedEventSchema = z.object({
      id: z.string(),                   // unique event identifier
      contractAddress: z.string(),       // ERC725 contract address
      dataKey: z.string(),              // the ERC725 data key (hex)
      dataValue: z.string().nullable(), // new value (hex, may be empty)
      blockNumber: z.number(),          // block where event was emitted
      transactionHash: z.string(),      // transaction hash
      timestamp: z.string().nullable(), // ISO timestamp
    })
    ```
    Adapt based on actual `data_changed` table schema in graphql.ts.
    Filter schema `DataChangedFilterSchema`: contractAddress (string), dataKey (string), blockNumberMin (number), blockNumberMax (number)
    Sort: `DataChangedSortFieldSchema`: z.enum(['blockNumber', 'contractAddress', 'dataKey'])
    Hook param schemas: UseDataChangedEventsParams { filter?, sort?, limit?, offset? }, UseInfiniteDataChangedEventsParams { filter?, sort?, pageSize? }
    Export all schemas + inferred types.

    **packages/node/src/documents/data-changed.ts:**
    `GetDataChangedEventsDocument`: query `data_changed(where: $where, order_by: $order_by, limit: $limit, offset: $offset)` + `data_changed_aggregate(where: $where)` for count.
    Fields: id, contract_address, data_key, data_value, block_number, transaction_hash, timestamp (or block_timestamp — check schema).

    **packages/node/src/parsers/data-changed.ts:**
    `parseDataChangedEvent(raw)` → DataChangedEvent: map snake_case to camelCase.
    `parseDataChangedEvents(raw[])` → DataChangedEvent[]

    **packages/node/src/services/data-changed.ts:**
    `buildDataChangedWhere(filter?)` → Data_Changed_Bool_Exp:
    - filter.contractAddress → `{ contract_address: { _ilike: contractAddress } }`
    - filter.dataKey → `{ data_key: { _ilike: dataKey } }`
    - filter.blockNumberMin → `{ block_number: { _gte: blockNumberMin } }`
    - filter.blockNumberMax → `{ block_number: { _lte: blockNumberMax } }`
    `buildDataChangedOrderBy(sort?)` → Data_Changed_Order_By[] | undefined

    **IMPORTANT:** Default sort for event domains is `block_number DESC` (newest first) per research.
    If no sort is provided in params, the service should use `[{ block_number: 'desc' }]` as the default order_by.
    Example:
    ```ts
    const orderBy = buildDataChangedOrderBy(params.sort) ?? [{ block_number: 'desc' }];
    ```

    `fetchDataChangedEvents(url, params)` → FetchDataChangedEventsResult { events: DataChangedEvent[]; totalCount: number }

    **packages/node/src/keys/data-changed.ts:**
    `dataChangedKeys` factory:

    - `.list(filter, sort, limit, offset)` → [...all, 'list', { filter, sort, limit, offset }]
    - `.infinite(filter, sort)` → [...all, 'infinite', { filter, sort }]
