    **Research note:** The `follower` table represents CURRENT follow state (not the historical `follow` log table). A row exists when an active follow relationship exists.

    **packages/types/src/social.ts:**
    ```ts
    FollowerSchema = z.object({
      followerAddress: z.string(),  // address that IS following
      followedAddress: z.string(),  // address being followed
    })
    FollowCountSchema = z.object({
      followerCount: z.number(),
      followingCount: z.number(),
    })
    ```
    Filter schemas:
    - `FollowerFilterSchema`: no additional fields beyond the address used in the query itself (filtering by address is implicit in `useFollowers`/`useFollowing`)
    Sort: `FollowerSortFieldSchema`: z.enum(['followerAddress', 'followedAddress'])
    Hook param schemas:
    - `UseFollowersParams`: { address: string; sort?, limit?, offset? } — address is required (whose followers)
    - `UseFollowingParams`: { address: string; sort?, limit?, offset? } — address is required (who they follow)
    - `UseInfiniteFollowersParams`: { address: string; sort?, pageSize? }
    - `UseInfiniteFollowingParams`: { address: string; sort?, pageSize? }
    - `UseFollowCountParams`: { address: string }
    Export all schemas + inferred types.

    **packages/node/src/documents/social.ts:**
    `GetFollowersDocument`: query `follower(where: { followed_address: { _ilike: $address } }, order_by: $order_by, limit: $limit, offset: $offset)` + `follower_aggregate(where: { followed_address: { _ilike: $address } })` for count.
    Fields: follower_address, followed_address
    `GetFollowingDocument`: query `follower(where: { follower_address: { _ilike: $address } }, ...)` — the same table but filtered by who is following.
    `GetFollowCountDocument`: query `follower_aggregate` twice — once for followerCount, once for followingCount — using GraphQL aliases or two aggregate queries:
    ```graphql
    followerCount: follower_aggregate(where: { followed_address: { _ilike: $address } }) { aggregate { count } }
    followingCount: follower_aggregate(where: { follower_address: { _ilike: $address } }) { aggregate { count } }
    ```

    **packages/node/src/parsers/social.ts:**
    `parseFollower(raw)` → Follower: { followerAddress: raw.follower_address, followedAddress: raw.followed_address }
    `parseFollowers(raw[])` → Follower[]
    `parseFollowCount(raw)` → FollowCount: { followerCount: raw.followerCount?.aggregate?.count ?? 0, followingCount: raw.followingCount?.aggregate?.count ?? 0 }

    **packages/node/src/services/social.ts:**
    `fetchFollowers(url, { address, sort?, limit?, offset? })` → FetchFollowersResult { followers: Follower[]; totalCount: number }
    `fetchFollowing(url, { address, sort?, limit?, offset? })` → FetchFollowingResult { following: Follower[]; totalCount: number }
    `fetchFollowCount(url, { address })` → FollowCount

    **packages/node/src/keys/social.ts:**
    `followerKeys` factory:
    - `.all` → ['followers']
    - `.followers(address)` → [...all, 'followers', address]
    - `.following(address)` → [...all, 'following', address]
    - `.count(address)` → [...all, 'count', address]
    - `.infiniteFollowers(address, sort)` → [...all, 'infinite', 'followers', { address, sort }]
    - `.infiniteFollowing(address, sort)` → [...all, 'infinite', 'following', { address, sort }]
