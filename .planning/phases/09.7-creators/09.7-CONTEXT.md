# Phase 9.7: Creators - Context

**Gathered:** 2026-02-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Developer can use `useCreators`, `useInfiniteCreators` for LSP4 creator data — querying which addresses created which digital assets. Follows the established 4-plan vertical-slice pattern (types+docs → parsers+services+keys → hooks+actions+wiring → playground+verification). No singular `useCreator` hook (creator records are a join table with no natural key — developers query by filter).

</domain>

<decisions>
## Implementation Decisions

### Domain naming

- Domain name: **"Creator"** everywhere (not "CreatorAddress")
- Types: `Creator`, `CreatorInclude`, `CreatorResult<I>`, `PartialCreator`
- Hooks: `useCreators`, `useInfiniteCreators` (update ROADMAP.md QUERY-06 to match)
- Query keys: `creatorKeys`
- Files: `creators.ts` across all packages
- No singular `useCreator` hook — no natural key exists (opaque Hasura ID only)

### Nested relation includes

- `creatorProfile` → full sub-include support via `ProfileInclude` / `ProfileResult<I>`
  - `include: { creatorProfile: true }` fetches all profile fields
  - `include: { creatorProfile: { name: true, avatar: true } }` narrows to specified fields
  - Reuses `ProfileIncludeSchema`, `buildProfileIncludeVars`, `parseProfile`
- `digitalAsset` → full sub-include support via `DigitalAssetInclude` / `DigitalAssetResult<I>`
  - `include: { digitalAsset: true }` fetches all DA fields
  - `include: { digitalAsset: { name: true, symbol: true } }` narrows to specified fields
  - Reuses `DigitalAssetIncludeSchema`, `buildDigitalAssetIncludeVars`, `parseDigitalAsset`

### Base fields (always present)

- `creatorAddress` (from `creator_address`) — who created
- `digitalAssetAddress` (from `address`) — what was created
- These are the composite natural key of the relationship

### Includable scalar fields

- `arrayIndex` (from `array_index`: `numeric`) — position in LSP4 creators array
- `interfaceId` (from `interface_id`: `String`) — ERC165 interface ID
- `timestamp` (from `timestamp`: `timestamptz`) — when indexed

### Filter fields (7 total)

- `creatorAddress` → `creator_address._ilike` (case-insensitive)
- `digitalAssetAddress` → `address._ilike` (case-insensitive)
- `interfaceId` → `interface_id._ilike` (case-insensitive)
- `creatorName` → `creatorProfile.lsp3Profile.name._ilike` (nested, same as followers' `followerName`)
- `digitalAssetName` → `digitalAsset.lsp4TokenName.name._ilike` (nested, same as owned-assets)
- `timestampFrom` → `timestamp: { _gte }` (range lower bound)
- `timestampTo` → `timestamp: { _lte }` (range upper bound)

### Sort fields (6 total)

- `timestamp` → `timestamp` (default sort: descending)
- `creatorAddress` → `creator_address`
- `digitalAssetAddress` → `address`
- `arrayIndex` → `array_index`
- `creatorName` → `creatorProfile.lsp3Profile.name` (nested sort)
- `digitalAssetName` → `digitalAsset.lsp4TokenName.name` (nested sort)
- Default sort: `timestamp` descending
- All sorts support `SortNulls` via `orderDir()` helper

### Playground card layout

- **Top-level row:** All scalar fields — `creatorAddress`, `digitalAssetAddress`, `arrayIndex`, `interfaceId`, `timestamp` (render via `'key' in obj` guards)
- **Collapsible 1:** Creator Profile (when `creatorProfile` included)
- **Collapsible 2:** Digital Asset (when `digitalAsset` included)
- Collapsibles use ghost Button + ChevronDown pattern (matches FollowerCard)
- Plus standard RawJsonToggle

### Playground page

- Two tabs: **List** (`useCreators` with filters/sort/include toggles) + **Infinite** (`useInfiniteCreators`)
- Route: `apps/test/src/app/creators/page.tsx`
- Add to nav sidebar

### Claude's Discretion

- Exact Hasura field → camelCase mapping details beyond what's specified
- CreatorCard component internal styling/spacing
- Include toggle UI layout in playground
- Error state handling in playground

</decisions>
