## Decisions Made

- **`as any` cast for creatorProfile sub-selection:** The codegen type for `creatorProfile` doesn't exactly match `RawProfile` (Omit<..., 'id'>) — safe because `parseProfile` uses optional chaining on all fields. Matches the established cross-domain parser pattern from owned-assets.
- **stripExcluded nestedConfig for digitalAsset:** Includes `derivedFields: { standard: 'decimals' }` so that `standard` field follows `decimals` include toggle in nested digital asset stripping.
- **Name sort defaults to `nulls: 'last'`:** Both `creatorName` and `digitalAssetName` sorts default to nulls last when not specified, matching the established pattern from followers and profiles.
