**totalSupply.handler.ts:**

1. Add `import { resolveEntities } from '@/core/handlerHelpers';` and remove `import { In } from 'typeorm';` (no longer needed directly).

2. Replace the manual DB load + batch merge pattern (lines 65-74):

```typescript
// Load existing TotalSupply entities from DB
const existingEntities = new Map(
  await hctx.store
    .findBy(TotalSupply, { id: In(addresses) })
    .then((entities) => entities.map((entity) => [entity.id, entity])),
);

// Also check what's already in the BatchContext (from a previous trigger invocation)
const batchEntities = hctx.batchCtx.getEntities<TotalSupply>(ENTITY_TYPE);
```

With a single `resolveEntities` call:

```typescript
// Resolve existing TotalSupply entities from batch + DB
const existingEntities = await resolveEntities(
  hctx.store,
  hctx.batchCtx,
  ENTITY_TYPE,
  TotalSupply,
  addresses,
);
```

3. In the processing loop, update the entity resolution chain. Currently:

```typescript
let entity =
  updatedEntities.get(address) ?? batchEntities.get(address) ?? existingEntities.get(address);
```

`batchEntities` no longer exists as a separate variable — it's merged into `existingEntities` by `resolveEntities`. Change to:

```typescript
let entity = updatedEntities.get(address) ?? existingEntities.get(address);
```

4. The rest of the handler (entity construction with `...entity`, `...current` spread, addEntity, enrichment) is ALREADY correct and unchanged.

**ownedAssets.handler.ts:**

```typescript
  ownedAssetIds.size > 0
    ? hctx.store.findBy(OwnedAsset, { id: In([...ownedAssetIds]) })...
    : Promise.resolve(new Map<string, OwnedAsset>()),
```

With two parallel `resolveEntities` calls:

```typescript

const [existingOwnedAssetsMap, existingOwnedTokensMap] = await Promise.all([
  resolveEntities(hctx.store, hctx.batchCtx, OWNED_ASSET_TYPE, OwnedAsset, [...ownedAssetIds]),
  resolveEntities(hctx.store, hctx.batchCtx, OWNED_TOKEN_TYPE, OwnedToken, [...ownedTokenIds]),
]);
```

```typescript
const parentExists =
```
