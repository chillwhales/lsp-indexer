  <action>
Replace the entire content of `handlerHelpers.ts` with:

1. **`resolveEntity<T>(store, batchCtx, entityType, entityClass, id): Promise<T | null>`**

   - Check `batchCtx.getEntities<T>(entityType).get(id)` first — return if found
   - If not in batch, query `store.findOneBy(entityClass, { id } as FindOptionsWhere<T>)`
   - Return the DB entity or `null`
   - Import `FindOptionsWhere` from `typeorm`

2. **`resolveEntities<T>(store, batchCtx, entityType, entityClass, ids): Promise<Map<string, T>>`**

   - Start with ALL batch entities: `const batchEntities = batchCtx.getEntities<T>(entityType); const merged = new Map<string, T>(batchEntities);`
   - Filter `ids` to those NOT already in `batchEntities` (using `batchEntities.has(id)`)
   - If any remain, query `store.findBy(entityClass, { id: In(idsNotInBatch) } as FindOptionsWhere<T>)`
   - Add DB results to `merged` map
   - Return `merged`
   - This has IDENTICAL semantics to the old `mergeEntitiesFromBatchAndDb` — same batch-first, DB-second merge

3. **Delete `mergeEntitiesFromBatchAndDb`** entirely from the file.

4. Keep the existing imports for `Entity, IBatchContext` from `@/core/types`, `Store` from `@subsquid/typeorm-store`, `FindOptionsWhere, In` from `typeorm`.

5. The `EntityConstructor<T>` type is already exported from `@/core/types/entity.ts` — use it for the `entityClass` parameter type in both functions (instead of `new (props?: Partial<T>) => T`).

6. Update module JSDoc to describe the new resolve pattern.

IMPORTANT: Do NOT update any handler imports yet — those are in Plans 02–04. Handlers will fail to compile until they're updated, which is expected.
