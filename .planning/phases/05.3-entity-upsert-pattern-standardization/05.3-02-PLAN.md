**lsp5ReceivedAssets.handler.ts — BUG: missing addEntity on cross-batch merge**

1. Replace `import { mergeEntitiesFromBatchAndDb }` with `import { resolveEntities }` from `@/core/handlerHelpers`.

2. Replace the `mergeEntitiesFromBatchAndDb` call (line ~87) with `resolveEntities` (identical arguments — store, batchCtx, RECEIVED_ASSET_TYPE, LSP5ReceivedAsset, potentialIds).

3. **Critical fix in `extractFromIndex` and `extractFromMap`:** When an existing entity is found (the `if (existing)` branch), the handler currently mutates `existing` fields in-place and returns WITHOUT calling `addEntity()`. This means DB-sourced entities that get merged data are NEVER added back to the batch — their updates are silently lost. Fix both functions:

In `extractFromIndex`, change the existing-entity branch from:

```typescript
if (existing) {
  existing.arrayIndex = existing.arrayIndex ?? arrayIndex;
  existing.timestamp = timestamp;
  return;
}
```

To:

```typescript
if (existing) {
  const updated = new LSP5ReceivedAsset({
    ...existing,
    arrayIndex: existing.arrayIndex ?? arrayIndex,
    timestamp,
  });
  existingAssets.set(id, updated);
  hctx.batchCtx.addEntity(RECEIVED_ASSET_TYPE, id, updated);
  return;
}
```

In `extractFromMap`, change the existing-entity branch from:

```typescript
if (existing) {
  existing.interfaceId = interfaceId ?? existing.interfaceId;
  existing.arrayIndex = arrayIndex ?? existing.arrayIndex;
  existing.timestamp = timestamp;
  return;
}
```

To:

```typescript
if (existing) {
  const updated = new LSP5ReceivedAsset({
    ...existing,
    interfaceId: interfaceId ?? existing.interfaceId,
    arrayIndex: arrayIndex ?? existing.arrayIndex,
    timestamp,
  });
  existingAssets.set(id, updated);
  hctx.batchCtx.addEntity(RECEIVED_ASSET_TYPE, id, updated);
  return;
}
```

This ensures cross-batch merged entities are added to the batch for persistence.

**Note on enrichment for existing entities:** When an entity already exists (from DB or prior batch), its FKs are already populated from the enrichment queue in the original creation. We do NOT re-queue enrichment here — only new entities get enrichment queued (the `else` branch below the `if (existing)` is unchanged).
