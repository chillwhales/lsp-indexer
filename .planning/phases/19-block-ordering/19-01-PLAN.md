<task type="auto">
  <name>Task 2: Update EnrichmentRequest interface + run codegen + verify build</name>
  <files>packages/indexer/src/core/types/verification.ts</files>
  <action>
**Step 1: Update EnrichmentRequest interface**

In `packages/indexer/src/core/types/verification.ts`, add 3 new required fields to the `EnrichmentRequest<T>` interface:

```typescript
/** Block number from the event that created this enrichment request */
blockNumber: number;

/** Transaction index within the block */
transactionIndex: number;

/** Log index within the transaction */
logIndex: number;
```

Place them after the `fkField` field. These fields carry the blockchain position of the event that triggered the enrichment request, enabling the verification step to set block position on core entities (UP/DA/NFT) from the earliest request.

The `StoredEnrichmentRequest` type at the bottom of the file already includes all fields from `EnrichmentRequest` via `Omit<EnrichmentRequest<Entity>, 'fkField'>`, so it will automatically include the new fields.

**Step 2: Rebuild typeorm codegen**

Run:
```bash
pnpm --filter=@chillwhales/typeorm build
```

This regenerates TypeORM entity classes from the updated schema.graphql. All entities will now have `blockNumber`, `transactionIndex`, and `logIndex` properties.

**Step 3: Verify indexer build**

Run:
```bash
pnpm --filter=@chillwhales/indexer build
```

This will produce TypeScript compilation errors because:
1. All `queueEnrichment()` calls are now missing the 3 required block fields — this is expected and will be fixed in Plans 02 and 03.
2. OwnedAsset/OwnedToken handlers reference `block` which was renamed to `blockNumber` — this needs fixing here.

**Step 4: Fix OwnedAsset/OwnedToken `block` → `blockNumber` rename**

In `packages/indexer/src/handlers/ownedAssets.handler.ts`, replace all occurrences of `block: blockNumber` with `blockNumber: transfer.blockNumber` (or just `blockNumber` since it comes from Transfer destructuring). The Transfer entity already has `blockNumber` — the handler was aliasing `blockNumber` to `block` for the OwnedAsset constructor. After the rename, the field names align.

Specifically:
- `block: blockNumber` → `blockNumber` (uses Transfer's blockNumber directly)
- For newly created OwnedAsset/OwnedToken: add `transactionIndex: transfer.transactionIndex, logIndex: transfer.logIndex`

**Step 5: Make enrichment requests compile with temporary 0 values**

To let the build succeed (so subsequent plans can build on it), temporarily add `blockNumber: 0, transactionIndex: 0, logIndex: 0` to ALL existing `queueEnrichment()` calls across plugins and handlers. This is a mechanical search-and-replace operation.

Search for all occurrences of `queueEnrichment<` or `queueEnrichment({` and add the 3 fields. Plans 02 and 03 will replace these placeholder values with real block data from events.

**Step 6: Verify full build**

Run:
```bash
pnpm --filter=@chillwhales/typeorm build && pnpm --filter=@chillwhales/indexer build
```

Both packages must build with zero errors.
