## Decisions Made

- **Direction-aware fetchFollowers:** Single function serves both useFollowers and useFollowing via `direction` param that selects the primary WHERE field (`followed_address` for followers, `follower_address` for following)
- **Recursive profile sub-include:** `parseProfile` called with `include?.followerProfile` / `include?.followedProfile` sub-includes — each side independently narrowable without nestedConfig in stripExcluded
- **fetchIsFollowing efficiency:** All 22 include variables set to `false` explicitly, limit: 1 — minimal data transfer for boolean existence check
- **Primary vs secondary filter matching:** Primary address uses exact `escapeLike()` (no % wrapping), filter fields use partial `%${escapeLike()}%`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed parseProfile sub-include type compatibility**

- **Found during:** Task 1 (Parser creation)
- **Issue:** `parseProfile(raw, include?.followerProfile)` failed type check because `include?.followerProfile` can be `undefined`, but parseProfile's second overload requires non-undefined `ProfileInclude`
- **Fix:** Added conditional branches: `include?.followerProfile ? parseProfile(raw, include.followerProfile) : parseProfile(raw)` — matches overload signatures correctly
- **Files modified:** packages/node/src/parsers/followers.ts
- **Verification:** `pnpm --filter @lsp-indexer/node typecheck` passes
- **Committed in:** 2f72914 (Task 1 commit)

**2. [Rule 1 - Bug] Fixed Follower literal type incompatibility with PartialProfile**

- **Found during:** Task 1 (Parser creation)
- **Issue:** Building `result: Follower` with a `PartialProfile` value for `followerProfile` field failed because `Follower.followerProfile` expects `Profile | null`, not `PartialProfile`
- **Fix:** Built profiles separately, then assembled result with `as Follower` cast — profiles are pre-narrowed by sub-include, outer stripExcluded handles field removal
- **Files modified:** packages/node/src/parsers/followers.ts
- **Verification:** `pnpm --filter @lsp-indexer/node typecheck` passes
- **Committed in:** 2f72914 (Task 1 commit)
