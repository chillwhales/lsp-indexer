## Decisions Made

- **getFollowers server action accepts direction param:** Single server action serves both useFollowers and useFollowing Next.js hooks. The direction param (`'followers'` or `'following'`) is passed through to `fetchFollowers`.
- **useFollowing returns `{ following }` not `{ followers }`:** Semantic clarity — consumers see `following` for "who this address follows" and `followers` for "who follows this address". Internally both map from `data?.followers` returned by `fetchFollowers`.
- **useFollowCount returns 0 defaults:** `followerCount: data?.followerCount ?? 0` and `followingCount: data?.followingCount ?? 0` — safe for UI rendering before data loads.
- **useIsFollowing enabled guard:** `Boolean(followerAddress) && Boolean(followedAddress)` — prevents queries when either address is empty/falsy.
- **Infinite hooks have enabled guard on address:** Unlike digital-assets/nfts infinite hooks, follower infinite hooks require `address` and use `enabled: Boolean(address)` since they're always address-scoped.
