**Create `packages/node/src/keys/followers.ts`** — query key factory with expanded namespaces to support the 6 hooks.

The follower domain needs more key namespaces than a typical domain because it has 6 hooks with different query shapes:

```typescript
import type { FollowerFilter, FollowerInclude, FollowerSort } from '@lsp-indexer/types';

/**
 * Query key factory for the follower domain.
 *
 * Supports 6 hooks with distinct cache key namespaces:
 * - `followers.*` — useFollowers (list of who follows an address)
 * - `following.*` — useFollowing (list of who an address follows)
 * - `follow-count.*` — useFollowCount (aggregate counts)
 * - `is-following.*` — useIsFollowing (boolean existence check)
 *
 * Each namespace has its own `list` and `infinite` variants where applicable.
 */
export const followerKeys = {
  // Root key for entire domain
  all: ['followers'] as const,

  // --- Followers (who follows address X) ---
  followers: () => [...followerKeys.all, 'followers'] as const,
  followersList: (
    address: string,
    filter?: FollowerFilter,
    sort?: FollowerSort,
    limit?: number,
    offset?: number,
    include?: FollowerInclude,
  ) =>
    [...followerKeys.followers(), 'list', address, filter, sort, limit, offset, include] as const,
  followersInfinite: (
    address: string,
    filter?: FollowerFilter,
    sort?: FollowerSort,
    include?: FollowerInclude,
  ) => [...followerKeys.followers(), 'infinite', address, filter, sort, include] as const,

  // --- Following (who does address X follow) ---
  following: () => [...followerKeys.all, 'following'] as const,
  followingList: (
    address: string,
    filter?: FollowerFilter,
    sort?: FollowerSort,
    limit?: number,
    offset?: number,
    include?: FollowerInclude,
  ) =>
    [...followerKeys.following(), 'list', address, filter, sort, limit, offset, include] as const,
  followingInfinite: (
    address: string,
    filter?: FollowerFilter,
    sort?: FollowerSort,
    include?: FollowerInclude,
  ) => [...followerKeys.following(), 'infinite', address, filter, sort, include] as const,

  // --- Follow Count (aggregate counts for an address) ---
  counts: () => [...followerKeys.all, 'count'] as const,
  count: (address: string) => [...followerKeys.counts(), address] as const,

  // --- Is Following (boolean check: does A follow B?) ---
  isFollowings: () => [...followerKeys.all, 'is-following'] as const,
  isFollowing: (followerAddress: string, followedAddress: string) =>
    [...followerKeys.isFollowings(), followerAddress, followedAddress] as const,
};
```

**Create `packages/node/src/parsers/followers.ts`** — Hasura → Follower transformation with conditional include support.

The parser must:

1. Map `follower_address` → `followerAddress` (snake_case → camelCase)
2. Map `followed_address` → `followedAddress`
3. Map `timestamp`, `address` directly (already camelCase in Hasura)
4. Parse nested `followerUniversalProfile` using `parseProfile` from `./profiles` — passing sub-include for narrowing
5. Parse nested `followedUniversalProfile` using `parseProfile` — passing sub-include for narrowing
6. Support conditional include narrowing via `stripExcluded`
7. Use `as FollowerResult<I>` cast at service boundary (established pattern)

```typescript
import type {
  Follower,
  FollowerInclude,
  FollowerResult,
  PartialFollower,
} from '@lsp-indexer/types';
import { parseProfile } from './profiles';
import { stripExcluded } from './strip';

/**
 * Parse a Hasura follower response into a clean Follower type.
 *
 * Maps snake_case columns (follower_address, followed_address) to camelCase.
 * Delegates profile parsing to parseProfile for nested relations.
 *
 * @overload No include param — returns full Follower type
 * @overload With include param — returns narrowed FollowerResult<I>
 */
export function parseFollower(raw: any): Follower;
export function parseFollower<const I extends FollowerInclude>(
  raw: any,
  include: I,
): FollowerResult<I>;
export function parseFollower(raw: any, include?: FollowerInclude): Follower | PartialFollower {
  const result: Follower = {
    followerAddress: raw.follower_address,
    followedAddress: raw.followed_address,
    timestamp: raw.timestamp,
    address: raw.address,
    followerProfile: raw.followerUniversalProfile
      ? parseProfile(raw.followerUniversalProfile, include?.followerProfile)
      : null,
    followedProfile: raw.followedUniversalProfile
      ? parseProfile(raw.followedUniversalProfile, include?.followedProfile)
      : null,
  };

  if (include) {
    return stripExcluded(result, include, ['followerAddress', 'followedAddress']) as FollowerResult<
      typeof include
    >;
  }
  return result;
}

/**
 * Parse an array of Hasura follower responses.
 */
export function parseFollowers(raw: any[]): Follower[];
export function parseFollowers<const I extends FollowerInclude>(
  raw: any[],
  include: I,
): FollowerResult<I>[];
export function parseFollowers(
  raw: any[],
  include?: FollowerInclude,
): (Follower | PartialFollower)[] {
  if (include) return raw.map((r) => parseFollower(r, include));
  return raw.map((r) => parseFollower(r));
}
```

**IMPORTANT:** The parseProfile function MUST be called with the sub-include param (`include?.followerProfile` and `include?.followedProfile`) for recursive nested stripping to work. This matches the established pattern in NFT parser (which calls `parseDigitalAsset(raw, include?.collection)`).

Also note: when `include` is provided but `followerProfile`/`followedProfile` are not in the include, `stripExcluded` will remove those fields from the result. When they ARE in the include, `parseProfile` handles sub-field narrowing.

**`buildFollowerWhere(filter, primaryAddress, primaryField)`** — translate FollowerFilter + primary address to Hasura `follower_bool_exp`.

The `primaryAddress` and `primaryField` params allow the same builder to serve both useFollowers and useFollowing:

- useFollowers: `primaryField = 'followed_address'`, `primaryAddress = params.address` (find followers OF this address)
- useFollowing: `primaryField = 'follower_address'`, `primaryAddress = params.address` (find who this address FOLLOWS)

Filter → Hasura mapping:

- `primaryAddress` → `{ [primaryField]: { _ilike: escapeLike(primaryAddress) } }` — EXACT match, not partial (specific address lookup)
- `filter.followerAddress` → `{ follower_address: { _ilike: \`%${escapeLike(followerAddress)}%\` } }` — partial match for filtering within results
- `filter.followedAddress` → `{ followed_address: { _ilike: \`%${escapeLike(followedAddress)}%\` } }` — partial match
- `filter.followerName` → `{ followerUniversalProfile: { lsp3Profile: { name: { _ilike: \`%${escapeLike(followerName)}%\` } } } }` — nested profile name search
- `filter.followedName` → `{ followedUniversalProfile: { lsp3Profile: { name: { _ilike: \`%${escapeLike(followedName)}%\` } } } }` — nested profile name search
- `filter.timestampFrom` → `{ timestamp: { _gte: timestampFrom } }` — ISO timestamp lower bound
- `filter.timestampTo` → `{ timestamp: { _lte: timestampTo } }` — ISO timestamp upper bound

Multiple conditions combine with `_and`. Empty filter + no primary = empty object.

**IMPORTANT for primary address:** Use `escapeLike(primaryAddress)` NOT `%${escapeLike(primaryAddress)}%` — the primary address is an exact match (we want followers of EXACTLY this address, not addresses containing it). The Hasura `_ilike` is used for case-insensitivity only.

Sort field → Hasura mapping:

- `'timestamp'` → `[{ timestamp: orderDir(direction, nulls) }]`

**`buildFollowerIncludeVars(include)`** — translate FollowerInclude to GraphQL boolean variables.

**Export this function** — it may be needed by future domains that embed followers.

**Inverted default pattern:**

- When `include` is undefined → return `{}` (GraphQL defaults all Boolean! = true)
- When `include` is provided → each field defaults to `false` unless explicitly set

```typescript
export function buildFollowerIncludeVars(include?: FollowerInclude): Record<string, boolean> {
  if (!include) return {};

  const activeFollowerProfile = hasActiveIncludes(include.followerProfile);
  const activeFollowedProfile = hasActiveIncludes(include.followedProfile);

  const vars: Record<string, boolean> = {
    includeTimestamp: include.timestamp ?? false,
    includeAddress: include.address ?? false,
    includeFollowerProfile: activeFollowerProfile,
    includeFollowedProfile: activeFollowedProfile,
  };

  // Follower profile sub-includes: reuse profile include builder with "FollowerProfile" prefix
  if (activeFollowerProfile) {
    const profileVars = buildProfileIncludeVars(include.followerProfile);
    for (const [key, val] of Object.entries(profileVars)) {
      // includeProfileX → includeFollowerProfileX
      vars[key.replace('includeProfile', 'includeFollowerProfile')] = val;
    }
  }

  // Followed profile sub-includes: same pattern with "FollowedProfile" prefix
  if (activeFollowedProfile) {
    const profileVars = buildProfileIncludeVars(include.followedProfile);
    for (const [key, val] of Object.entries(profileVars)) {
      // includeProfileX → includeFollowedProfileX
      vars[key.replace('includeProfile', 'includeFollowedProfile')] = val;
    }
  }

  return vars;
}
```

This follows the exact pattern from NFT service (which does the same prefix replacement for collection/holder).

```typescript
export async function fetchFollowers(
  url: string,
  params: {
    direction: 'followers' | 'following';
    address: string;
    filter?: FollowerFilter;
    sort?: FollowerSort;
    limit?: number;
    offset?: number;
    include: FollowerInclude;
  },
): Promise<FetchFollowersResult<PartialFollower>>;
export async function fetchFollowers(
  url: string,
  params: {
    direction: 'followers' | 'following';
    address: string;
    filter?: FollowerFilter;
    sort?: FollowerSort;
    limit?: number;
    offset?: number;
    include?: FollowerInclude;
  },
): Promise<FetchFollowersResult | FetchFollowersResult<PartialFollower>> {
  const primaryField =
    params.direction === 'followers'
      ? 'followed_address' // "followers of X" → where followed_address = X
      : 'follower_address'; // "following by X" → where follower_address = X

  const where = buildFollowerWhere(params.filter, params.address, primaryField);
  const orderBy = buildFollowerOrderBy(params.sort);
  const includeVars = buildFollowerIncludeVars(params.include);

  const result = await execute(url, GetFollowersDocument, {
    where: Object.keys(where).length > 0 ? where : undefined,
    order_by: orderBy,
    limit: params.limit,
    offset: params.offset,
    ...includeVars,
  });

  if (params.include) {
    return {
      followers: parseFollowers(result.follower, params.include),
      totalCount: result.follower_aggregate?.aggregate?.count ?? 0,
    };
  }
  return {
    followers: parseFollowers(result.follower),
    totalCount: result.follower_aggregate?.aggregate?.count ?? 0,
  };
}
```

The `direction` param tells the service which primary field to use. Hooks set this internally.

**`fetchFollowCount(url, params)` — aggregate counts:**

```typescript
export async function fetchFollowCount(
  url: string,
  params: { address: string },
): Promise<FollowCount> {
  const addressPattern = escapeLike(params.address);

  const result = await execute(url, GetFollowCountDocument, {
    followerWhere: { followed_address: { _ilike: addressPattern } },
    followingWhere: { follower_address: { _ilike: addressPattern } },
  });

  return {
    followerCount: result.followerCount?.aggregate?.count ?? 0,
    followingCount: result.followingCount?.aggregate?.count ?? 0,
  };
}
```

**IMPORTANT:** Uses exact-match `_ilike` (no `%` wrapping) — we want the count for EXACTLY this address, case-insensitively.

**`fetchIsFollowing(url, params)` — boolean existence check:**

```typescript
export async function fetchIsFollowing(
  url: string,
  params: { followerAddress: string; followedAddress: string },
): Promise<boolean> {
  const result = await execute(url, GetFollowersDocument, {
    where: {
      _and: [
        { follower_address: { _ilike: escapeLike(params.followerAddress) } },
        { followed_address: { _ilike: escapeLike(params.followedAddress) } },
      ],
    },
    limit: 1,
    // Disable all includes — we only need to know if a record exists
    includeTimestamp: false,
    includeAddress: false,
    includeFollowerProfile: false,
    includeFollowedProfile: false,
  });

  return result.follower.length > 0;
}
```

This reuses `GetFollowersDocument` with minimal fields (all includes disabled) and `limit: 1` for efficiency.
