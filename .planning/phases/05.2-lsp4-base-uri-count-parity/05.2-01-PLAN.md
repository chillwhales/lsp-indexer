Fix OwnedAsset double-processing bug and mark LSP8ReferenceContract as known V1 divergence.

**ownedAssets.handler.ts — Fix double-processing (GAP-08):**

The bug is on line 43: `_triggeredBy` (unused) and lines 45-46 where BOTH bags are always read. Fix by using `triggeredBy` to select only the relevant bag's transfers:

1. Change `_triggeredBy: string` to `triggeredBy: string` (remove underscore prefix)
2. Replace the dual-bag read logic (lines 45-48) with conditional logic based on `triggeredBy`:
   - If `triggeredBy === 'LSP7Transfer'`: only read LSP7Transfer bag
   - If `triggeredBy === 'LSP8Transfer'`: only read LSP8Transfer bag
   - Combine into `allTransfers` array (only from the triggered bag)
3. The rest of the handler logic stays the same — it already works with any `allTransfers` array

This follows the same pattern used by `nft.handler.ts` which correctly branches on `triggeredBy`.

The key insight: the handler is called twice per batch (once per bag key). Previously it processed ALL transfers from BOTH bags each time = 2x processing. With the fix, each invocation only processes its own bag's transfers = correct 1x processing.
