Create a new handler file: `packages/indexer-v2/src/handlers/lsp4MetadataBaseUri.handler.ts`

**Handler structure:**

```typescript
const LSP4MetadataBaseUriHandler: EntityHandler = {
  name: 'lsp4MetadataBaseUri',
  listensToBag: ['LSP8Transfer', 'LSP8TokenMetadataBaseURI'],
  dependsOn: ['lsp8MetadataBaseURI', 'nft', 'formattedTokenId'],

  async handle(hctx: HandlerContext, triggeredBy: string): Promise<void> {
    if (triggeredBy === 'LSP8TokenMetadataBaseURI') {
      await handleBaseUriChanged(hctx);
    } else if (triggeredBy === 'LSP8Transfer') {
      await handleMints(hctx);
    }
  },
};
```

**Key design decisions:**

- `listensToBag: ['LSP8Transfer', 'LSP8TokenMetadataBaseURI']` — triggered by either new mints or base URI changes
- `dependsOn: ['lsp8MetadataBaseURI', 'nft', 'formattedTokenId']` — needs `LSP8TokenMetadataBaseURI` entities decoded, `NFT` entities created, and `formattedTokenId` populated before this handler runs
- Uses `triggeredBy` to branch between the two paths

**Path 1: handleBaseUriChanged(hctx) — when LSP8TokenMetadataBaseURI changes:**

1. Get `LSP8TokenMetadataBaseURI` entities from batch: `hctx.batchCtx.getEntities<LSP8TokenMetadataBaseURI>('LSP8TokenMetadataBaseURI')`
2. Collect unique addresses from these entities
3. Load ALL NFTs for those addresses from DB: `hctx.store.findBy(NFT, { address: In([...addresses]) })`
4. Also check NFTs in current batch (may have mints in same batch as base URI change): `hctx.batchCtx.getEntities<NFT>('NFT')`
5. Merge DB NFTs + batch NFTs (batch takes priority if same ID exists in both)
6. For each base URI entity, find all NFTs with matching address
7. For each NFT, derive URL: `baseUri.endsWith('/') ? baseUri + nft.formattedTokenId : baseUri + '/' + nft.formattedTokenId`
   - If `nft.formattedTokenId` is null, use the raw `nft.tokenId` as fallback (V1 uses `formattedTokenId` from the NFT entity which may have been populated by the formattedTokenId handler)
8. Create LSP4Metadata entity:
   ```typescript
   new LSP4Metadata({
     id: `BaseURI - ${generateTokenId({ address, tokenId: nft.tokenId })}`,
     address,
     timestamp: baseUriEntity.timestamp,
     tokenId: nft.tokenId,
     url: derivedUrl,
     rawValue: baseUriEntity.rawValue,
     isDataFetched: false,
     retryCount: 0,
     digitalAsset: null,
     nft: null,
   });
   ```
9. Add to BatchContext: `hctx.batchCtx.addEntity('LSP4Metadata', entity.id, entity)`
10. Queue enrichment for digitalAsset FK (category: DigitalAsset, address)
11. Queue enrichment for nft FK (category: NFT, address + tokenId)

**Path 2: handleMints(hctx) — when new NFTs are minted:**

1. Get LSP8Transfer entities from batch: `hctx.batchCtx.getEntities<Transfer>('LSP8Transfer')`
2. Filter to mints only: `isAddressEqual(getAddress(transfer.from), zeroAddress)` AND `transfer.tokenId` is truthy (LSP8 only, not LSP7)
3. Collect unique contract addresses from mints
4. Load LSP8TokenMetadataBaseURI from DB for those addresses: `hctx.store.findBy(LSP8TokenMetadataBaseURI, { address: In([...addresses]) })`
5. Also check batch for LSP8TokenMetadataBaseURI entities (may be in same batch)
6. Merge DB + batch base URIs (batch takes priority). For each address, take the most recent (by timestamp, or batch over DB)
7. For each mint that has a matching base URI:
   - Look up the NFT entity from batch to get formattedTokenId: `hctx.batchCtx.getEntities<NFT>('NFT').get(generateTokenId({ address, tokenId }))`
   - Derive URL using same formula
   - Create LSP4Metadata entity with same structure as Path 1
   - Add to batch + queue enrichment

**Imports needed:**

- `{ EntityCategory, EntityHandler, HandlerContext }` from `@/core/types`
- `{ generateTokenId }` from `@/utils`
- `{ LSP4Metadata, LSP8TokenMetadataBaseURI, NFT, Transfer }` from `@chillwhales/typeorm`
- `{ In }` from `typeorm`
- `{ getAddress, isAddressEqual, zeroAddress }` from `viem`

**Entity type constant:** `const LSP4_ENTITY_TYPE = 'LSP4Metadata';`

**IMPORTANT — URL derivation must use formattedTokenId when available:**
V1 code at line 103-106 uses `formattedTokenId` for URL construction, not raw `tokenId`. When `formattedTokenId` is null (unknown format), fall back to raw `tokenId`. This matches V1 where `nft.formattedTokenId` is always populated (V1 returns raw tokenId for unknown formats, but V2 returns null — so we need the fallback).

**IMPORTANT — deduplication:**
Use a Map keyed by entity ID to avoid duplicate LSP4Metadata entities when the same NFT appears in both mint and base URI change paths within the same batch.

**Export:** `export default LSP4MetadataBaseUriHandler;`
