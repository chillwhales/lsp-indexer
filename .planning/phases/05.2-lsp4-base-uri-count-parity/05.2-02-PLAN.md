    packages/indexer-v2/src/handlers/chillwhales/orbLevel.handler.ts

  <action>
Modify orbLevel.handler.ts to:

1. **Add 'LSP8Transfer' to listensToBag:** Change `listensToBag: ['TokenIdDataChanged']` to `listensToBag: ['LSP8Transfer', 'TokenIdDataChanged']`

2. **Add Transfer import:** Add `Transfer` to the `@chillwhales/typeorm` import

3. **Add viem imports:** Add `zeroAddress` to the viem import (isAddressEqual and getAddress are already imported)

4. **Branch on triggeredBy:** Use the `triggeredBy` parameter to branch logic (same pattern as nft.handler.ts):

   - If `triggeredBy === 'LSP8Transfer'`: handle mints (new code)
   - If `triggeredBy === 'TokenIdDataChanged'`: existing logic (unchanged)

5. **Implement mint detection branch:**

   - Get LSP8Transfer entities from batch: `hctx.batchCtx.getEntities<Transfer>(triggeredBy)`
   - Filter to: `isAddressEqual(getAddress(event.address), ORBS_ADDRESS)` AND `isAddressEqual(getAddress(event.from), zeroAddress)` (mint from zero address)
   - For each mint, create **OrbLevel** with `value: 0` AND **OrbCooldownExpiry** with `value: 0`
   - Entity ID format: `generateTokenId({ address: event.address, tokenId: event.tokenId })`
   - Set `digitalAsset: null, nft: null` (FKs resolved via enrichment)
   - Add both entities to BatchContext
   - Queue enrichment for digitalAsset and nft FKs on both entities (same enrichment pattern as existing code)

6. **Key behavior:** On mint, defaults are created. If a `TokenIdDataChanged` event arrives in the same or later batch, it overwrites the defaults (because `addEntity` with the same ID replaces the existing entity in the bag). This matches V1 behavior where the Map.set() overwrites.

Reference: V1's orbsLevelHandler.ts lines 35-84 for the mint default logic.
