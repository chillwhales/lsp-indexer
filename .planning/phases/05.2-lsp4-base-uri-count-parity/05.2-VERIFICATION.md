## Summary

### GAP-06: LSP4Metadata Base URI Derivation (✓ CLOSED)

New `lsp4MetadataBaseUri.handler.ts` implements both trigger paths:

1. On mint: checks if parent collection has base URI, derives per-token URL
2. On base URI change: queries ALL existing NFTs, derives URLs for each

Entity ID format matches V1 (`BaseURI - {address} - {tokenId}`). URL derivation handles trailing slash correctly and falls back to raw tokenId when formattedTokenId is null. Created entities automatically picked up by existing lsp4MetadataFetch handler. Expected to close ~84K entity gap.

### GAP-07: LSP8ReferenceContract Count Mismatch (✓ DOCUMENTED)

Marked as known V1 divergence in comparison tool. V1 has switch fall-through bug where unrelated DataChanged events fall through to LSP8ReferenceContract case, creating phantom entities. V2 behavior is correct. Comparison tool will skip this entity type in v1-v2 mode.

### GAP-08: OwnedAsset Double-Processing (✓ FIXED)

Fixed `ownedAssets.handler.ts` to use triggeredBy parameter for conditional bag filtering. Handler is called twice per batch (once for LSP7Transfer, once for LSP8Transfer). Previously read both bags each time = 2x processing. Now reads only triggered bag per invocation = correct 1x processing. Unit tests prove no double-counting occurs. Expected to reduce OwnedAsset count by ~14K entities.

### GAP-09: Orb Entity Counts (✓ FIXED)

Updated `orbLevel.handler.ts` and `orbFaction.handler.ts` to listen to both LSP8Transfer and TokenIdDataChanged. On Orb NFT mint (from zero address), creates default entities: OrbLevel(value=0), OrbCooldownExpiry(value=0), OrbFaction(value='Neutral'). Subsequent TokenIdDataChanged events overwrite defaults with actual on-chain values via addEntity() Map.set() semantics. Matches V1 behavior exactly.
