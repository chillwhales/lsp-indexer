# Phase 20.2: Pipeline Instrumentation — Context

## Problem

The pipeline (`processBatch()` in `core/pipeline.ts`) has zero timing instrumentation. We can't answer "how long does VERIFY take?" or "what's the total batch processing time?" Additionally, EXTRACT (Step 1) and HANDLE (Step 3) have no logger at all — they're completely silent.

## What to Add

### 1. Step Timing

Wrap each step with `performance.now()`:
```typescript
const extractStart = performance.now();
// ... EXTRACT step ...
const extractDurationMs = Math.round(performance.now() - extractStart);
```

Each step's existing logger (or new logger for EXTRACT/HANDLE) should emit:
```typescript
stepLog.info({ durationMs: extractDurationMs, ...existingFields }, 'Step complete');
```

### 2. Missing Loggers

- **EXTRACT**: Create `createStepLogger(context.log, 'EXTRACT', blockRange)` at line ~209
- **HANDLE**: Create `createStepLogger(context.log, 'HANDLE', blockRange)` at line ~262

### 3. Batch Summary Log

At end of `processBatch()` (line 613-614), emit a single summary:

```typescript
batchLog.info({
  step: 'BATCH_SUMMARY',
  blockRange: `${firstBlock}-${lastBlock}`,
  blockCount: context.blocks.length,
  totalEntities: batchCtx.getTotalEntityCount(),
  totalEnrichments: batchCtx.getEnrichmentQueue().length,
  stepTimings: {
    extract: extractMs,
    persistRaw: persistRawMs,
    handle: handleMs,
    clearSubEntities: clearMs,
    deleteEntities: deleteMs,
    persistDerived: persistDerivedMs,
    verify: verifyMs,
    enrich: enrichMs,
    resolve: resolveMs,
  },
  totalDurationMs: totalMs,
}, 'Batch complete');
```

### 4. PipelineStep Type Update

Add to `core/logger.ts` PipelineStep union:
- `'BATCH_SUMMARY'` — for the summary log

## Entry Point
