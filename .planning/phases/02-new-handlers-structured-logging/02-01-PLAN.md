Install pino and pino-roll in the indexer-v2 package:
```
pnpm add pino pino-roll --filter indexer-v2
```

If package.json doesn't exist yet (Phase 1 creates it), create a minimal one with name `indexer-v2` first, then install.

Create `packages/indexer-v2/src/core/logger.ts` with these exports:

1. **`PipelineStep` type** — union of string literals for all V2 pipeline steps:
   `'EXTRACT' | 'PERSIST_RAW' | 'HANDLE' | 'CLEAR_SUB_ENTITIES' | 'DELETE_ENTITIES' | 'PERSIST_DERIVED' | 'VERIFY' | 'ENRICH'`

2. **`createStepLogger(baseLogger, step, blockRange?)`** — creates a child logger via `baseLogger.child({ step, ...blockRange })` that injects the `step` field on every log call. `blockRange` is optional `{ from: number; to: number }` formatted as `"${from}-${to}"` string. The `baseLogger` parameter accepts Subsquid's `Logger` type from `@subsquid/logger`.

3. **`initFileLogger(logDir)`** — creates a singleton pino logger instance that:

   - Writes structured JSON lines to rotating files via `pino-roll` transport
   - File pattern: `${logDir}/indexer` with `.log` extension, daily rotation, `mkdir: true`
   - Level defaults to `debug` when `NODE_ENV !== 'production'`, `info` when `NODE_ENV === 'production'`
   - `LOG_LEVEL` env var overrides the auto-detected default (per user decision)
   - Should be called once at application startup

4. **`getFileLogger()`** — returns the singleton pino logger or null if not initialized.

5. **`createDualLogger(subsquidLogger, step, blockRange?)`** — convenience wrapper that returns an object with `info(attrs, msg)`, `warn(attrs, msg)`, `error(attrs, msg)`, `debug(attrs, msg)` methods that:
   - Call subsquid child logger's method with attributes directly (NOT JSON.stringify)
   - Also call pino file logger if initialized
   - Base fields on every line: `level` (from method name), `step` (from parameter), `message` (from msg), `timestamp` (auto from pino/subsquid)
   - Additional fields passed as `attrs` object (entity type, count, handler name, block range, etc.)

Import types: use `import type { Logger } from '@subsquid/logger'` for the base logger parameter type.

IMPORTANT per user decision: Four severity levels only (debug, info, warn, error). No trace/fatal.
IMPORTANT per user decision: Structured JSON lines everywhere — both stdout (via subsquid) and file (via pino).
