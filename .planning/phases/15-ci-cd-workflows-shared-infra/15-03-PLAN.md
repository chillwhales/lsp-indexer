  <action>
    **Create the `chillwhales/.github` org repo** using the GitHub CLI, then populate it with reusable workflows and composite actions.

    **Step 1: Check if repo exists and create if not:**
    ```bash
    gh repo view chillwhales/.github 2>/dev/null || gh repo create chillwhales/.github --public --description "Org-wide GitHub Actions workflows and composite actions"
    ```
    Clone the repo to a temp directory for workflow creation.

    **Step 2: Create composite actions (building blocks used within jobs):**

    **`setup-pnpm` composite action** (`.github/actions/setup-pnpm/action.yml`):
    - Inputs: `node-version` (default: '22'), `fetch-depth` (default: '1')
    - Steps: checkout@v4 → pnpm/action-setup@v4 → setup-node@v4 with cache: pnpm → pnpm install --frozen-lockfile
    - This eliminates 4-step boilerplate from every job

    **`build-and-upload` composite action** (`.github/actions/build-and-upload/action.yml`):
    - Inputs: `build-command` (required), `artifact-name` (default: 'build-output'), `artifact-paths` (required, newline-separated)
    - Steps: Run build command → upload-artifact@v4
    - Abstracts build + artifact upload pattern

    **Step 3: Create reusable workflows:**

    **`ci-build-lint-test.yml`** (`.github/workflows/ci-build-lint-test.yml`):
    - `on: workflow_call` with inputs:
      - `node-version` (number, default: 22)
      - `build-command` (string, required) — e.g., "pnpm build"
      - `lint-command` (string, optional) — e.g., "pnpm lint"
      - `format-command` (string, optional) — e.g., "pnpm format:check"
      - `test-command` (string, optional) — e.g., "pnpm test:coverage"
      - `typecheck-command` (string, optional) — e.g., "pnpm typecheck"
      - `pre-lint-command` (string, optional) — for building deps before lint (e.g., "pnpm --filter=@chillwhales/abi build && pnpm --filter=@chillwhales/typeorm build")
      - `test-node-versions` (string, optional, default: '[20, 22]') — JSON array for test matrix
      - `artifact-paths` (string, required) — newline-separated paths to upload after build
      - `coverage-artifact-name` (string, default: 'coverage-reports')
    - Jobs: install → {format, lint, build} → {typecheck, test, pkg-verify (skipped here)} → coverage-report
    - Each job uses setup-pnpm composite action internally (checkout + pnpm + node + install steps)
    - Build job uploads artifacts. Downstream jobs download them.
    - Test job uses matrix from `test-node-versions` input. Uploads coverage from highest version only.
    - Coverage job runs `davelosert/vitest-coverage-report-action@v2` on PRs only.

    **`ci-quality.yml`** (`.github/workflows/ci-quality.yml`):
    - `on: workflow_call` with inputs:
      - `node-version` (number, default: 22)
      - `verify-command` (string, optional) — e.g., "pnpm validate:publish" (publint + attw)
      - `changeset-check` (boolean, default: true) — whether to run changeset status check
    - Jobs: pkg-verify (downloads build artifact, runs verify-command), changeset-check (PR only, runs `pnpm changeset status --since=origin/main`)
    - Needs build artifact from caller workflow — use `actions/download-artifact@v4` (artifacts are shared within the same workflow run when using reusable workflows from the same caller)

    **IMPORTANT about artifact sharing with reusable workflows:** Reusable workflows called via `uses:` run in the same workflow run context as the caller. Artifacts uploaded in one reusable workflow job are available to other jobs in the same caller workflow via `actions/download-artifact@v4`.

    **`ci-publish-validation.yml`** (`.github/workflows/ci-publish-validation.yml`):
    - `on: workflow_call` with inputs:
      - `node-version` (number, default: 22)
      - `preview-packages` (string, optional) — space-separated list of package dirs for pkg-pr-new
    - Jobs: preview (runs pkg-pr-new on PRs with fork security guard)
    - This is thinner than the others — release is repo-specific (Docker, specific publish commands)

    **Step 4: Add README.md** to the repo explaining:
    - Available composite actions and their inputs
    - Available reusable workflows and their inputs
    - Example caller workflow
    - How to add new workflows

    **Step 5: Commit and push** to the chillwhales/.github repo.

    **IMPORTANT CONSTRAINTS:**
    - Reusable workflows MUST use `workflow_call` trigger
    - Composite actions use `using: 'composite'` with `shell: bash` on run steps
    - All inputs should have sensible defaults so callers only override what's different
    - Keep workflows generic — no lsp-indexer-specific logic inside shared workflows

    **Step 1: Refactor `.github/workflows/ci.yml`** to consume the reusable workflows from `chillwhales/.github`:

    The CI workflow should call the shared reusable workflows where they provide value, while keeping repo-specific jobs inline. The pattern:

    ```yaml
    name: CI
    on:
      push:
        branches: [main, refactor/indexer-v2-react]
      pull_request:
        branches: [main, refactor/indexer-v2-react]

    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true

    jobs:
      # Shared build-lint-test pipeline
      ci:
        uses: chillwhales/.github/.github/workflows/ci-build-lint-test.yml@main
        with:
          node-version: 22
          build-command: pnpm build
          lint-command: pnpm lint
          format-command: pnpm format:check
          typecheck-command: pnpm typecheck
          test-command: pnpm test:coverage
          pre-lint-command: "pnpm --filter=@chillwhales/abi build && pnpm --filter=@chillwhales/typeorm build"
          artifact-paths: |
            packages/*/dist
            packages/abi/lib
            packages/typeorm/lib

      # Shared quality checks
      quality:
        needs: [ci]
        uses: chillwhales/.github/.github/workflows/ci-quality.yml@main
        with:
          verify-command: pnpm validate:publish
          changeset-check: true
    ```

    **HOWEVER** — there's a critical constraint: reusable workflows have limitations. If the shared workflows can't handle all the nuances (pre-lint build of dependency packages, artifact sharing across reusable workflow boundaries, test matrix customization), then the refactored CI should use the shared **composite actions** (setup-pnpm) within inline jobs instead. The executor should:

    1. First try the full reusable workflow approach
    2. If it hits limitations (artifact sharing, secrets propagation, etc.), fall back to using composite actions within inline jobs
    3. Document which approach was used and why in the decision document

    **The goal is maximum reuse while maintaining a working pipeline.** If reusable workflows create more complexity than they solve for lsp-indexer's specific needs, it's valid to use composite actions for the common setup steps and keep jobs inline.

    **Step 2: Create SHARED-INFRA-DECISION.md** with:
    - **Decision:** Shared workflows live in `chillwhales/.github` with both composite actions and reusable workflows
    - **Architecture:** Three-layer structure (build-lint-test, quality, publish-validation) + two composite actions (setup-pnpm, build-and-upload)
    - **What was implemented:** List exactly what was created and consumed
    - **What works well:** Composite actions for setup boilerplate, reusable workflows for standard pipelines
    - **Limitations encountered:** Any issues with artifact sharing, secrets, or customization constraints
    - **Migration path for LSPs:** How LSPs would adopt the shared workflows (which parts apply, which need customization)
    - **Consumer pattern:** Example of how a new repo would adopt the shared workflows
    - **Maintenance considerations:** Version pinning strategy (@main vs @v1 tags), testing changes to shared workflows
    - **Rationale:** Why this approach (vs copy-paste, vs monorepo CI tool like nx/turborepo)

    The document should be honest about tradeoffs. If reusable workflows add friction for lsp-indexer's specific needs, say so. The value is the investigation + working proof of concept, not forced adoption.
