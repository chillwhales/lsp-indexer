<task type="auto">
  <name>Task 1: Create release workflow with changesets + Docker image publishing</name>
  <files>.github/workflows/release.yml</files>
  <action>
    **Create `.github/workflows/release.yml`** following the RESEARCH.md Pattern 4 with these exact specifications:

    **Trigger:** `on: push: branches: [main]`

    **Concurrency:** `group: ${{ github.workflow }}-${{ github.ref }}` with `cancel-in-progress: false` — CRITICAL: never cancel a release mid-flight.

    **Job: `release`**
    - `runs-on: ubuntu-latest`
    - Permissions: `contents: write`, `pull-requests: write`, `packages: write` (GHCR needs packages:write), `id-token: write` (npm provenance — requires `--provenance` flag on publish)
    - Steps:
      1. `actions/checkout@v4`
      2. `pnpm/action-setup@v4`
      3. `actions/setup-node@v4` with node-version: 22, cache: pnpm, registry-url: `https://registry.npmjs.org` (needed for npm publish auth)
      4. `pnpm install --frozen-lockfile`
      5. `pnpm build`
      6. **Set npm provenance config** (required for `id-token: write` to work):
         ```yaml
         - name: Configure npm provenance
           run: npm config set provenance true
         ```
         This ensures all packages published via `changeset publish` include npm provenance attestation. Setting it via npm config avoids needing to modify the changesets publish command.
      7. **Changesets action** (`changesets/action@v1`):
         - `id: changesets`
         - `commit: "chore: version packages"`
         - `title: "chore: version packages"`
         - `publish: pnpm ci:publish`
         - `version: pnpm changeset version`
         - env: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`, `NPM_TOKEN: ${{ secrets.NPM_TOKEN }}`
      8. **Extract version tag** (conditional: `if: steps.changesets.outputs.published == 'true'`):
         - `id: version`
         - Parse version from `publishedPackages` JSON: `VERSION=$(echo '${{ steps.changesets.outputs.publishedPackages }}' | jq -r '.[0].version')` then `echo "tag=v${VERSION}" >> $GITHUB_OUTPUT`
      9. **Login to GHCR** (conditional: `if: steps.changesets.outputs.published == 'true'`):
         - `docker/login-action@v3`
         - registry: `ghcr.io`, username: `${{ github.actor }}`, password: `${{ secrets.GITHUB_TOKEN }}`
      10. **Build and push Docker image** (conditional: `if: steps.changesets.outputs.published == 'true'`):
         - `docker/build-push-action@v6`
         - context: `.`
         - file: `docker/Dockerfile`
         - push: `true`
         - tags: `ghcr.io/chillwhales/lsp-indexer:latest` and `ghcr.io/chillwhales/lsp-indexer:${{ steps.version.outputs.tag }}`
         - labels: `org.opencontainers.image.source=https://github.com/chillwhales/lsp-indexer`, `org.opencontainers.image.description=LUKSO blockchain indexer`, `org.opencontainers.image.licenses=MIT`

    **Key behaviors:**
    - When there are unreleased changesets on main → creates a "Version Packages" PR
    - When the "Version Packages" PR is merged → publishes to npm and builds/pushes Docker image
    - Docker image only built when packages are actually published (not on every push to main)
    - All 4 packages in the fixed group share the same version number, so `.[0].version` gives the correct tag for all
  </action>

  </verify>
  <done>
    - `.github/workflows/release.yml` triggers on push to main

    - Docker image built and pushed to GHCR only when packages published

  </done>

<task type="auto">
  <name>Task 2: Create preview release workflow with pkg-pr-new</name>
  <files>.github/workflows/preview.yml</files>
  <action>
    **Create `.github/workflows/preview.yml`** following the RESEARCH.md Pattern 5:

    **Name:** `Preview Release`

    **Trigger:** `on: pull_request: branches: [main]`
    Note: Only PRs targeting main (not refactor/indexer-v2-react) — previews are for pre-merge testing of packages that will be released from main.

    **Job: `preview`**
    - `if: github.event.pull_request.head.repo.full_name == github.repository` — SECURITY: fork PRs are excluded (they could contain malicious publish scripts)
    - `runs-on: ubuntu-latest`
    - Permissions: `pull-requests: write` (for PR comment), `contents: read`
    - Steps:
      1. `actions/checkout@v4`
      2. `pnpm/action-setup@v4`
      3. `actions/setup-node@v4` with node-version: 22, cache: pnpm
      4. `pnpm install --frozen-lockfile`
      5. `pnpm build`
      6. **Publish previews:**
         ```yaml
         - name: Publish preview packages
           run: pnpx pkg-pr-new publish './packages/types' './packages/node' './packages/react' './packages/next' --compact --comment=update --packageManager=pnpm
           env:
             GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
         ```

    **IMPORTANT:**
    - List only the 4 publishable packages explicitly — do NOT use `'./packages/*'` glob (would include indexer, abi, typeorm, comparison-tool)
    - `--compact` flag produces condensed PR comment
    - `--comment=update` flag updates existing comment instead of creating new ones
    - `--packageManager=pnpm` ensures correct registry interaction
    - GITHUB_TOKEN is needed for PR comment creation
    - No `NPM_TOKEN` needed — pkg-pr-new uses its own infrastructure, not the npm registry
  </action>
  <verify>
    <automated>grep "pkg-pr-new" .github/workflows/preview.yml && grep "head.repo.full_name" .github/workflows/preview.yml && grep -c "packages/types" .github/workflows/preview.yml && echo "Preview workflow structure verified"</automated>
  </verify>
  <done>
    - `.github/workflows/preview.yml` triggers on PRs to main
    - Fork PRs excluded via repository check
    - Only 4 publishable packages are published as previews
    - Uses pkg-pr-new with --compact --comment=update flags
    - PR comment shows installable preview package URLs
  </done>
</task>
