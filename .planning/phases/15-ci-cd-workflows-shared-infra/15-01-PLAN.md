<task type="auto">
  <name>Task 2: Expand CI workflow to full layered pipeline with coverage reporting</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    **Rewrite `.github/workflows/ci.yml`** to implement the full 4-layer pattern from RESEARCH.md. The existing workflow has 3 independent jobs (format, lint, build). Replace with a layered dependency graph:

    **Triggers** (keep existing):
    - push: `[main, refactor/indexer-v2-react]`
    - pull_request: `[main, refactor/indexer-v2-react]`
    - Concurrency: `cancel-in-progress: true` (safe for CI, NOT for releases)

    **Layer 1 — Install** (`install` job):
    - Checkout, pnpm/action-setup@v4, setup-node@v4 with node 22 + pnpm cache
    - `pnpm install --frozen-lockfile`
    - This job exists to warm the pnpm cache. All downstream jobs still run their own install (resolved from cache in ~5s).

    **Layer 1.5 — Changeset Check** (`changeset-check` job):
    - `if: github.event_name == 'pull_request'`
    - `needs: [install]`
    - Checkout with `fetch-depth: 0` (full history needed for changeset comparison)
    - `pnpm changeset status --since=origin/main`
    - This enforces that PRs touching publishable packages include a changeset file.

    **Layer 2 — Parallel validation + build** (3 jobs, all `needs: [install]`):
    - `format`: `pnpm format:check`
    - `lint`: Build dependency packages first (`pnpm --filter=@chillwhales/abi build && pnpm --filter=@chillwhales/typeorm build`), then `pnpm lint` (matching existing pattern — ESLint needs type declarations)
    - `build`: `pnpm build`, then upload `packages/*/dist` as artifact using `actions/upload-artifact@v4` (name: `build-output`)

    **Layer 2.5 — Typecheck** (`typecheck` job):
    - `needs: [build]`
    - Download build artifact, then `pnpm typecheck` (cross-package .d.ts files must exist)

    **Layer 3 — Test + Package verify** (2 jobs, both `needs: [build]`):
    - `test`: Node matrix [20, 22]. Download build artifact. Run `pnpm test:coverage`. On Node 22 only: upload `packages/*/coverage/` as artifact (name: `coverage-reports`). Use `continue-on-error: true` for coverage thresholds initially (infrastructure phase — tests are minimal).
    - `pkg-verify`: Download build artifact. Run `pnpm validate:publish` (publint + attw on all 4 packages).

    **Layer 4 — Coverage report** (`coverage` job):
    - `needs: [test]`
    - `if: github.event_name == 'pull_request'`
    - Download coverage artifact
    - Use `davelosert/vitest-coverage-report-action@v2` with:
      - `json-summary-path` and `json-final-path` pointing to coverage output
      - Posts PR comment with coverage summary table per package
    - Note: The vitest-coverage-report-action reads json-summary from coverage directories. Configure with `working-directory` or explicit paths per package. If the action doesn't natively support multi-package, run it once per package or use a matrix.

    **Artifact handling pattern:**
    - Build job uploads: `packages/types/dist`, `packages/node/dist`, `packages/react/dist`, `packages/next/dist` (also include `packages/abi/lib`, `packages/typeorm/lib` since they're needed)
    - Use path patterns: `packages/*/dist/**` and `packages/abi/lib/**` and `packages/typeorm/lib/**`
    - Downstream jobs download artifact, restoring exact paths

    **IMPORTANT:** Do NOT use `actions/cache` for build artifacts — use `actions/upload-artifact@v4` / `actions/download-artifact@v4`. Cache is for dependencies (pnpm handles this via setup-node cache). Artifacts are for build output shared between jobs.

    **Keep concurrency group** with `cancel-in-progress: true` (CI runs are safe to cancel).
  </action>
