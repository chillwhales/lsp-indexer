# Phase 9.3 Plan 01: Owned Asset & Owned Token Types + Documents Summary

**OwnedAsset/OwnedToken Zod schemas with bigint balance and 4 GraphQL documents with nested @include directives for DA sub-fields, NFT metadata, and universal profiles.**

## Performance

- Duration: ~5 minutes
- Codegen: Clean pass — all 4 new documents generated types correctly
- Typecheck: Both `@lsp-indexer/types` and `@lsp-indexer/node` pass
- Build: `@lsp-indexer/types` builds cleanly (24.81 KB ESM, 30.75 KB CJS)

## Accomplishments

1. **Created OwnedAsset domain types** (`packages/types/src/owned-assets.ts`) — OwnedAssetSchema with `balance: z.bigint()`, OwnedAssetFilterSchema (4 fields), OwnedAssetSortFieldSchema (7 fields including nested sorts), OwnedAssetIncludeSchema with DigitalAssetIncludeSchema for nested DA sub-includes, and 3 hook parameter schemas.

2. **Created OwnedToken domain types** (`packages/types/src/owned-tokens.ts`) — OwnedTokenSchema with 4 nullable relation schemas (digitalAsset, nft, ownedAsset, universalProfile), OwnedTokenFilterSchema (7 fields), OwnedTokenSortFieldSchema (5 fields), OwnedTokenIncludeSchema with DA sub-includes, and 3 hook parameter schemas.

3. **Created 4 GraphQL documents** — GetOwnedAssetDocument, GetOwnedAssetsDocument, GetOwnedTokenDocument, GetOwnedTokensDocument. All use `@include(if:)` directives with `Boolean! = true` defaults. OwnedAsset documents include 17 DA sub-include variables + universalProfile + tokenIds_aggregate. OwnedToken documents include DA sub-includes + NFT (with lsp4Metadata + lsp4MetadataBaseUri) + ownedAsset + universalProfile.

4. **Codegen output updated** — `graphql.ts` and `gql.ts` contain GetOwnedAssetQuery, GetOwnedAssetsQuery, GetOwnedTokenQuery, GetOwnedTokensQuery types plus Owned_Asset_Bool_Exp and Owned_Token_Bool_Exp Hasura types.

5. **Barrel exports updated** — `packages/types/src/index.ts` re-exports both `./owned-assets` and `./owned-tokens`.

## Decisions Made

1. **OwnedAsset.balance = z.bigint()** — Per locked decision. Parser will convert Hasura numeric string to BigInt. Consumer uses formatTokenAmount helpers.

2. **OwnedAssetFilter: 4 string fields only** — owner, address, digitalAssetId, universalProfileId. Balance and timestamp are comparison-based, not string-ilike. Range filters deferred to future.

3. **OwnedAssetSortField includes nested sorts** — `digitalAssetName` maps to `digitalAsset.lsp4TokenName` and `tokenIdCount` maps to `tokenIds_aggregate.count` at the service layer.

4. **Universal profile fragment: all LSP3 fields, no aggregates** — Nested universalProfile includes name, description, tags, links, avatar, profileImage, backgroundImage but NOT followerCount/followingCount (those are aggregate-heavy and not needed for ownership context).

5. **OwnedToken NFT relation: full metadata** — Includes lsp4Metadata + lsp4MetadataBaseUri for the established fallback pattern (direct first → baseUri second → null).
