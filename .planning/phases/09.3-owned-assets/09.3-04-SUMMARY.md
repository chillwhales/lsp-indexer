## Decisions Made

1. **Natural key lookup instead of opaque IDs** — Single lookup tab uses holder address + asset address as inputs, queried via `useOwnedAssets({ filter, limit: 1 })` instead of `useOwnedAsset(id)`. More developer-friendly for testing.

2. **Preload prevention via limit:0** — When no query is present, pass `limit: 0` to prevent hooks from firing and returning random data.

3. **Full addresses, no truncation** — User preference: display full hex addresses with `break-all` CSS wrapping rather than truncating with ellipsis.

4. **Unified ghost Button collapsible triggers** — All collapsible sections across all cards use `<Button variant="ghost" size="sm" className="gap-1.5 text-muted-foreground">` with domain-specific icon + text + ChevronDown.

5. **NFT holder UP as full block** — Unlike NFT metadata fields (per-field `@include`), the holder's Universal Profile is fetched as a complete block when holder is included. Simpler and matches the ownership context use case.

6. **parseHolderProfile() inline type derivation** — The raw type from codegen (`RawNft['ownedToken']['universalProfile']`) differs from `RawProfile` (derived from `GetProfileQuery`), so a separate inline parser was created rather than reusing `parseProfile` directly.

7. **NftCard Collection section moved to last** — User preference: show NFT Metadata before Collection in the NftCard layout.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] BigInt serialization crash in RawJsonToggle**

- **Found during:** Task 1 initial testing
- **Issue:** `JSON.stringify` throws on BigInt values; OwnedAsset.balance is BigInt
- **Fix:** Added `bigintReplacer` function that converts BigInt to string representation
- **Files modified:** `apps/test/src/components/playground/shared.tsx`
- **Committed in:** `4b9dd31`

**2. [Rule 1 - Bug] Preload firing on empty filter**

- **Found during:** Task 1 refinement
- **Issue:** `useOwnedAssets({ filter: undefined, limit: 1 })` fires immediately and returns random data
- **Fix:** Pass `limit: hasQuery ? 1 : 0` to prevent premature fetching
- **Files modified:** Both page files
- **Committed in:** `630091b`

**3. [Rule 2 - Missing Critical] UP sub-includes pipeline for owned assets**

- **Found during:** Task 1 testing
- **Issue:** Plan mentioned UP include toggles but the GraphQL documents/services lacked `$includeProfile*` variables
- **Fix:** Added 9 profile sub-include variables to owned-assets documents, created `buildProfileIncludeVars()` in profiles service, wired into owned-assets and owned-tokens `buildIncludeVars`
- **Files modified:** Documents, services, index.ts for both domains
- **Committed in:** `630091b`

**4. [Rule 2 - Missing Critical] NFT sub-includes for owned-tokens**

- **Found during:** User review — owned-tokens needed per-field NFT metadata toggles

- **Files modified:** Types, documents, codegen, services, page
- **Committed in:** `834fd88`

**5. [Rule 2 - Missing Critical] NFT holder Universal Profile pipeline**
