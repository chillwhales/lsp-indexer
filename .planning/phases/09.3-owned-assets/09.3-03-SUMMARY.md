# Phase 9.3 Plan 03: React Hooks + Server Actions + Next.js Hooks Summary

TanStack Query hooks for OwnedAsset/OwnedToken domains in @lsp-indexer/react and @lsp-indexer/next, plus server actions routing browser→server→Hasura.

## Performance

- **Duration:** 3m 16s
- **Tasks:** 2/2 completed
- **Builds:** All 4 packages (types, node, react, next) pass with zero errors
- **Typechecks:** Both react and next packages pass `tsc --noEmit`

## Accomplishments

1. **6 React hooks** (3 per domain) created in `@lsp-indexer/react`:

   - `useOwnedAsset` — single owned asset by ID, disabled when ID falsy
   - `useOwnedAssets` — paginated list with filter/sort/pagination/include
   - `useInfiniteOwnedAssets` — infinite scroll with offset-based pagination
   - `useOwnedToken` — single owned token by ID, disabled when ID falsy
   - `useOwnedTokens` — paginated list with filter/sort/pagination/include
   - `useInfiniteOwnedTokens` — infinite scroll with offset-based pagination

2. **4 server actions** (2 per domain) created in `@lsp-indexer/next`:

   - `getOwnedAsset(id, include?)` — server-side single fetch
   - `getOwnedAssets(params?)` — server-side paginated list fetch
   - `getOwnedToken(id, include?)` — server-side single fetch
   - `getOwnedTokens(params?)` — server-side paginated list fetch

3. **6 Next.js hooks** (3 per domain) routing through server actions:

   - Same API surface as React hooks, but `queryFn` calls server actions instead of direct Hasura
   - GraphQL endpoint stays hidden from the browser

4. **Entry points wired** — `export *` barrel pattern in both `react/index.ts` and `next/index.ts`

5. **All 4 packages build clean** — types, node, react, next with zero errors
