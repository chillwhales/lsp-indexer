**Create `packages/node/src/parsers/owned-assets.ts`** — Hasura → OwnedAsset transformation:

The parser must:

1. Map `address`, `owner`, `id`, `block`, `timestamp` directly (camelCase already)
2. **Convert `balance` from Hasura numeric (string) to `bigint`:** Use `BigInt(raw.balance)` — Hasura returns numeric as string which BigInt() handles correctly
3. Parse nested `digitalAsset` using `parseDigitalAsset` from `./digital-assets` (same pattern as NFT collection)
4. Parse nested `universalProfile` using `parseProfile` from `./profiles`
5. Extract `tokenIdCount` from `tokenIds_aggregate?.aggregate?.count ?? null`
6. Return `null` for nested fields when they are undefined (not included in query)

```typescript
import type { OwnedAsset } from '@lsp-indexer/types';
import { parseDigitalAsset } from './digital-assets';
import { parseProfile } from './profiles';

export function parseOwnedAsset(raw: any): OwnedAsset {
  return {
    id: raw.id,
    address: raw.address,
    owner: raw.owner,
    balance: BigInt(raw.balance),
    block: raw.block,
    timestamp: raw.timestamp,
    digitalAsset: raw.digitalAsset ? parseDigitalAsset(raw.digitalAsset) : null,
    universalProfile: raw.universalProfile ? parseProfile(raw.universalProfile) : null,
    tokenIdCount: raw.tokenIds_aggregate?.aggregate?.count ?? null,
  };
}

export function parseOwnedAssets(raw: any[]): OwnedAsset[] {
  return raw.map(parseOwnedAsset);
}
```

**Create `packages/node/src/services/owned-assets.ts`** — service functions following the pattern in `services/digital-assets.ts`:

The service must:

1. `fetchOwnedAsset(url, params)` — fetches single owned asset by ID (where: { id: { \_eq: id } })
2. `fetchOwnedAssets(url, params)` — fetches list with filter/sort/include translation

**buildWhere(filter):** Translate OwnedAssetFilter to Hasura `owned_asset_bool_exp`:

- `filter.owner` → `{ owner: { _ilike: \`%${escapeLike(owner)}%\` } }`
- `filter.address` → `{ address: { _ilike: \`%${escapeLike(address)}%\` } }`
- `filter.digitalAssetId` → `{ digital_asset_id: { _ilike: \`%${escapeLike(digitalAssetId)}%\` } }`
- `filter.universalProfileId` → `{ universal_profile_id: { _ilike: \`%${escapeLike(universalProfileId)}%\` } }`All use`\_ilike`+`escapeLike` per established patterns.

**buildOrderBy(sort):** Translate OwnedAssetSort to Hasura `owned_asset_order_by`:

- Direct columns: `balance`, `timestamp`, `address`, `owner`, `block` → `{ [field]: orderDir(direction, nulls) }`
- Nested `digitalAssetName` → `{ digitalAsset: { lsp4TokenName: { value: orderDir(direction, nulls) } } }`
- Nested `tokenIdCount` → `{ tokenIds_aggregate: { count: orderDir(direction, nulls) } }`
  Use `orderDir()` helper from `./utils`.

**buildIncludeVars(include):** Translate OwnedAssetInclude to Boolean include variables:

- When `include` is undefined → all variables `true` (inverted default)
- When `include` is provided → only explicitly set fields are `true`, rest `false`
- When `include.digitalAsset` is provided → call `buildDigitalAssetIncludeVars(include.digitalAsset)` from `./digital-assets` to get the 17 DA sub-variables
- `includeUniversalProfile` and `includeTokenIdCount` derived from include object

Return type: `FetchOwnedAssetsResult = { ownedAssets: OwnedAsset[]; totalCount: number; }`

Use `execute` from `../client` for GraphQL execution, `parseOwnedAsset`/`parseOwnedAssets` from `../parsers/owned-assets`.
