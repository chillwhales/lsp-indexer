**Healthcheck note:** Existing compose services (Hasura, data-connector) use `curl -f` for healthchecks. The monitoring images (Grafana, Loki, Prometheus, Alloy) are Alpine-based and ship `wget` but not `curl`. Use `wget --quiet --tries=1 --output-document=-` for monitoring service healthchecks. cAdvisor ships neither — use `wget` via busybox (included in cAdvisor image).

1. **loki** service:
   - Image: `grafana/loki:3.4` (or latest stable 3.x)
   - container_name: lsp-indexer-loki
   - restart: unless-stopped
   - Volumes: `loki-data:/loki`, `./loki/config.yaml:/etc/loki/config.yaml:ro`
   - Command: `-config.file=/etc/loki/config.yaml`
   - Healthcheck: `['CMD-SHELL', 'wget --quiet --tries=1 --output-document=- http://localhost:3100/ready || exit 1']`, interval 10s, timeout 5s, retries 5, start_period 10s
   - Memory limit: 512M, reservation: 128M
