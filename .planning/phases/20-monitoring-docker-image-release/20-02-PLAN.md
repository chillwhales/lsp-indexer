  <files>docker/grafana/provisioning/dashboards/indexer-monitoring.json</files>
  <action>
Create a comprehensive Grafana dashboard JSON file. Use datasource UIDs `loki` and `prometheus` (matching Plan 01 provisioning). Use Grafana dashboard JSON model format (not the API save model — no `id`, no `version`).

**Dashboard settings:**
- title: "LSP Indexer Monitoring"
- uid: "lsp-indexer-monitoring"
- timezone: "browser"
- refresh: "30s"
- time.from: "now-1h", time.to: "now"
- editable: true
- tags: ["indexer", "monitoring", "lsp"]
- schemaVersion: 39 (or current)

**Template variable:**
- Name: `service`, label: "Service", type: query
- Query: `label_values(compose_service)` against Loki datasource
- Multi-select enabled, include "All" option
- Default: "$__all"

**Row 1: Overview** (collapsed: false)
Four stat panels in a single row:

- **Current Block Height** — Stat panel. LogQL: `max_over_time({compose_service="indexer"} | json | unwrap blockNumber [1h])`. Use `last` reduction on the stat panel. Unit: none. Title: "Current Block". Datasource: loki.

- **Blocks/Minute** — Stat panel. LogQL: rate of distinct blockNumber values from indexer logs over 5m window. Title: "Blocks/min". Datasource: loki.

- **Sync Lag** — Stat panel. If indexer logs contain a "sync lag" or similar field, extract it. Otherwise, show time since last block was processed using `{compose_service="indexer"} | json | blockNumber > 0` and computing staleness. Title: "Last Block Age". Datasource: loki.

- **Error Count (1h)** — Stat panel. LogQL: `count_over_time({compose_service=~"$service"} | json | level >= 50 [1h])`. Red threshold if > 0. Title: "Errors (1h)". Datasource: loki.

**Row 2: Error Analysis** (collapsed: false)
Two time series panels:

- **Error Rate Over Time** — Time series. LogQL: `sum by (compose_service) (count_over_time({compose_service=~"$service"} | json | level >= 50 [$__interval]))`. Legend: {{compose_service}}. Title: "Error Rate by Service". Datasource: loki.

- **Errors by Pipeline Step** — Time series. LogQL: `sum by (step) (count_over_time({compose_service="indexer"} | json | level >= 50 | step != "" [$__interval]))`. Legend: {{step}}. Title: "Indexer Errors by Pipeline Step". Datasource: loki. Note: `step` field contains pipeline step names (EXTRACT, PERSIST_RAW, HANDLE, PERSIST_DERIVED, VERIFY, ENRICH).

**Row 3: Block Progress** (collapsed: false)
Two time series panels:

- **Block Processing Over Time** — Time series. LogQL extracting blockNumber from indexer logs: `{compose_service="indexer"} | json | unwrap blockNumber [$__interval]` with max_over_time. Title: "Block Height Over Time". Datasource: loki.

- **Processing Rate** — Time series. LogQL computing rate of log entries from indexer service: `sum(rate({compose_service="indexer"} | json | blockNumber > 0 [$__interval]))`. Title: "Log Entries/sec (Indexer)". Datasource: loki.

**Row 4: Log Explorer** (collapsed: true — expand to view)
One logs panel:

- **All Service Logs** — Logs panel. LogQL: `{compose_service=~"$service"}`. Show all fields. Sort: newest first. Default wrap: true. Title: "Service Logs". Datasource: loki. Enable filtering by service via the `$service` template variable.

**Row 5: Container Metrics** (collapsed: true — expand to view)
Four time series panels:

- **CPU Usage** — PromQL: `rate(container_cpu_usage_seconds_total{name=~"lsp-indexer.*"}[5m]) * 100`. Legend: {{name}}. Unit: percent. Title: "CPU Usage by Container". Datasource: prometheus.

- **Memory Usage** — PromQL: `container_memory_usage_bytes{name=~"lsp-indexer.*"}`. Legend: {{name}}. Unit: bytes(IEC). Title: "Memory Usage by Container". Datasource: prometheus.

- **Network I/O** — PromQL with two queries: `rate(container_network_receive_bytes_total{name=~"lsp-indexer.*"}[5m])` (legend: "{{name}} rx") and `rate(container_network_transmit_bytes_total{name=~"lsp-indexer.*"}[5m])` (legend: "{{name}} tx"). Unit: bytes/sec. Title: "Network I/O". Datasource: prometheus.

- **Disk I/O** — PromQL with two queries: `rate(container_fs_reads_bytes_total{name=~"lsp-indexer.*"}[5m])` (legend: "{{name}} read") and `rate(container_fs_writes_bytes_total{name=~"lsp-indexer.*"}[5m])` (legend: "{{name}} write"). Unit: bytes/sec. Title: "Disk I/O". Datasource: prometheus.

**Alert rules (embedded in panels, visual only — no notification channel):**
- On "Error Rate by Service" panel: threshold annotation at y=10 (red line), alert condition if sum of errors > 10 in 5m window. Visual indicator only.
- On "Block Height Over Time" panel: "no data" alert if no new data points in 10 minutes. Visual indicator only.

**Panel layout:** Use gridPos to arrange panels. Overview row: 4 stat panels at y=0, each w=6 h=4. Error row at y=5. Block row at y=13. Log row at y=21. Metrics row at y=22. Adjust heights so the dashboard reads top-to-bottom logically.

**IMPORTANT:** All datasource references must use the `uid` format: `{"type": "loki", "uid": "loki"}` for Loki panels, `{"type": "prometheus", "uid": "prometheus"}` for Prometheus panels. Do NOT use datasource names.
