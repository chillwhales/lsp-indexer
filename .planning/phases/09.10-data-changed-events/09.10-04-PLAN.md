```
│ ▸ Raw JSON [collapsible]                     │
└─────────────────────────────────────────────┘
```

**Data key display (per CONTEXT.md — Claude's discretion for layout):**

If `dataKeyName` is present (not null), show:

- **Bold label:** the resolved name (e.g., "LSP3Profile", "LSP5ReceivedAssets[]")
- **Below in monospace:** the raw hex truncated to ~20 chars with `break-all`

If `dataKeyName` is null (unknown key):

- **Mono text:** the raw hex key (truncated)
- **Label:** "(Unknown Key)" in muted text

**Data value display:**

- For values that are hex-encoded addresses (20-byte or 32-byte): show the full hex value in monospace with `break-all` and **no truncation**
- For all other raw hex data values: show in monospace with `break-all`, truncated to the first ~80 chars (including `0x` prefix) followed by `…` to indicate truncation

**Collapsible sections:**

- **Universal Profile** — use `CollapsibleProfileSection` component from `collapsible-sections.tsx` if it exists, or create a simple collapsible using `<Button variant="ghost">` + `<Collapsible>` from shadcn/ui (matching existing card patterns with unified ghost Button triggers)
- **Digital Asset** — collapsible showing DA name, symbol, token type, etc. Use same field-presence rendering as DigitalAssetCard

**Imports needed:**

- `Button` from `@/components/ui/button`
- `Collapsible`, `CollapsibleContent`, `CollapsibleTrigger` from `@/components/ui/collapsible`
- `ChevronDown` from `lucide-react`
- Existing collapsible section components from `@/components/collapsible-sections` (check what's available)

**NOTE:** Use `'use client'` directive at top of file.
