## Decisions Made

- **dataKeyName as parser-derived base field:** Always present (null when hex key is unknown), resolved by parser at runtime — NOT fetched from Hasura
- **TokenIdDataChangedEventNftSchema is lightweight:** Only 5 fields (address, tokenId, isBurned, isMinted, name) — avoids importing the full Nft domain type and keeps event queries fast
- **NFT include is boolean-only:** `nft: z.boolean().optional()` — no sub-include object needed since the NFT sub-selection is fixed (5 fields always fetched together)
- **universalProfile sub-fields use `$includeUniversalProfile*` prefix:** Matches encrypted-assets and issued-assets patterns
- **`followed_aggregate` for UP following count:** Consistent with all other domains
