**Create `packages/types/src/data-changed-events.ts`** — follows issued-assets.ts pattern closely since both have dual heterogeneous relations (UP + DA). The key difference: data change events add a derived `dataKeyName` field (resolved by parser, not from Hasura).

**Hasura schema reference for `data_changed` (schema.graphql lines 256–273):**

```
address: String!              → address (emitting contract, base field)
block_number: Int!            → blockNumber (includable scalar)
data_key: String!             → dataKey (raw hex, base field)
data_value: String!           → dataValue (raw hex, base field)
digitalAsset: digital_asset   → digitalAsset (DigitalAssetInclude sub-narrowing)
digital_asset_id: String      → NOT exposed (internal FK)
id: String!                   → NOT exposed (opaque Hasura ID)
log_index: Int!               → logIndex (includable scalar)
timestamp: timestamptz!       → timestamp (includable scalar)
transaction_index: Int!       → transactionIndex (includable scalar)
universalProfile: universal_profile → universalProfile (ProfileInclude sub-narrowing)
universal_profile_id: String  → NOT exposed (internal FK)
```

**Parser-derived field (NOT from Hasura):**

- `dataKeyName: string | null` — resolved name for known ERC725Y keys (e.g., `"LSP3Profile"`, `"LSP5ReceivedAssets[]"`). **Always present** as a base field; null when the hex key is unknown.
