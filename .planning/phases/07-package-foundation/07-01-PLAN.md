    **src/errors/indexer-error.ts** — Create the full IndexerError class following the taxonomy from 07-RESEARCH.md:

    Types to define:
    - `IndexerErrorCategory`: union of `'NETWORK' | 'HTTP' | 'GRAPHQL' | 'CONFIGURATION' | 'PARSE'`
    - `IndexerErrorCode`: union of 17 codes grouped by category:
      - Network: `NETWORK_TIMEOUT`, `NETWORK_UNREACHABLE`, `NETWORK_ABORTED`, `NETWORK_UNKNOWN`
      - HTTP: `HTTP_UNAUTHORIZED`, `HTTP_FORBIDDEN`, `HTTP_NOT_FOUND`, `HTTP_TOO_MANY_REQUESTS`, `HTTP_SERVER_ERROR`, `HTTP_UNKNOWN`
      - GraphQL: `GRAPHQL_VALIDATION`, `GRAPHQL_EXECUTION`, `PERMISSION_DENIED`, `GRAPHQL_UNKNOWN`
      - Configuration: `MISSING_ENV_VAR`, `INVALID_URL`
      - Parse: `RESPONSE_NOT_JSON`, `EMPTY_RESPONSE`
    - `IndexerErrorOptions`: interface with `category`, `code`, `message`, optional `statusCode`, `originalError`, `query`, `graphqlErrors`

    Class `IndexerError extends Error`:
    - Constructor accepts `IndexerErrorOptions`, sets `this.name = 'IndexerError'`
    - All fields as `readonly` properties: `category`, `code`, `statusCode`, `originalError`, `query`, `graphqlErrors`
    - `toJSON()` method returning serializable record (name, category, code, message, statusCode, query, graphqlErrors — no originalError since it's not serializable)
    - Static factory `fromHttpResponse(response: Response)`: maps status codes (401→HTTP_UNAUTHORIZED, 403→HTTP_FORBIDDEN, 404→HTTP_NOT_FOUND, 429→HTTP_TOO_MANY_REQUESTS, 5xx→HTTP_SERVER_ERROR, other→HTTP_UNKNOWN). Include recovery hint strings per code.
    - Static factory `fromGraphQLErrors(errors, query?)`: detects Hasura permission errors (`extensions.code === 'access-denied'`, message includes 'not allowed'/'permission'), validation errors (`extensions.code === 'validation-failed'`), defaults to GRAPHQL_UNKNOWN. Include recovery hints.
    - Static factory `fromNetworkError(error: Error)`: detects AbortError/timeout→NETWORK_TIMEOUT, else NETWORK_UNKNOWN. Include recovery hint about checking network/endpoint.

    Follow the exact implementation patterns shown in 07-RESEARCH.md sections "IndexerError Taxonomy" and "Pattern 2". Use JSDoc on the class and each public method.

    - All error messages must include the env var name and a helpful example (e.g., "Set NEXT_PUBLIC_INDEXER_URL to your Hasura endpoint, e.g., https://indexer.example.com/v1/graphql")

    - `execute<TResult, TVariables>(url: string, document: TypedDocumentString<TResult, TVariables>, ...[variables])` — follows the exact pattern from 07-RESEARCH.md "Pattern 2":
      - POST to url with Content-Type: application/json, Accept: application/graphql-response+json

      - If `!json.data`: throw `IndexerError` with EMPTY_RESPONSE
      - Return `json.data`
