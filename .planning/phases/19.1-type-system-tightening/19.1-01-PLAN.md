**metadataFetch.ts** (4 errors — lines 177, 181, 187, 210):
The function `handleMetadataFetch<TEntity extends MetadataEntity>()` calls `getEntities<TEntity>()`. Fix line ~177 by removing the generic and adding a cast to the function's generic type:
```typescript
// Before:
const entities = hctx.batchCtx.getEntities<TEntity>(triggeredBy);
// After:
const entities = hctx.batchCtx.getEntities(triggeredBy) as Map<string, TEntity>;
```
This single change resolves all 4 errors because `entities.values()` then yields `TEntity` values, satisfying `config.getUrl(entity)` and other calls that expect `TEntity`.

**pipeline.test.ts** (21 errors):
This test creates synthetic entities with extra properties (e.g., `{ id: 'e1', blockNumber: 1, transactionIndex: 0, logIndex: 0, data: '0x' }`) and passes them to `addEntity()`. Since `addEntity()` now requires `Entity`, TypeScript rejects excess properties on object literals.

Fix approach: For each `addEntity()` call with extra properties, extract the object into a typed variable first (breaks excess property checking), then pass it:
```typescript
// Before (inline object literal — excess property check):
ctx.addEntity('Event', 'e1', { id: 'e1', blockNumber: 1, transactionIndex: 0, logIndex: 0, data: '0x' });

// After (variable — no excess property check on assignment to wider type):
const e1 = { id: 'e1', blockNumber: 1, transactionIndex: 0, logIndex: 0, data: '0x' };
ctx.addEntity('Event', e1.id, e1 as Entity);
```

For `getEntities<T>()` calls in test handlers (lines ~431, ~1034-1035), apply the same cast pattern as handlers:
```typescript
// Before:
const events = hctx.batchCtx.getEntities<Entity & { value: number }>(triggeredBy);
// After:
const events = hctx.batchCtx.getEntities(triggeredBy) as Map<string, Entity & { value: number }>;
```

For property access on Entity (e.g., `event.to`, `event.amount`), cast the value when accessing:
```typescript
// Before:
const transfer = events.get('t1')!;
const owner = transfer.to;
// After:
const transfer = events.get('t1') as Entity & { to: string; amount: bigint };
const owner = transfer.to;
```

The key principle: test entities are synthetic — use `as Entity` on variables passed to `addEntity()`, and type-widen when reading properties back. This matches how real handlers will work.
