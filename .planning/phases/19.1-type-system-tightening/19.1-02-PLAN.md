<objective>
Create a full entity registry that maps every BatchContext bag key to its TypeORM entity
class ‚Äî both at the type level and at runtime. Make `addEntity()` and `getEntities()`
fully type-safe: compile-time enforcement that the entity matches the bag key, plus
runtime `instanceof` validation. Delete `entityTypeMap.ts` and `getTypedEntities()` ‚Äî
`getEntities()` itself returns typed entities directly.

This eliminates all `as Map<string, T>` casts from handlers and helpers, prevents bag
key typos at compile time, and catches entity type mismatches at runtime immediately
instead of letting bad data propagate.
</objective>

## Architecture

### EntityRegistry (new file: `packages/indexer/src/core/entityRegistry.ts`)

Single source of truth for all bag keys in the system. Two exports:

1. **`EntityRegistry` interface** ‚Äî type-level mapping of every bag key to its
   concrete TypeORM entity class. 71 entries grouped by category (event entities,
   data key handlers, metadata sub-entities, LSP6 sub-entities, etc.)

2. **`ENTITY_CONSTRUCTORS` const** ‚Äî runtime mapping of the same 71 keys to
   constructor functions. Typed as `{ [K in keyof EntityRegistry]: new (...args) => EntityRegistry[K] }`
   so the compiler enforces it stays in sync with the interface.

3. **`getEntityConstructor<K>(key: K)` helper** ‚Äî returns the constructor for a
   bag key from the runtime map. Used by `handlerHelpers.ts` for DB queries.

### IBatchContext changes

```typescript
// Typed methods ‚Äî used by plugins, handlers, and handlerHelpers
addEntity<K extends keyof EntityRegistry>(type: K, id: string, entity: EntityRegistry[K]): void;
getEntities<K extends keyof EntityRegistry>(type: K): Map<string, EntityRegistry[K]>;
removeEntity<K extends keyof EntityRegistry>(type: K, id: string): void;
hasEntities(type: keyof EntityRegistry): boolean;

// Untyped methods ‚Äî used by pipeline and fkResolution for dynamic iteration
getEntitiesUntyped(type: string): Map<string, Entity>;
hasEntitiesUntyped(type: string): boolean;
getEntityTypeKeys(): string[];
sealRawEntityTypes(): void;
```

The typed methods use `keyof EntityRegistry` ‚Äî typos are compile errors.
The untyped methods accept `string` ‚Äî for pipeline infrastructure that iterates
dynamically via `getEntityTypeKeys()`.

### BatchContext implementation

```typescript
addEntity<K extends keyof EntityRegistry>(type: K, id: string, entity: EntityRegistry[K]): void {
  // Runtime validation: instanceof check against ENTITY_CONSTRUCTORS[type]
  const ctor = ENTITY_CONSTRUCTORS[type];
  if (!(entity instanceof ctor)) {
    throw new Error(`addEntity('${type}'): expected ${ctor.name}, got ${entity?.constructor?.name}`);
  }
  // ... existing seal check + map storage
}

getEntities<K extends keyof EntityRegistry>(type: K): Map<string, EntityRegistry[K]> {
  const raw = this.entities.get(type);
  if (!raw || raw.size === 0) return new Map() as Map<string, EntityRegistry[K]>;

  // Runtime validation: spot-check first entity
  const ctor = ENTITY_CONSTRUCTORS[type];
  const first = raw.values().next().value;
  if (!(first instanceof ctor)) {
    throw new Error(`getEntities('${type}'): expected ${ctor.name}, got ${first?.constructor?.name}`);
  }
  return raw as Map<string, EntityRegistry[K]>;
}

// For pipeline/fkResolution ‚Äî no validation, returns Entity
getEntitiesUntyped(type: string): Map<string, Entity> {
  return this.entities.get(type) ?? new Map();
}
```

### handlerHelpers.ts changes

```typescript
// Before:
export async function resolveEntity<T extends Entity>(
  store: Store, batchCtx: IBatchContext, entityType: string,
  entityClass: EntityConstructor<T>, id: string,
): Promise<T | null>

// After:
export async function resolveEntity<K extends keyof EntityRegistry>(
  store: Store, batchCtx: IBatchContext, entityType: K, id: string,
): Promise<EntityRegistry[K] | null> {
  const batchEntity = batchCtx.getEntities(entityType).get(id);
  if (batchEntity) return batchEntity;
  const ctor = getEntityConstructor(entityType);
  const dbEntity = await store.findOneBy(ctor, { id } as FindOptionsWhere<EntityRegistry[K]>);
  return dbEntity ?? null;
}
```

Callers drop the `entityClass` argument:
```typescript
// Before:
const existing = await resolveEntity(hctx.store, hctx.batchCtx, ENTITY_TYPE, OrbFaction, id);
// After:
const existing = await resolveEntity(hctx.store, hctx.batchCtx, 'OrbFaction', id);
```

### metadataFetch.ts

This is the one exception. `handleMetadataFetch<TEntity>` is a generic utility called
with different entity types at different call sites. It receives `triggeredBy: string`
from the handler dispatch. It needs to remain generic because the config object carries
the entity class. This file keeps its existing `as Map<string, TEntity>` cast ‚Äî it's
justified because the generic is caller-determined and validated by the config pattern.

### Handler `listensToBag` type

Change from `string[]` to `(keyof EntityRegistry)[]` on `EntityHandler` interface.
This ensures handler subscriptions reference valid bag keys.

### Handler `triggeredBy` parameter

Change from `string` to `keyof EntityRegistry` on `EntityHandler.handle()`. This
flows naturally: the pipeline passes the bag key from `listensToBag`, which is now
typed. Handlers can use `triggeredBy` directly in `getEntities(triggeredBy)`.

### Pipeline / fkResolution

These iterate dynamically via `getEntityTypeKeys()` (returns `string[]`) and call
`getEntitiesUntyped(type)`. No changes to their logic ‚Äî just method rename from
`getEntities` to `getEntitiesUntyped` at the call sites.

`fkResolution.ts` accesses entities by dynamic rule-based keys (`rule.sourceType`,
`rule.targetType`) which are strings. Uses `getEntitiesUntyped`.

### Test strategy

**Unit tests (batchContext.test.ts):** Use real entity types from `@chillwhales/typeorm`
instead of synthetic `{ id, ...B }` objects. This validates the instanceof checks work.

**Pipeline tests (pipeline.test.ts):** Uses `getEntitiesUntyped` for dynamic iteration
paths. For handler-simulation paths that call `addEntity`, use real entity constructors.

**Handler tests:** Already use real entity types ‚Äî just remove `getTypedEntities` import
and use `batchCtx.getEntities(key)` directly.

**handlerHelpers tests:** Update to drop `entityClass` parameter from resolveEntity calls.
</design>
