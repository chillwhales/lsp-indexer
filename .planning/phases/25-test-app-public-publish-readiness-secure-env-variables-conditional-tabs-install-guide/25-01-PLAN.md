Create two new files:

**`apps/test/src/lib/env-config.ts`** — Server-only utility (no 'use client' directive):
- Export `interface EnvAvailability { hasClientUrl: boolean; hasServerUrl: boolean; hasClientWs: boolean; hasServerWs: boolean; }`
- Export `function getEnvAvailability(): EnvAvailability` that returns boolean flags by checking `Boolean(process.env.VARIABLE_NAME)` for each:
  - `hasClientUrl`: `NEXT_PUBLIC_INDEXER_URL` is set
  - `hasServerUrl`: `INDEXER_URL` is set (explicit only — do NOT include NEXT_PUBLIC fallback; user decision PUB-05 says show Server tab only if server-side env is set)
  - `hasClientWs`: `NEXT_PUBLIC_INDEXER_WS_URL` is set OR `NEXT_PUBLIC_INDEXER_URL` is set (derived WS URL)
  - `hasServerWs`: `INDEXER_WS_URL` is set OR `INDEXER_URL` is set (derived WS URL)
- CRITICAL: This file must NEVER export or return actual env var values — only booleans

**`apps/test/src/components/env-provider.tsx`** — Client component ('use client'):
- Import `type { EnvAvailability }` from `@/lib/env-config`
- Create `EnvContext` via `createContext<EnvAvailability | null>(null)`
- Export `function EnvProvider({ value, children }: { value: EnvAvailability; children: ReactNode }): ReactNode` that wraps children in context
- Export `function useEnvAvailability(): EnvAvailability` that reads context, throws if null (missing provider)

Follow project conventions: explicit return types, single quotes, 2-space indent, JSDoc on exports.

**`apps/test/src/app/layout.tsx`** (RSC — no 'use client'):
- Import `getEnvAvailability` from `@/lib/env-config` and `EnvProvider` from `@/components/env-provider`
- Call `const envAvailability = getEnvAvailability()` in the component body (runtime detection)
- Wrap children flow: `<Providers hasClientWs={envAvailability.hasClientWs} hasServerWs={envAvailability.hasServerWs}>` then inside `<EnvProvider value={envAvailability}>`
- The EnvProvider should be INSIDE Providers (since it's a 'use client' component and Providers already is)
- Layout structure becomes: `<Providers hasClientWs={...} hasServerWs={...}><EnvProvider value={envAvailability}><SidebarProvider>...</SidebarProvider></EnvProvider></Providers>`

**`apps/test/src/app/providers.tsx`** ('use client'):
- Add props: `{ hasClientWs: boolean; hasServerWs: boolean; children: ReactNode }`
- Conditionally wrap `<ReactSubscriptionProvider>` only if `hasClientWs` is true
- Conditionally wrap `<NextSubscriptionProvider>` only if `hasServerWs` is true
- Remove hardcoded `proxyUrl="ws://localhost:4000"` from NextSubscriptionProvider — let it use its default (`/api/graphql`) which works for non-localhost deployments
- If neither WS is available, just render `<TooltipProvider>{children}</TooltipProvider>` without any subscription providers
- Keep QueryClientProvider and ThemeProvider unconditional (always needed)

**`apps/test/src/components/playground/page-layout.tsx`** ('use client'):
- Import `useEnvAvailability` from `@/components/env-provider`
- Call `const { hasClientUrl, hasServerUrl } = useEnvAvailability()` inside the component
- Build `availableModes: HookMode[]` — push 'client' if hasClientUrl, push 'server' if hasServerUrl
- Default `mode` state to `availableModes[0] ?? 'client'` (fallback to client if nothing available)
- Only render the mode toggle button group if `availableModes.length > 1`
- If only one mode available, still show the mode indicator Badge but hide the toggle buttons
- If NO modes available (neither env set), show a warning message: "No environment variables configured. Set NEXT_PUBLIC_INDEXER_URL for client mode or INDEXER_URL for server mode."
