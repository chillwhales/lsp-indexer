<task type="auto">
  <name>Task 1: Fix case-sensitive contract filter address comparison</name>
  <files>packages/indexer-v2/src/core/pipeline.ts</files>
  <action>
At line 205 in pipeline.ts, change the contract filter address comparison from strict equality to case-insensitive comparison:

**Before:**

```typescript
if (plugin.contractFilter && log.address !== plugin.contractFilter.address) {
```

**After:**

```typescript
if (plugin.contractFilter && log.address.toLowerCase() !== plugin.contractFilter.address.toLowerCase()) {
```

This is a one-line change. The root cause: Subsquid delivers `log.address` as lowercase hex strings, but the V2 constants (`LSP26_ADDRESS`, `LSP23_ADDRESS`) use mixed-case EIP-55 checksummed addresses. The strict `!==` comparison never matches, silencing ALL events from:

- LSP26 contract → Follow plugin, Unfollow plugin (>100K + >2K entities)
- LSP23 contract → DeployedContracts plugin, DeployedERC1167Proxies plugin (>35K entities)

Do NOT change the constants or normalize them elsewhere — the fix belongs at the comparison point where addresses from two different sources are compared.
</action>

3. Verify no other address comparisons in pipeline.ts use strict equality against mixed-case constants
   </verify>
