Signature:

```ts
export function buildBlockOrderSort(direction: 'asc' | 'desc'): Follower_Order_By[];
```

Wait — the function needs to return a generic order_by array that works for ALL 4 Hasura table types (`Follower_Order_By`, `Data_Changed_Order_By`, `Token_Id_Data_Changed_Order_By`, `Universal_Receiver_Order_By`). All 4 types have `block_number`, `transaction_index`, `log_index` fields typed as `Order_By | null`.

Use a generic return type. Since all 4 Hasura order_by types share the same `block_number`/`transaction_index`/`log_index` fields, return as `Record<string, Order_By>[]` which is assignable to all 4.

Implementation:

```ts
/**
 * Build a deterministic block-order sort for event domains.
 *
 * Returns a 3-level tiebreaker: block_number → transaction_index → log_index.
 * Multiple events can share the same block timestamp, so this 3-level sort
 * guarantees deterministic ordering at the DB/Hasura layer.
 *
 * Used as default sort (newest first) when no sort is specified, and also
 * when consumer selects 'newest' or 'oldest' sort fields.
 *
 * @param direction - 'asc' for oldest-first, 'desc' for newest-first
 * @returns Hasura order_by array with 3 sort levels
 */
export function buildBlockOrderSort(direction: 'asc' | 'desc'): Record<string, Order_By>[] {
  return [{ block_number: direction }, { transaction_index: direction }, { log_index: direction }];
}
```

The `Order_By` type is already imported at the top of the file. The return type `Record<string, Order_By>[]` is structurally compatible with all 4 Hasura `*_Order_By[]` types because TypeScript uses structural typing (each object has the matching field).
</action>
