# Phase 20.3: Grafana Dashboard Redesign — Context

## Problem

The current dashboard was built before structured logging was in place. After Phases 20.1 and 20.2, we'll have rich queryable fields that can power much better monitoring panels.

## New Panels to Build

### 1. Pipeline Step Latency (timeseries)
```logql
# Per-step duration over time
max by (step) (
  {compose_service="indexer"} | json | step != "" | durationMs > 0 | unwrap durationMs [$__interval]
)
```
Visualization: stacked area or grouped bar, one series per step.

### 2. Entity Throughput (timeseries)
```logql
# Entities persisted per interval by type
sum by (entityType) (
  {compose_service="indexer"} | json | step="PERSIST_RAW" | unwrap count [$__interval]
)
```

### 3. Verification Health (stat + timeseries)
```logql
# Valid vs invalid ratio
sum({compose_service="indexer"} | json | step="VERIFY" | unwrap validCount [1h])
sum({compose_service="indexer"} | json | step="VERIFY" | unwrap invalidCount [1h])
```

### 4. Metadata Fetch Progress (stat + timeseries)
```logql
# Backlog depth by entity type
max by (entityKey) (
  {compose_service="indexer"} | json | unfetchedCount > 0 | unwrap unfetchedCount [$__interval]
)
# Fetch duration
max by (entityKey) (
  {compose_service="indexer"} | json | msg="Worker pool returned results" | unwrap durationMs [$__interval]
)
```

### 5. Batch Processing Time (timeseries)
```logql
# Total batch duration
max(
  {compose_service="indexer"} | json | step="BATCH_SUMMARY" | unwrap totalDurationMs [$__interval]
)
```
