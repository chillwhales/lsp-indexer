# Phase 20.3 Plan — Grafana Dashboard Redesign

## Goal
Add 5 new sections with 16 new panels to the Grafana dashboard, leveraging structured fields introduced in Phases 20.1 and 20.2. Keep all 13 existing panels.

## New Sections

### 1. Pipeline Performance (y=23)
**New panels:**
- Batch Processing Time (timeseries): `step="BATCH_SUMMARY" | unwrap totalDurationMs`
- Pipeline Step Latency (timeseries): per-step `durationMs` grouped by `step`
- Avg Batch Duration (stat)
- Entities per Batch (stat): `unwrap totalEntities`
- Enrichments per Batch (stat): `unwrap totalEnrichments`
- Blocks per Batch (stat): `unwrap blockCount`

### 2. Entity & Verification (y=36)
**New panels:**
- Entity Throughput (timeseries): `step="PERSIST_RAW" | unwrap count` grouped by `entityType`
- Verification Health (timeseries): validCount + newCount + invalidCount from `step="VERIFY"`

### 3. Metadata Fetch (y=45)
**New panels:**
- Metadata Fetch Backlog (timeseries): `unfetchedCount` grouped by `entityKey`
- Metadata Fetch Duration (timeseries): drain `durationMs` grouped by `entityKey`
- LSP3 Fetch Success Rate (stat): totalProcessed / (totalProcessed + totalFailed)
- LSP4 Fetch Success Rate (stat)
- LSP29 Fetch Success Rate (stat)

## Files Changed
- `docker/grafana/provisioning/dashboards/indexer-monitoring.json`
  - 13 existing panels preserved unchanged
  - 16 new panels added across 3 new sections
  - 2 collapsed sections (Log Explorer, Container Metrics) moved to bottom

## Key Queries
All queries use Loki json parser — no regex needed since fields are native structured JSON.

## Dependencies
- Phase 20.1: structured logging fields (entityKey, unfetchedCount, backlogCount, etc.)
- Phase 20.2: timing fields (durationMs, BATCH_SUMMARY step)
