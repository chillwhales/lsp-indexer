# Phase 10.1: Subscription Foundation — Context

**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Base subscription infrastructure only. Type-safety refactor to eliminate `unknown`/`any` from the subscription data path, export where-clause builders for domain sub-phases to use, wire provider in test app, create playground skeleton. Does NOT include subscription documents or domain hooks — those are added per-domain in sub-phases 10.2–10.13.

</domain>

<decisions>
## Implementation Decisions

### Sub-phase scope split

- 10.1 delivers the infrastructure that all domain sub-phases build on
- 10.1 does NOT create subscription documents — each domain sub-phase (10.2–10.13) creates its own document alongside its hook and playground demo
- This matches the Phase 9 pattern where each domain sub-phase is self-contained

### What 10.1 covers

- Type-safety refactor: `SubscriptionConfig<T>` → `SubscriptionConfig<TResult, TVariables, TRaw, TParsed>` with typed `extract` function replacing string `dataKey`
- Export `buildXWhere` functions from all 12 service files (add `export` keyword)
- Wire `IndexerSubscriptionProvider` in test app layout
- Create `/subscriptions` playground skeleton page

### What 10.1 does NOT cover

- Subscription GraphQL documents (per-domain in 10.2–10.13)
- Domain subscription hooks (per-domain in 10.2–10.13)
- Playground demo tabs (per-domain in 10.2–10.13)

### Hook factory pattern

- Every domain subscription hook follows the `createUseSubscription` factory pattern already established
- Factory lives in `@lsp-indexer/react` (e.g., `create-use-{domain}-subscription.ts`)
- Both `@lsp-indexer/react` and `@lsp-indexer/next` import the factory and bind it to their own context
- This is the same approach as the existing `createUseSubscription` in `packages/react/src/subscriptions/create-use-subscription.ts`

### Claude's Discretion

- Exact generic parameter naming in the type-safety refactor
- Whether `extract` function or other mechanism replaces string `dataKey`
- Playground skeleton layout and styling

</decisions>

<specifics>
## Specific Ideas

- The existing `createUseSubscription` factory is the reference pattern — domain hooks should follow the same structure
- Type-safety goal: zero `as` casts, zero `unknown`, zero `any` in the subscription data path from document through to consumer

</specifics>
