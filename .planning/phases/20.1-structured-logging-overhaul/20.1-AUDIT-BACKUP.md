# Phase 20.1: Structured Logging Overhaul — Context

## Problem

The indexer has 40 log instances across 8 files that use inconsistent patterns, making them invisible or unqueryable in Grafana/Loki. This blocks effective monitoring.

## Subsquid Logger API

The Subsquid Logger (`@subsquid/logger@1.4.0`) **fully supports** pino-style structured attrs:

```typescript
log.info({ key: value }, 'message');  // ✅ attrs as first arg
log.info('message');                   // ✅ string-only
log.child({ persistent: 'field' });    // ✅ child loggers
```

All current anti-patterns can be migrated without infrastructure changes.

## Instances to Fix (40 total, 8 files)

### Category 1: `console.*` calls (12 instances) — CRITICAL

These bypass structured logging entirely and are invisible to Loki field queries.

| File | Line(s) | Current | Migration Target |
|------|---------|---------|-----------------|
| `core/registry.ts` | 91, 98, 111, 116, 136, 143, 156, 161 | `console.warn/info(template)` | Requires passing a logger — registry currently has no logger reference. Options: (a) accept logger in constructor, (b) use `@subsquid/logger` `createLogger()` directly |
