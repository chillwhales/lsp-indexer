**1. Create `packages/indexer/LOGGING.md`** — Field naming reference document per user decision.

Include:
- Core fields (MUST be on every log call):
  - `step`: Pipeline step identifier (BOOTSTRAP, EXTRACT, HANDLE, PERSIST_RAW, etc.)
  - `handler` or `plugin` or `component`: Component name emitting the log
- Common optional fields with recommended vocabulary:
  - `address` — Ethereum address
  - `category` — EntityCategory value
  - `durationMs` — Timing in milliseconds
  - `entityType` — Entity bag key (e.g., 'Transfer', 'LSP4Metadata')
  - `count` — Numeric count of items
  - `file` — Source file path (for discovery logs)
  - `pluginName` / `handlerName` — Specific plugin/handler name
  - `error` — Error message string
  - `batchIndex` — Batch processing index
  - `totalCount` — Total items in set
  - `workerId` — Worker thread identifier
  - `threadId` — Node.js thread ID
- 2-3 examples showing correct `(attrs, message)` usage
- Anti-patterns: JSON.stringify in log args, template strings embedding data

**2. Modify `PluginRegistry` constructor** to accept a Logger parameter per user decision (constructor injection, step: BOOTSTRAP, component: registry).

In `packages/indexer/src/core/registry.ts`:
- Add `import type { Logger } from '@subsquid/logger';` at top
- Add `private readonly logger: Logger;` field
- Add constructor: `constructor(logger: Logger)` that creates a child logger:
  `this.logger = logger.child({ step: 'BOOTSTRAP', component: 'registry' });`
- Replace all 10 `console.*` calls with structured logger calls:
  - `console.warn(\`[Registry] Invalid module export in ${file}, skipping\`)` →
    `this.logger.warn({ file }, 'Invalid module export, skipping')`
  - `console.warn(\`[Registry] No default/plugin export in ${file}, skipping\`)` →
    `this.logger.warn({ file }, 'No default/plugin export, skipping')`
  - `console.warn(\`[Registry] Export in ${file} does not implement EventPlugin, skipping\`)` →
    `this.logger.warn({ file }, 'Export does not implement EventPlugin, skipping')`
  - `console.info(\`[Registry] Discovered ${this.eventPlugins.size} event plugins\`)` →
    `this.logger.info({ pluginCount: this.eventPlugins.size }, 'Discovered event plugins')`
  - Same pattern for all 6 handler discovery console.* calls (3 warn + 1 info in discoverHandlers)
- Do NOT modify `registerEventPlugin()` or `registerEntityHandler()` — those throw errors, not log
- Do NOT modify the topological sort — that throws errors, not log
