# Plan 09.6-02: Actions + Hooks — Generic Overloads + Return Type Aliases (next + react packages)

**Phase:** 9.6 — Generic Type Propagation
**Requirement:** DX-05
**Packages:** `@lsp-indexer/react`, `@lsp-indexer/next`
**Depends on:** 09.6-01 (parser + service overloads must exist for type propagation)

## Goal

Add the 3-overload generic `<const I>` pattern and explicit `UseXReturn<F>` / `UseInfiniteXReturn<F>` return type aliases to all hooks and server actions across 5 domains in the `@lsp-indexer/react` and `@lsp-indexer/next` packages. After this plan:

- All hooks return `XResult<I>` when a typed include is provided
- `error` is typed as `Error` (not `unknown`) on all hook returns
- All server actions propagate generic narrowing to hooks

## Reference Pattern

**Followers hook (gold standard) — from `packages/react/src/hooks/followers.ts`:**

```typescript
import type { InfiniteData, UseInfiniteQueryResult, UseQueryResult } from '@tanstack/react-query';

type UseFollowsReturn<F> = { follows: F[]; totalCount: number } & Omit<
  UseQueryResult<FetchFollowsResult<F>, Error>, 'data'
>;

type UseInfiniteFollowsReturn<F> = {
  follows: F[];
  hasNextPage: boolean;
  fetchNextPage: UseInfiniteQueryResult['fetchNextPage'];
  isFetchingNextPage: boolean;
} & Omit<
  UseInfiniteQueryResult<InfiniteData<FetchFollowsResult<F>>, Error>,
  'data' | 'hasNextPage' | 'fetchNextPage' | 'isFetchingNextPage'
>;

export function useFollows<const I extends FollowerInclude>(
  params: UseFollowsParams & { include: I },
): UseFollowsReturn<FollowerResult<I>>;
export function useFollows(
  params: Omit<UseFollowsParams, 'include'> & { include?: never },
): UseFollowsReturn<Follower>;
export function useFollows(
  params: UseFollowsParams & { include?: FollowerInclude },
): UseFollowsReturn<PartialFollower>;
export function useFollows(
  params: UseFollowsParams & { include?: FollowerInclude },
): UseFollowsReturn<PartialFollower> { ... }
```

**Followers action (gold standard) — from `packages/next/src/actions/followers.ts`:**

```typescript
export async function getFollows(params: { ... }): Promise<FetchFollowsResult>;
export async function getFollows<const I extends FollowerInclude>(params: { ...; include: I }): Promise<FetchFollowsResult<FollowerResult<I>>>;
export async function getFollows(params: { ...; include?: FollowerInclude }): Promise<FetchFollowsResult<PartialFollower>>;
export async function getFollows(params: { ...; include?: FollowerInclude }): Promise<FetchFollowsResult<PartialFollower>> { ... }
```

### Task 6: Update Profile React hooks with 3-overload pattern + return type aliases

**File:** `packages/react/src/hooks/profiles.ts`

**Actions:**

1. Add imports: `import type { InfiniteData, UseInfiniteQueryResult, UseQueryResult } from '@tanstack/react-query'`
2. Add imports from types: `Profile, ProfileResult, FetchProfilesResult` (FetchProfilesResult already from node)
3. Add `UseProfileReturn<F>` type alias for detail hook:
   ```typescript
   type UseProfileReturn<F> = { profile: F | null } & Omit<UseQueryResult<F | null, Error>, 'data'>;
   ```
4. Add `UseProfilesReturn<F>` type alias for list hook:
   ```typescript
   type UseProfilesReturn<F> = { profiles: F[]; totalCount: number } & Omit<
     UseQueryResult<FetchProfilesResult<F>, Error>,
     'data'
   >;
   ```
5. Add `UseInfiniteProfilesReturn<F>` type alias for infinite hook:
   ```typescript
   type UseInfiniteProfilesReturn<F> = {
     profiles: F[];
     hasNextPage: boolean;
     fetchNextPage: UseInfiniteQueryResult['fetchNextPage'];
     isFetchingNextPage: boolean;
   } & Omit<
     UseInfiniteQueryResult<InfiniteData<FetchProfilesResult<F>>, Error>,
     'data' | 'hasNextPage' | 'fetchNextPage' | 'isFetchingNextPage'
   >;
   ```
6. `useProfile` — add 3 overloads:
   - Overload 1 (generic): `useProfile<const I extends ProfileInclude>(params: UseProfileParams & { include: I }): UseProfileReturn<ProfileResult<I>>`
   - Overload 2 (no include): `useProfile(params: Omit<UseProfileParams, 'include'> & { include?: never }): UseProfileReturn<Profile>`
   - Overload 3 (dynamic): `useProfile(params: UseProfileParams & { include?: ProfileInclude }): UseProfileReturn<PartialProfile>`
   - Implementation: `useProfile(params): UseProfileReturn<PartialProfile>`
   - Remove explicit `const profile: PartialProfile | null = data ?? null` — let type flow from return type
7. `useProfiles` — add 3 overloads (same pattern, list shape)
8. `useInfiniteProfiles` — add 3 overloads (same pattern, infinite shape)

## Notes

- Implementation bodies do NOT change — only overload declarations and return type annotations
- The `Omit<UseProfileParams, 'include'> & { include?: never }` on the no-include overload prevents TypeScript from matching the wrong overload when include is omitted
- Remove explicit type annotations on intermediate variables (e.g., `const profiles: PartialProfile[] = ...`) — let the return type alias handle typing
- The return type aliases ensure `error: Error` (not `error: unknown`) for all hook consumers
