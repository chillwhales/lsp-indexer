  <action>
    Create one file per domain (12 files total — 11 user-facing domains where owned-assets and owned-tokens are separate hooks), each exporting a single `use{Domain}Subscription` hook function. All hooks follow the same pattern — they are thin wrappers around the generic `useSubscription` hook from `./use-subscription.ts`.

    **Pattern for each domain hook:**

    ```typescript
    import type { {Domain}, {Domain}Filter } from '@lsp-indexer/types';
    import { build{Domain}Where, {domain}Keys, parse{Domain}s } from '@lsp-indexer/node';
    // OR parseProfile (singular) if parseProfiles doesn't exist — check existing parser exports
    import { useQueryClient, type QueryClient } from '@tanstack/react-query';
    import { {Domain}SubscriptionDocument } from './documents';
    import { useSubscription } from './use-subscription';
    import type { UseSubscriptionReturn } from './use-subscription';

    const DEFAULT_LIMIT = 10;

    interface Use{Domain}SubscriptionParams {
      filter?: {Domain}Filter;
      limit?: number;
      enabled?: boolean;
      invalidate?: boolean;
      onData?: (data: {Domain}[]) => void;
      onReconnect?: () => void;
    }

    export function use{Domain}Subscription(
      params: Use{Domain}SubscriptionParams = {},
    ): UseSubscriptionReturn<{Domain}> {
      const {
        filter,
        limit = DEFAULT_LIMIT,
        enabled = true,
        invalidate = false,
        onData,
        onReconnect,
      } = params;

      // Build Hasura where clause from filter
      const where = build{Domain}Where(filter);

      // Always call useQueryClient unconditionally (Rules of Hooks), but only USE it when invalidate=true
      let queryClient: QueryClient | undefined;
      try {
        queryClient = useQueryClient();
      } catch {
        // No QueryClientProvider — cache invalidation won't work but hook still functions
      }

      return useSubscription({
        document: {Domain}SubscriptionDocument,
        dataKey: '{hasura_table_name}',
        variables: {
          where: Object.keys(where).length > 0 ? where : undefined,
          order_by: {defaultSort},
          limit,
        },
        parser: (raw) => parse{Domain}s(raw),
        enabled,
        invalidate,
        // keys.all prefix-matches all detail + list + infinite queries for this domain.
        // This achieves two-tier invalidation: TanStack Query's prefix matching means
        // keys.all invalidates both detail keys (e.g., profileKeys.detail(addr)) and
        // list keys (profileKeys.list(...)). Only observed queries actually refetch.
        invalidateKeys: invalidate ? [{domain}Keys.all] : undefined,
        queryClient: invalidate ? queryClient : undefined,
        onData,
        onReconnect,
      });
    }
    ```

    **IMPORTANT: The `useQueryClient()` hook call pattern.**
    React hooks MUST be called unconditionally (Rules of Hooks). Always call `useQueryClient()` at the top level of the hook function, wrapped in try/catch (it throws if no QueryClientProvider exists). Only PASS the result to `useSubscription` when `invalidate` is true. Never conditionally call `useQueryClient()` based on the `invalidate` parameter — that violates React's Rules of Hooks and will crash at runtime.

    **Domain-specific details:**

    | Domain | Hook Name | buildXWhere | parser | keys | Hasura Table | Default Sort |
    |--------|-----------|-------------|--------|------|-------------|-------------|
    | profiles | useProfileSubscription | buildProfileWhere | parseProfiles (or map with parseProfile) | profileKeys | universal_profile | `[{ followedBy_aggregate: { count: desc } }]` (most followed) — or no sort (Hasura default newest by ID) |
    | digital-assets | useDigitalAssetSubscription | buildDigitalAssetWhere | parseDigitalAssets | digitalAssetKeys | digital_asset | undefined (newest by Hasura default) |
    | nfts | useNftSubscription | buildNftWhere | parseNfts | nftKeys | nft | undefined |
    | owned-assets | useOwnedAssetSubscription | buildOwnedAssetWhere | parseOwnedAssets | ownedAssetKeys | owned_asset | undefined |
    | owned-tokens | useOwnedTokenSubscription | buildOwnedTokenWhere | parseOwnedTokens | ownedTokenKeys | owned_token | undefined |
    | followers | useFollowerSubscription | buildFollowerWhere | parseFollowers | followerKeys | follower | `[{ block_number: desc }, { transaction_index: desc }, { log_index: desc }]` (block-order desc — event domain) |
    | creators | useCreatorSubscription | buildCreatorWhere | parseCreators | creatorKeys | lsp4_creator | undefined |
    | issued-assets | useIssuedAssetSubscription | buildIssuedAssetWhere | parseIssuedAssets | issuedAssetKeys | lsp12_issued_asset | undefined |
    | encrypted-assets | useEncryptedAssetSubscription | buildEncryptedAssetWhere | parseEncryptedAssets | encryptedAssetKeys | lsp29_encrypted_asset | undefined |
    | data-changed-events | useDataChangedEventSubscription | buildDataChangedEventWhere | parseDataChangedEvents | dataChangedEventKeys | data_changed | `[{ block_number: desc }, { transaction_index: desc }, { log_index: desc }]` (block-order desc — event domain) |
    | token-id-data-changed-events | useTokenIdDataChangedEventSubscription | buildTokenIdDataChangedEventWhere | parseTokenIdDataChangedEvents | tokenIdDataChangedEventKeys | token_id_data_changed | `[{ block_number: desc }, { transaction_index: desc }, { log_index: desc }]` (block-order desc — event domain) |
    | universal-receiver-events | useUniversalReceiverEventSubscription | buildUniversalReceiverEventWhere | parseUniversalReceiverEvents | universalReceiverEventKeys | universal_receiver | `[{ block_number: desc }, { transaction_index: desc }, { log_index: desc }]` (block-order desc — event domain) |

    **Default sort per the locked decision:** "fixed newest-first sort (block-order desc for event domains, timestamp desc for entity domains)."
    - Event domains (followers, data-changed, token-id-data-changed, universal-receiver): Use block-order desc sort: `[{ block_number: 'desc' }, { transaction_index: 'desc' }, { log_index: 'desc' }]`
    - Entity domains (profiles, digital-assets, nfts, owned-assets, owned-tokens, creators, issued-assets, encrypted-assets): Use `undefined` for order_by — Hasura default ordering by primary key (newest first for auto-increment IDs). Alternatively use `[{ id: 'desc' }]` if explicit ordering desired.

    **Parser usage:** Each domain has `parseX` (singular, for one item) and `parseXs` (plural, for arrays). The subscription hook should use the plural parser. Check existing exports from `@lsp-indexer/node` — all domains export both.

    **IMPORTANT for parsers:** Subscription documents return ALL fields (no include), so call parsers WITHOUT the include parameter. This returns the full domain type.
    - For profiles: `parseProfiles(raw)` — no second arg
    - For digital assets: `parseDigitalAssets(raw)` — no second arg
    - Same pattern for all domains

    **IMPORTANT for owned-tokens and other complex domains:** The parsers may have nested parsing (e.g., parseOwnedToken parses nested digitalAsset, nft, holder). Since subscription documents select ALL nested fields, the parsers will correctly parse everything.

    **NOTE on where clause builders:** Some buildXWhere functions accept `filter: XFilter | undefined` while others accept `filter?: XFilter`. Check each function's signature. All should work with optional/undefined filter gracefully (returning `{}` for no filter).
