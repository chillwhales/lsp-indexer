    **Create `packages/react/src/subscriptions/client.ts` — SubscriptionClient class:**

    This class wraps `graphql-ws`'s `createClient` and adds:
    1. Connection state tracking (`disconnected` | `connecting` | `connected`)
    2. Subscriber pattern for `useSyncExternalStore` (subscribe/getSnapshot)
    3. Reconnection detection (abruptlyClosed flag pattern from graphql-ws recipes)
    4. Reconnect callback registry
    5. Subscription execution via sink API

    Key implementation details:
    - Constructor accepts optional `url?: string` parameter. If not provided, calls `getClientWsUrlOrDerive()` from `@lsp-indexer/node`.
    - graphql-ws `createClient` options: `lazy: true`, `lazyCloseTimeout: 3000`, `retryAttempts: Infinity`, `shouldRetry: () => true`
    - Track connection state via `on.connecting`, `on.connected`, `on.closed` event handlers
    - Reconnection detection: set `abruptlyClosed = true` in `closed` handler when `(event as CloseEvent).code !== 1000`; in `connected` handler, check if `abruptlyClosed` was true (meaning this is a reconnect, not first connect) and fire reconnect callbacks
    - `subscribe(listener)` / `getSnapshot()` methods for `useSyncExternalStore` pattern — listeners are notified when connection state changes
    - `getServerSnapshot()` returns `'disconnected'` (SSR fallback)
    - `get isConnected(): boolean` property
    - `onReconnect(cb)` method — returns unregister function
    - `executeSubscription<TData>(payload, sink)` method — creates graphql-ws client lazily on first call, calls `client.subscribe(payload, sink)`, returns cleanup function
    - `dispose()` method — calls `client?.dispose()`, resets state

    Export the `SubscriptionClient` class and the `ConnectionState` type.

    **Create `packages/react/src/subscriptions/context.ts` — React context:**

    ```typescript
    import { createContext, useContext } from 'react';
    import type { SubscriptionClient } from './client';

    const SubscriptionClientContext = createContext<SubscriptionClient | null>(null);

    export { SubscriptionClientContext };

    /**
     * Get the SubscriptionClient from context.
     * Throws a helpful error if used outside IndexerSubscriptionProvider.
     */
    export function useSubscriptionClient(): SubscriptionClient {
      const client = useContext(SubscriptionClientContext);
      if (!client) {
        throw new Error(
          'useSubscriptionClient must be used within an <IndexerSubscriptionProvider>. ' +
          'Wrap your component tree with <IndexerSubscriptionProvider> to use subscription hooks.'
        );
      }
      return client;
    }
    ```

    **Create `packages/react/src/subscriptions/provider.tsx` — Provider component:**

    ```tsx
    'use client';

    import { useRef } from 'react';
    import { SubscriptionClient } from './client';
    import { SubscriptionClientContext } from './context';

    interface IndexerSubscriptionProviderProps {
      /** Optional explicit WebSocket URL. If omitted, derived from NEXT_PUBLIC_INDEXER_WS_URL or NEXT_PUBLIC_INDEXER_URL */
      wsUrl?: string;
      children: React.ReactNode;
    }

    /**
     * Provides a shared WebSocket subscription client to all subscription hooks.
     *
     * Place inside your QueryClientProvider (for cache invalidation support):
     * ```tsx
     * <QueryClientProvider client={queryClient}>
     *   <IndexerSubscriptionProvider>
     *     <App />
     *   </IndexerSubscriptionProvider>
     * </QueryClientProvider>
     * ```
     */
    export function IndexerSubscriptionProvider({ wsUrl, children }: IndexerSubscriptionProviderProps) {
      const clientRef = useRef<SubscriptionClient | null>(null);
      if (!clientRef.current) {
        clientRef.current = new SubscriptionClient(wsUrl);
      }

      return (
        <SubscriptionClientContext.Provider value={clientRef.current}>
          {children}
        </SubscriptionClientContext.Provider>
      );
    }
    ```

    Use `useRef` to ensure the client is created once (singleton per provider instance). Do NOT use `useState` for the client — it would cause unnecessary re-renders. Do NOT use `useMemo` — React docs warn it may be called multiple times in concurrent mode.

    The `'use client'` directive is needed because this is a React component.

    **Create `packages/react/src/subscriptions/use-subscription.ts` — generic hook:**

    This is the core reusable hook that all 11 domain hooks wrap. It manages:

    1. **Connection state** — `useSyncExternalStore` reading from `SubscriptionClient`
    2. **Subscription lifecycle** — `useEffect` that creates/destroys subscription when enabled/variables change
    3. **Data state** — `useState<T[] | null>(null)` for the latest subscription result
    4. **Subscribed state** — `useState<boolean>(false)` tracking if this specific subscription is active
    5. **Error state** — `useState<IndexerError | null>(null)`
    6. **Cache invalidation** — When `invalidate: true` and data arrives, call `queryClient.invalidateQueries()` for each key in `invalidateKeys`
    7. **Callbacks** — Fire `onData` when data arrives, fire `onReconnect` when WebSocket reconnects

    Interface:
    ```typescript
    export interface UseSubscriptionOptions<TRaw, TParsed> {
      /** GraphQL subscription document string */
      document: string;
      /** The key in the GraphQL response object (e.g., 'universal_profile') */
      dataKey: string;
      /** GraphQL variables (where, order_by, limit) */
      variables: Record<string, unknown>;
      /** Parser function to transform raw Hasura data to clean types */
      parser: (raw: TRaw[]) => TParsed[];
      /** Enable/disable the subscription */
      enabled?: boolean;
      /** Whether to invalidate TanStack Query cache on data */
      invalidate?: boolean;
      /** Query keys to invalidate when data arrives */
      invalidateKeys?: readonly (readonly unknown[])[];
      /** Callback when new data arrives */
      onData?: (data: TParsed[]) => void;
      /** Callback on WebSocket reconnect */
      onReconnect?: () => void;
    }

    export interface UseSubscriptionReturn<T> {
      data: T[] | null;
      isConnected: boolean;
      isSubscribed: boolean;
      error: IndexerError | null;
    }
    ```

    Implementation details:
    - Get SubscriptionClient from context via `useSubscriptionClient()`
    - Get connection state via `useSyncExternalStore(client.subscribe, client.getSnapshot, client.getServerSnapshot)`
    - `isConnected` derived from connection state === 'connected'
    - Use `useRef` for `onData` and `onReconnect` callbacks to avoid stale closures and unnecessary effect re-runs
    - The subscription `useEffect`:
      - Skip if `enabled === false`
      - Call `client.executeSubscription(payload, sink)` where:
        - sink.next: parse data, set state, call onData ref, and if invalidate is true call `queryClient.invalidateQueries` for each invalidateKeys entry
        - sink.error: create IndexerError from the error, set error state
        - sink.complete: set isSubscribed to false
      - Set `isSubscribed` to `true` after subscribing
      - Return cleanup function from executeSubscription
      - Dependencies: `[client, document, stableVariables, enabled]` where stableVariables is JSON.stringify(variables) to avoid object reference issues
    - For cache invalidation: Accept `queryClient?: QueryClient` as an optional parameter. Domain hooks (Plan 03) unconditionally call `useQueryClient()` (wrapped in try/catch) and pass it through. The generic hook does NOT call `useQueryClient()` itself — it receives it as a parameter, avoiding conditional hook call issues entirely.
    - Reset data/error/isSubscribed when enabled changes from true to false
    - Register reconnect callback with client when `onReconnect` is provided
    - On reconnect with invalidate: true, also fire cache invalidation (stale caches after reconnect)
