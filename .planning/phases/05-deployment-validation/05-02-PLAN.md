  <action>
Create `packages/indexer-v2/src/comparison/comparisonEngine.ts` implementing the three-phase comparison run per user decisions.

Export `async function runComparison(config: ComparisonConfig): Promise<ComparisonReport>`:

**Phase 1 — Aggregate row counts:**

1. Create two GraphQL clients: `createGraphqlClient(config.v1Url, config.v1Secret)` and `createGraphqlClient(config.v2Url, config.v2Secret)`
2. Health-check both endpoints. If either fails, throw with clear error message.
3. Determine entity list: if `config.entities` is provided, filter `ENTITY_REGISTRY` to those names; otherwise use all.
4. For each entity in the list, query `queryCount(entity.hasuraTable)` on both endpoints in parallel (Promise.all for the two endpoints per entity, but process entities sequentially to avoid overwhelming Hasura).
5. If count returns -1 (table missing), record in `missingEntityTypes`.
6. Build `CountResult[]` with match = (v1Count === v2Count).

**Phase 2 — Sampled content diffs:**

1. For each entity where both endpoints have rows (count > 0 on both sides):
   a. Fetch sample IDs from V1: `querySampleIds(entity.hasuraTable, config.sampleSize)` (default 100)
   b. Fetch those same IDs from both V1 and V2: `queryRowsByIds(entity.hasuraTable, sampleIds)` on both clients
   c. Match rows by `id` field
   d. For each matched row pair, compare all fields:
   - Skip `id` field (it's the key)
   - For each field where v1Value !== v2Value (use JSON.stringify for deep comparison of objects/arrays): - Check if this field is in `getKnownDivergences(entity.name)` — if yes, add to `knownDivergences` - Otherwise add to `unexpectedDiffs`
     e. Only record RowDiff entries that have at least one diff (known or unexpected)
     f. If V2 is missing a row that V1 has, record it as an unexpected diff with all V1 fields

**Phase 3 — Summary:**

1. `passed` = true only if:
   - All entity counts match (excluding metadata sub-entities where `isMetadataSub: true` — timing differences are expected)
   - Zero unexpected diffs in sampled content
   - No missing entity types on V2 (V1 missing types are warnings, not failures)
2. Return the complete `ComparisonReport`

Important details:

- Process entities sequentially (not all 72 in parallel) to avoid rate limiting Hasura
- Use a small delay between entities if needed (100ms)
- Log progress to stderr (not stdout) so the reporter controls stdout: `process.stderr.write(`\rComparing ${entity.name}...`)`
- Handle null vs undefined vs empty string differences: treat null, undefined, and '' as equivalent (Hasura may serialize these differently between V1 and V2)

**Reporter** (`packages/indexer-v2/src/comparison/reporter.ts`):

Create `printReport(report: ComparisonReport): void` that outputs a structured colored report to stdout.

Use ANSI escape codes directly (no chalk dependency needed — keep it simple):

```typescript
const GREEN = '\x1b[32m';
const RED = '\x1b[31m';
const YELLOW = '\x1b[33m';
const CYAN = '\x1b[36m';
const BOLD = '\x1b[1m';
const RESET = '\x1b[0m';
const DIM = '\x1b[2m';
```

Report sections:

1. **Header:**

   ```
   ═══════════════════════════════════════════════════
   V1 vs V2 Data Parity Comparison
   ═══════════════════════════════════════════════════
   ```

2. **Missing Entity Types** (if any):
   Warn about entities missing from either endpoint.

3. **Row Counts Table:**

   ```
   Entity Type               V1 Count    V2 Count    Status
   ─────────────────────────────────────────────────────────
   UniversalProfile             1,234       1,234    ✓ MATCH
   DigitalAsset                   567         565    ✗ MISMATCH
   LSP3ProfileImage*              890         887    ~ METADATA
   ```

   - Green ✓ for matches, Red ✗ for mismatches, Yellow ~ for metadata sub-entities (marked with \*)
   - Right-align numbers with comma formatting (e.g., 1,234)

4. **Sampled Content Diffs** (if any):

   ```
   ─── UniversalProfile (2 rows with diffs) ───

   Row: 0x1234...abcd
     field: V1 value → V2 value

   ─── Known Divergences ───

   OwnedAsset (1 row)
   Row: 0xabcd...1234
     owner: 0x5678 → null  (V2 uses null FK instead of entity removal)
   ```

   - Show only differing fields, not full rows
   - Truncate long values to 60 chars
   - Known divergences in a separate section with DIM styling and reason text

5. **Summary:**
   ```
   ═══════════════════════════════════════════════════
   RESULT: PASS ✓  (or FAIL ✗)
   ═══════════════════════════════════════════════════
   Entity types compared: 72
   Row count matches: 70/72  (2 metadata timing)
   Unexpected diffs: 0
   Known divergences: 3
   ```

**CLI** (`packages/indexer-v2/src/comparison/cli.ts`):

Parse `process.argv` manually (no yargs/commander dependency — keep it zero-dep):

```
Usage: pnpm compare --v1=<url> --v2=<url> [options]

Options:
  --v1=<url>           V1 Hasura GraphQL endpoint (required)
  --v2=<url>           V2 Hasura GraphQL endpoint (required)
  --v1-secret=<secret> V1 Hasura admin secret
  --v2-secret=<secret> V2 Hasura admin secret
  --entities=<list>    Comma-separated entity filter (default: all)
  --sample-size=<n>    Rows to sample per entity (default: 100)
```

Implementation:

1. Parse args from `process.argv.slice(2)` looking for `--key=value` patterns

3. Build `ComparisonConfig` from parsed args
4. Call `runComparison(config)`
5. Call `printReport(report)`
6. Exit with `process.exit(report.passed ? 0 : 1)`
7. Wrap in try/catch — print error message and exit 1 on unexpected errors

Make the CLI file self-executing — it should work when run directly with ts-node.
