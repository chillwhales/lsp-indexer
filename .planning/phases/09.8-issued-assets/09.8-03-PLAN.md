Use `getClientUrl()` for direct browser → Hasura queries. Use 3-overload `<const I>` generic pattern with return type aliases.

---

**Hook 1: `useIssuedAssets`** — paginated list.

Return type alias: `UseIssuedAssetsReturn<F>` — same pattern as `UseCreatorsReturn<F>`:

```typescript
type UseIssuedAssetsReturn<F> = Omit<
  UseQueryResult<{ issuedAssets: F[]; totalCount: number }, Error>,
  'data'
> & {
  issuedAssets: F[];
  totalCount: number;
};
```

**NOTE:** The destructured property name is `issuedAssets` (matching `FetchIssuedAssetsResult.issuedAssets`), not `creators`.

3 overloads:

1. No include → `UseIssuedAssetsReturn<IssuedAsset>`
2. `<const I>` with include → `UseIssuedAssetsReturn<IssuedAssetResult<I>>`
3. Optional include → `UseIssuedAssetsReturn<PartialIssuedAsset>`

---

**Hook 2: `useInfiniteIssuedAssets`** — infinite scroll variant.

Return type alias: `UseInfiniteIssuedAssetsReturn<F>` — same pattern as `UseInfiniteCreatorsReturn<F>`.

**IMPORTANT:** Destructure infinite query properties before rest spread to avoid TS2783 (same pattern as creators):

```typescript
const { data, hasNextPage, fetchNextPage, isFetchingNextPage, ...rest } = result;
```

Flatten pages: `data?.pages.flatMap((page) => page.issuedAssets) ?? []`
