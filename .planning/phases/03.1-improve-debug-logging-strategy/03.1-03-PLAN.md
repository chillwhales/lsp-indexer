  <action>
Move `Date.now()` calculations and duration tracking inside the `isLevelEnabled('debug')` conditional blocks to eliminate overhead when debug logging is disabled.

**Pattern to fix (appears in 4 files):**

**BAD - Date calculations always happen:**

```typescript
const startTime = Date.now();
await handleMetadataFetch(hctx, fetchConfig, triggeredBy);
const duration = Date.now() - startTime;

if (logger.isLevelEnabled('debug')) {
  logger.debug({ duration }, 'Handler complete');
}
```

**GOOD - Date calculations only when debug enabled:**

```typescript
if (logger.isLevelEnabled('debug')) {
  const startTime = Date.now();
  await handleMetadataFetch(hctx, fetchConfig, triggeredBy);
  const duration = Date.now() - startTime;
  logger.debug({ duration }, 'Handler complete');
} else {
  await handleMetadataFetch(hctx, fetchConfig, triggeredBy);
}
```

**Files and locations:**

1. **lsp3ProfileFetch.handler.ts** (lines 228-230)

   - Wrap handleMetadataFetch call in if/else with startTime/duration inside debug block

2. **lsp4MetadataFetch.handler.ts** (lines 362-364)

   - Wrap handleMetadataFetch call in if/else with startTime/duration inside debug block

3. **lsp29EncryptedAssetFetch.handler.ts** (lines 342-344)

   - Wrap handleMetadataFetch call in if/else with startTime/duration inside debug block

4. **metadataWorkerPool.ts** (lines 187, 278)
   - This is trickier because startTime is at the top of the loop and duration is at the bottom
   - Move startTime inside the final debug block (line 279)
   - Calculate duration inline: `durationMs: Date.now() - Date.now()` won't work
   - **Solution:** Only log the completion debug block (lines 279-290), remove duration field
   - OR: Keep startTime outside, but only calculate duration inside the debug block:
     ```typescript
     if (this.logger?.isLevelEnabled?.('debug')) {
       const duration = Date.now() - startTime;
       this.logger.debug({ duration, ... }, 'Batch complete');
     }
     ```
   - **Chosen approach:** Keep startTime (minimal overhead), only calculate duration inside debug block

**Why this matters:**

- `Date.now()` is not free - it's a syscall on many platforms
- Called on every handler invocation and every worker pool batch
- Debug logging should have ZERO overhead when disabled
- Production runs with LOG_LEVEL=info, so these calculations are wasted work
  </action>
