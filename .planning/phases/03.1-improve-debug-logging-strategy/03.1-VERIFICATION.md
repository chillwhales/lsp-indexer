**Strengths:**

**Architecture notes:**

- DEBUG_COMPONENTS is advisory (documented in JSDoc) — filtering happens post-hoc via jq/grep
- This is intentional: runtime filtering would add overhead to every log call
- Post-hoc filtering is sufficient for debugging (can filter terabytes of logs with jq)
- LOG_LEVEL provides runtime control for overall verbosity

**Import path strategy:**

- Same directory: relative path `./logger` (metadataWorkerPool.ts)
- Cross directory: absolute path `@/core/logger` (handler files)
- Test directory: relative path `../logger` (logger.test.ts)
- Follows TypeScript module resolution conventions consistently
