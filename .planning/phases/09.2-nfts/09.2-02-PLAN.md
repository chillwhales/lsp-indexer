**Step 1: Extract `parseImage` to `packages/node/src/parsers/utils.ts`.**

The `parseImage` function in `parsers/digital-assets.ts` uses a structural `RawImage` interface to parse lsp4_metadata icons and images. NFTs use the exact same lsp4_metadata structure, so this should be shared.

Add to `packages/node/src/parsers/utils.ts` (after existing `numericToString`):

```typescript
import type { Lsp4Image } from '@lsp-indexer/types';

/**
 * Structural interface for raw LSP4 metadata image data.
 * Works with both lsp4_metadata_icon and lsp4_metadata_image types
 * without depending on codegen __typename.
 */
interface RawImage {
  url?: string | null;
  width?: number | null;
  height?: number | null;
  verification_method?: string | null;
  verification_data?: string | null;
}

/**
 * Parse a raw LSP4 metadata image into a clean Lsp4Image.
 * Shared across digital-assets, nfts, and future domains.
 */
export function parseImage(raw: RawImage): Lsp4Image {
  return {
    url: raw.url ?? '',
    width: raw.width ?? null,
    height: raw.height ?? null,
    verification:
      raw.verification_method != null
        ? { method: raw.verification_method, data: raw.verification_data ?? '' }
        : null,
  };
}
```

**Step 3: Create `packages/node/src/keys/nfts.ts`** following the TkDodo query key factory pattern.

```typescript
import type { NftFilter, NftInclude, NftSort } from '@lsp-indexer/types';

/**
 * Query key factory for NFT domain.
 *
 * Hierarchy:
 *   nftKeys.all                              → ['nfts']
 *   nftKeys.details()                        → ['nfts', 'detail']
 *   nftKeys.detail(address, tokenId, include?) → ['nfts', 'detail', { address, tokenId, include }]
 *   nftKeys.lists()                          → ['nfts', 'list']
 *   nftKeys.list(filter?, sort?, ...)        → ['nfts', 'list', { filter, sort, limit, offset, include }]
 *   nftKeys.infinites()                      → ['nfts', 'infinite']
 *   nftKeys.infinite(filter?, sort?, include?) → ['nfts', 'infinite', { filter, sort, include }]
 *
 * IMPORTANT: `list` and `infinite` use separate namespaces to prevent
 * TanStack Query cache corruption between useQuery and useInfiniteQuery.
 *
 * Cache invalidation examples:
 *   queryClient.invalidateQueries({ queryKey: nftKeys.all })           // all NFT queries
 *   queryClient.invalidateQueries({ queryKey: nftKeys.details() })     // all detail queries
 *   queryClient.invalidateQueries({ queryKey: nftKeys.lists() })       // all list queries
 *   queryClient.invalidateQueries({ queryKey: nftKeys.infinites() })   // all infinite queries
 */
export const nftKeys = {
  all: ['nfts'] as const,

  details: () => [...nftKeys.all, 'detail'] as const,
  detail: (address: string, tokenId: string, include?: NftInclude) =>
    [...nftKeys.details(), { address, tokenId, include }] as const,

  lists: () => [...nftKeys.all, 'list'] as const,
  list: (
    filter?: NftFilter,
    sort?: NftSort,
    limit?: number,
    offset?: number,
    include?: NftInclude,
  ) => [...nftKeys.lists(), { filter, sort, limit, offset, include }] as const,

  infinites: () => [...nftKeys.all, 'infinite'] as const,
  infinite: (filter?: NftFilter, sort?: NftSort, include?: NftInclude) =>
    [...nftKeys.infinites(), { filter, sort, include }] as const,
} as const;
```

**Key difference from digital-assets:** The `detail` key includes both `address` AND `tokenId` since an NFT is identified by the composite pair.

**Part A: Create `packages/node/src/parsers/nfts.ts`**

The parser transforms raw Hasura nft response to the clean `Nft` type. Follow the same pattern as `parsers/digital-assets.ts`.

**CRITICAL: Handle nullable lsp4Metadata.** Unlike digital assets where lsp4Metadata is always present, some NFTs may not have resolved metadata yet (the relation can be null). The parser must handle this with optional chaining on the entire `raw.lsp4Metadata` object.

**Type extraction from codegen:**

```typescript
import type { Nft } from '@lsp-indexer/types';
import type { GetNftQuery } from '../graphql/graphql';
import { parseImage } from './utils';

type RawNft = GetNftQuery['nft'][number];
```

**Main parseNft function:**

```typescript
export function parseNft(raw: RawNft): Nft {
  const metadata = raw.lsp4Metadata;

  return {
    address: raw.address,
    tokenId: raw.token_id,
    formattedTokenId: raw.formatted_token_id ?? null,
    isBurned: raw.is_burned,
    isMinted: raw.is_minted,
    // From digitalAsset relation (parent collection)
    collectionName: raw.digitalAsset?.lsp4TokenName?.value ?? null,
    collectionSymbol: raw.digitalAsset?.lsp4TokenSymbol?.value ?? null,
    // From ownedToken relation (current holder)
    owner: raw.ownedToken
      ? { address: raw.ownedToken.owner, timestamp: raw.ownedToken.timestamp ?? '' }
      : null,
    // From lsp4Metadata relation (individual token metadata — may be null)
    description: metadata?.description?.value ?? null,
    category: metadata?.category?.value ?? null,
    icons: metadata?.icon ? metadata.icon.map(parseImage) : null,
    images: metadata?.images ? metadata.images.map(parseImage) : null,
    links: metadata?.links
      ? metadata.links.map((l) => ({ title: l.title ?? '', url: l.url ?? '' }))
      : null,
    attributes: metadata?.attributes
      ? metadata.attributes.map((a) => ({
          key: a.key ?? '',
          value: a.value ?? '',
          type: a.type ?? '',
        }))
      : null,
  };
}

export function parseNfts(raw: RawNft[]): Nft[] {
  return raw.map(parseNft);
}
```

**NOTE on array fields:** Uses the `T[] | null` pattern established in 9.1. `null` = field not included/metadata absent, `[]` = fetched but legitimately empty.

Add JSDoc explaining the metadata resolution model (lsp4Metadata is per-token metadata).

**Part B: Create `packages/node/src/services/nfts.ts`**

Follow the exact same pattern as `services/digital-assets.ts`.

**Builder functions (internal, NOT exported):**

`buildNftWhere(filter?: NftFilter)`:

- `filter.collectionAddress` → `{ address: { _ilike: escapeLike(filter.collectionAddress) } }` (case-insensitive exact match — no wildcards for addresses)
- `filter.tokenId` → `{ token_id: { _ilike: escapeLike(filter.tokenId) } }` (case-insensitive exact match)
- `filter.ownerAddress` → `{ ownedToken: { owner: { _ilike: escapeLike(filter.ownerAddress) } } }`
- `filter.isBurned` → `{ is_burned: { _eq: filter.isBurned } }`
- `filter.isMinted` → `{ is_minted: { _eq: filter.isMinted } }`
- Multiple filters combine with `_and`

**IMPORTANT:** Import `escapeLike` from `./utils` for all string filter fields. Import `orderDir` from `./utils` for sort direction.

`buildNftOrderBy(sort?: NftSort)`:

- `'tokenId'` → `[{ token_id: orderDir(sort.direction, sort.nulls) }]`
- `'formattedTokenId'` → `[{ formatted_token_id: orderDir(sort.direction, sort.nulls ?? 'last') }]` — default nulls last since formatted_token_id can be null

`buildIncludeVars(include?: NftInclude)` — **INVERTED DEFAULT:**

- If `include` is undefined → return `{}` (GraphQL defaults all to `true`, so everything is included)
- If `include` is provided → each field defaults to `false` unless explicitly `true`:

```typescript
function buildIncludeVars(include?: NftInclude): Record<string, boolean> {
  if (!include) return {};
  return {
    includeFormattedTokenId: include.formattedTokenId ?? false,
    includeCollection: include.collection ?? false,
    includeOwner: include.owner ?? false,
    includeDescription: include.description ?? false,
    includeCategory: include.category ?? false,
    includeIcons: include.icons ?? false,
    includeImages: include.images ?? false,
    includeLinks: include.links ?? false,
    includeAttributes: include.attributes ?? false,
  };
}
```
