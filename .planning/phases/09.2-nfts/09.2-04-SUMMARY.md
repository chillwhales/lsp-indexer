## Post-Checkpoint Feedback Fixes

After the initial checkpoint, 6 schema discovery issues required refactoring across all layers:

1. **lsp4Metadata + lsp4MetadataBaseUri dual sources** — NFTs have both direct metadata and baseUri fallback. Parser checks direct first → baseUri second → null.
2. **lsp4_metadata has a `name` sub-relation** — Individual NFTs have names via metadata. Added name field to Nft type and name filter to list.
3. **ownedToken.owner = token HOLDER** — Renamed throughout from `owner` to `holder`/`NftHolder` for semantic clarity.
4. **Collection = full DigitalAsset** — Not just name+symbol. Reuses `parseDigitalAsset` from digital-assets parser. Collection sub-includes are DigitalAssetInclude.
5. **tokenId OR formattedTokenId for single lookup** — Not both required. Stacked vertical inputs.
6. **formattedTokenId filter for list** — Added alongside name filter.
