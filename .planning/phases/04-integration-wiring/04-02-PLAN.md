Create `packages/indexer-v2/src/app/bootstrap.ts` that handles registry creation, plugin/handler discovery, and validation.

**Function: `createRegistry(): PluginRegistry`**

1. **Instantiate registry:**

   - `const registry = new PluginRegistry()`

2. **Discover EventPlugins:**

   - Resolve plugin directory: `path.resolve(__dirname, '../plugins/events')`
   - Call `registry.discover([pluginDir])`
   - Registry will scan for `*.plugin.js` files, load them, validate EventPlugin interface
   - Registry throws on duplicate topic0 (per 04-CONTEXT.md decisions)

3. **Discover EntityHandlers:**

   - Resolve handler directory: `path.resolve(__dirname, '../handlers')`
   - Call `registry.discoverHandlers([handlerDir])`
   - Registry will scan for `*.handler.js` files, load them, validate EntityHandler interface
   - Registry throws on duplicate names or circular dependencies (per existing registry.ts validation)

4. **Structured logging of boot summary:**

   - Use `createStepLogger` from `@/core/logger` with step: 'BOOTSTRAP'
   - Log plugin count: `logger.info({ pluginCount: registry.getAllEventPlugins().length }, 'Discovered EventPlugins')`
   - Log handler count and dependency order:
     ```typescript
     const handlers = registry.getAllEntityHandlers();
     logger.info(
       {
         handlerCount: handlers.length,
         handlerOrder: handlers.map((h) => h.name),
       },
       'Discovered EntityHandlers in dependency order',
     );
     ```
   - Log subscription summary (count of log subscriptions):
     ```typescript
     const subscriptions = registry.getLogSubscriptions();
     logger.info({ subscriptionCount: subscriptions.length }, 'Generated log subscriptions');
     ```

5. **Return registry**

**Error handling:**

- NO try/catch — let registry validation errors (duplicates, cycles, unknown deps) propagate as fatal errors
- Per 04-CONTEXT.md: "Fail fast with detailed messages" — registry already throws with specific error context

**Imports:**

- Import `path` from Node.js
- Import `PluginRegistry` from `@/core/registry`
- Import `createStepLogger` from `@/core/logger`
- Import `Logger` from `@subsquid/logger` for logger type
