# Milestone v1.0: LSP Indexer V2 Rewrite

**Status:** ✅ SHIPPED 2026-02-16
**Phases:** 1-6 (11 phases total: 5 original + 5 inserted + 1 gap closure)
**Total Plans:** 36

## Overview

Complete the V2 rewrite of the LUKSO LSP Indexer by migrating remaining handlers to the EntityHandler interface, building new handlers and metadata fetchers, wiring the full pipeline together, and validating data parity against V1 before production cutover. The phases follow the natural dependency chain: existing handlers must be migrated before new ones can be built, all handlers must exist before integration wiring, and the full pipeline must work before deployment validation.

## Milestone Summary

**Decimal Phases:**

- Phase 3.1: Improve Debug Logging Strategy (inserted after Phase 3 for PR #152 debugging)
- Phase 3.2: Queue-Based Worker Pool Optimization (inserted after Phase 3.1 for throughput improvement)
- Phase 5.1: Pipeline Bug Fix & Missing Core Handlers (inserted after Phase 5 for data gaps)
- Phase 5.2: LSP4 Base URI & Count Parity (inserted after Phase 5.1 for remaining mismatches)
- Phase 5.3: Entity Upsert Pattern Standardization (inserted after Phase 5.2 for pattern bugs)

**Key Decisions:**

- Enrichment queue over populate phase (eliminates entity removal, simplifies FK resolution)
- EntityHandler replaces DataKeyPlugin (unified interface, self-selecting from BatchContext)
- Marketplace removed from V2 scope (high complexity, separate concern)
- postVerification as opt-in boolean flag (keeps all handlers as one type)
- Dual-output logging: Subsquid Logger.child() + pino (independent file rotation)
- Mock store pattern for pipeline integration testing (tracks operations without DB)
- Pipeline contract filter uses case-insensitive address comparison (Subsquid delivers lowercase)
- resolveEntity/resolveEntities replaces 4 ad-hoc patterns (single recognizable pattern)

**Issues Resolved:**

- Case-sensitive address comparison in pipeline.ts silenced 4 entity types (Follow, Unfollow, DeployedContracts, DeployedERC1167Proxies)
- ChillClaimed/OrbsClaimed FK wipe bug (Phase 2 verification created new entities with null FKs)
- lsp5ReceivedAssets cross-batch merge data silently lost (modified DB entities but never called addEntity)
- OwnedAsset double-processing (dual-bag read caused 2x processing without triggeredBy filter)
- LSP4Metadata ~84K missing entities from base URI derivation flow
- OrbLevel/OrbFaction missing mint-time defaults

**Issues Deferred:**

- Full automated V1/V2 comparison test suite with CI integration (DEPL-03)
- Production cutover procedure with rollback plan (DEPL-04)
- Performance benchmarks V2 vs V1 (DEPL-05)
- Subsquid Portal SDK migration (INFR-03)
- Multi-stage Docker build optimization (INFR-04)

**Technical Debt Incurred:**

- 3 phases missing formal VERIFICATION.md (03.2, 05, 05.3) — can be verified during production validation
- 4 pending human runtime tests for Phase 3.1 (logging output verification)
- Compiled .d.ts files in lib/ may contain stale references until next build
