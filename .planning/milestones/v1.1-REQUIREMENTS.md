# Requirements: LSP Indexer — React Hooks Package

**Defined:** 2026-02-16
**Core Value:** Any developer can query LUKSO blockchain data through type-safe React hooks backed by a reliable indexer.

## v1.1 Requirements

Requirements for the React hooks package milestone. All requirements shipped.

### Foundation

- [x] **FOUND-01**: Developer can install package and get working ESM+CJS+DTS builds with `"use client"` directives — ✅ validated
- [x] **FOUND-02**: Developer can run codegen to generate TypeScript types from Hasura GraphQL schema — ✅ validated
- [x] **FOUND-03**: Developer can configure GraphQL URL (HTTP + WebSocket) via environment variable — ✅ validated
- [x] **FOUND-04**: Developer can wrap app in `<IndexerProvider>` with optional existing QueryClient — ✅ validated
- [x] **FOUND-05**: Developer gets typed `IndexerError` with network, GraphQL, and Hasura permission error categories — ✅ validated
- [x] **FOUND-06**: Developer can import from `@lsp-indexer/react` (client hooks), `@lsp-indexer/node` (server), and `@lsp-indexer/types` (types) without bundle contamination — ✅ validated

## Out of Scope

| Feature | Reason |
| --- | --- |
| Custom cache layer | TanStack Query handles caching |
| Apollo/urql adapters | Package uses typed fetch |
| Complex query composition/joins | Consumers compose hooks; Hasura handles relationship joins |
| Schema watching/hot reload | Codegen is build-time |
| Mutation/write hooks | Indexer is read-only |
| Mobile-specific hooks | React Native support deferred; web-first |
